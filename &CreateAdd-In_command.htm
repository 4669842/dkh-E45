&CreateAdd-in

Syntax    

[{&CreateAdd-in}]

Example Scripts

Description

Creates the Add-in defined in the command.

Add-Ins can only be created a file via scripting.

Parameters

Agenda-folder="<TargetFolder>"

Agenda folder is displayed within the case. Sets to value of AI06 to the database menu folder the Add-in database should be placed in <folder name/subfolder name>. If not specified, the Add-in agenda shows on the root of the tree view.

A subfolder has to be specified (and hence will be created) when using agenda-folder parameter.

case-ref

If you are creating an Add-in for a case that is not the current owner.

Database-folder

Database menu folder is displayed within the case.

Sets to value of AI08 to the agenda menu folder the Add-in agenda should be placed in <folder name/subfolder name>. If not specified, the Add-in database shows on the root of the tree view.

Noagenda

No agenda folder is displayed within the case. Sets the value of AI07 to False to hide the Add-in agenda.

Nodatabase

No database menu folder displayed within the case. Sets the value of AI05 to False to hide the Add-in database.

Noscript

When the Create Add-In for Matter script is not to be run.

ref

Where an Add-in has been selected or created with reference.

type

Add-in code.

Example Scripts

1 Create an Add-in for a file

This example creates the money laundering Add-in and assigns it to the current file, giving it a reference of myMONEYL.

[&CreateAdd-In type="MONEYLAUNDERINGAA",ref="myMONEYL"]

2 Create an Add-in for a file and prevent the Create Add-In script running

This example creates the money laundering add-in and assigns it to the current file. Following this, the noscript parameter prevents the Create Add-In script running.

[&CreateAdd-In type="MONEYLAUNDERINGAA",noscript]

3 Using the case-ref parameter

This example creates the M2 add-in for all of the files that are linked to the client:

[&ForEach CLIENT_FILE]

   [&SelectFile code=CLIENT_FILE.MT05,ref="myFILE"]

   [&CreateAdd-In type="MATTERMNGTAA",case-ref="myFILE"]

[&EndFor]
