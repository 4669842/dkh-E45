&Addressbook

Syntax    

[&Addressbook search = "<SearchString,

Example Scripts

Description

To perform a search of the address data, regardless of the system in use, a common interface is defined. The search is always performed using the &Addressbook command which is used exclusively with the postcode database and is not related to the Visualfiles address book.

The postcode integration is driven by the &AddressBook command using two main options, search=<search text> and result=<field list>.

The search= parameter can take a single address item such as a postcode, street, town or organisation name. Alternatively, you can enter a two part search in the format, postcode comma house number.

The result= parameter must contain a comma delimited list of up to 12 local (LF), temporary (TF) or screen field codes that contain the selected address details.

AFD Postcode Integration

The &Addressbook command enables you to develop your integration with AFD Postcode Everywhere XML. Make use of the AFD parameters to report on DX data and other information you need.

Additional data is provided by the AFD Postcode Integration than is displayed in the default view provided to the user in the Select Address dialog. You can make use of this data, which includes a unique key for the address in AFD or the absolute coordinates of the address on a map, through scripting to export the data to an xml file.

The &Addressbook result= parameter includes two options related to the AFD integration; the first holds the DX Number for the selected address and the second holds the DX Exchange. These fields enable you to specify which fields are returned from AFD (this is dependent on the type of licence you have with AFD). 

For backwards compatibility and to ensure existing Visualfiles scripting can execute against AFD without modification, the result= parameter has been modified to work with AFD data (returned as XML in the XL01 field).

Note: All &Addressbook commands in existing scripts will work as before.

&Addressbook Status Property

Use the status property for the &AddressBook command to warn you if the command fails, e.g.:

[&Message status(Addressbook)]

Calling AFD Postcode

When the &AddressBook command is executed, a call is made to the AFD Server passing in the serial number, postcode and search string. Where multiple results are returned the Select Address screen is displayed. Where a single result is returned the address data is passed back silently to the script.

Select Address Screen

The number of results displayed defaults to the Max Rows setting specified in the Maintain Postcode Integration dialog when you Configure Postcode Integration.

When using AFD the content of the List field for the selected row is displayed in the grid

The OK button is only enabled when a row is selected in the grid. Click OK to return selected address to the script.

When Cancel is clicked the dialog is closed without returning an address to the script. This is the same behaviour as pressing the Close button 'X'.

Parameters

nodisplay

Prevent the Finder postcode lookup from displaying if no results are found.

result=

A comma-delimited list of Screen Fields (SF) that define where to return the search result.

The following additional options are available if you are integrate with AFD Postcode Everywhere XML:

Search=

Contains the string to search for: a field on a screen or a temporary field.

AFD Everywhere software returns data from a programmatic search in XML file format. Visualfiles extracts the data from the XML to display the information in the Select Address screen. The user is unaware of the XML. Developers can build a fully functional integration without ever using the XML. However, if you wish to return additional data for each address, you can work with the XML results.

To return the full XML results into your Visualfiles script you can specify the output="XL01" parameter on the &AddressBook command. This loads the full document into the XL fields which you can then parse. There are useful nodes that can be returned on the &AddressBook command:

  • Key="LF02" returns the unique address identifier held in the <Key> node into a Visualfiles field. This could be useful to store in Visualfiles alongside the address and potentially used to identify duplicate records.

  • List="LF03" returns the formatted address string held in the <List> node into a Visualfiles field, this information is displayed when an address is highlighted in the Select Address screen (See Figure 4).

Extending the &AddressBook Command

The following parameters can be added to the &AddressBook command as comma separated items:

AFD Parameters

Note: AFD parameters are only applicable to the AFD integration. The fields that can be searched on depend on the type licence you have purchased from AFD.

Note: When any of the AFD parameters below are specified on the &AddressBook command, theSearch=option is ignored.

host=

Override the default host setting.

serial=

Override the default serial number.

password=

Override the default password.

maxquantity=

Override the default maximum number of results.

output=

Load an XL field with the complete XML result for the selected address (for subsequent parsing in the script or saving as an XML file).

list=

Return the List field from the address data (the formatted address as a comma delimited list).

key=

Return the Key field from the address data (the unique ID for the address).

postcode=

Search on postcode. Must be entered as either a local, temporary or screen field.

organisation=

Search on organisation name.

street=

Search on street name.

town=

Search on town.

forename=

Search on first name.

surname=

Search on last name.

Example Scripts

1 Sample search

[&Addressbook search="BOO05",result="SF01,SF02,SF03,SF04,..."]

2 Prevent the Finder postcode lookup display; nodisplay parameter

[&AddressBook search="LF30",result="LF31,LF32,LF33,LF34,LF35,LF36,LF37,LF38,LF39,LF40,LF41,LF42", nodisplay]

Run the script with LF30 set to a postcode value which returns no results and Postcode Finder is not displayed.

3 Search returning all the default data for the selected address into local fields

[&Assign LF99 = "LS6 2UU"]

[&AddressBook search="LF99",result="LF01,LF02,LF03,LF04,LF05,LF06,LF07,LF08,LF09,LF10,LF11,LF12,LF13,LF14"]

4 Search returning all default data for the selected address

Search returning all default data for the selected address, the full XML, Key and List:

[&AddressBook search="LF01",output="XL01",key="LF02",list="LF03"]

5 Find the first 5 addresses

To find the first five addresses for Starbucks in Leeds:

[&Assign LF99 =25"]

[&AddressBook result="LF01,LF02,LF03,LF04,LF05,LF06",MaxQuantity ="LF99", Town="Leeds", Organisation="Starbucks"]

Note: If the MaxQuantity parameter is not included in the &AddressBook command the number of records returned defaults to the maximum number specified in the configuration screen.  You can use TF fields in the result parameters and assign the data to database fields in the usual way.

6 Find all addresses for postcode area

Find all the addresses for a postcode area:

[&Assign LF99 ="Enter postcode here"]

[&AddressBook Postcode="LF99",result="LF01,LF02,LF03,LF04,LF05,LF06"]

Note: When using the Postcode parameter the search criteria must be a local, temporary or screen field. Actual values are not permitted with this parameter as they could be confused with genuine Visualfiles screen fields.

7 Select an address by surname

To select an address for people with surname Smith in Huddersfield:

[&AddressBook Town="Huddersfield",Surname="Smith",result="LF01,LF02,LF03,LF04,LF05,LF06"]

Note: This example does not return the correct results if the AFD Licence is the Postcode Plus version as Names and Numbers are not enabled in this instance.

Note: All parameters on the &AddressBook command require values to be passed in as a local or temporary field (LF01, TF01) or as a screen field (ABC01). The only exceptions are the new AFD search parameters (Organisation, Town, Street, Surname, Forename) which can be entered as string values but the values must be entered within quotes (e.g. ”Leeds”).

8 Return selected address to an XL field XL01

[&Assign LF99 = "Enter postcode here"]

[&AddressBook result="LF01,LF02,LF03,LF04,LF05,LF06", Postcode ="LF99", output ="XL01"]

Note: Normally when returning address data from AFD the number of fields returned is restricted to those fields we report on, this helps reduce the volume of data that is passed through the network. If the output= parameter is used then all available fields for the selected address are read in as part of the XML file and are copied into the XL field specified. The extra data available includes Grid References, Census data, etc.

9 Combine search parameters

Combine several parameters in one search:

[&Assign LF99="S70 6RN,18"]

[&Addressbook search="LF99",result="LF01,LF02,LF03,LF04,LF05,LF06"]

Note: If you use any of the other &Addressbook command parameters such as postcode, town, organisation etc it will not work as it builds up a different search string which does not allow for postcode and house number.

10 Return the standard 14 address components

Return the standard 14 address components and the <AuthorityCode> into LF90 and <Affluence> into LF91:

[&Addressbook search="LF99",result="LF01,LF02,LF03,LF04,LF05,LF06,LF07,LF08,LF09,LF10,LF11,LF12,LF13,LF14",UserFields="AuthorityCode,Affluence",UserResults="LF90,LF91"]

Note: If you wish to retrieve additional data without having to navigate the XML, you can specify the relevant tags using UserFields="<ListOfTags>" and UserResults="<ListOfVariables>".

11 Return XML using a button on a screen

Here is an example of how to use the &Addressbook command in a working system.

You may want to make use of the XML data, such as a unique key for the address in AFD, or the absolute coordinates of the address on a map.

You can add a button is to the Data Input Screen on a Commercial Entity to allow the user to search the AFD software each time a new address is created in Visualfiles using the standard entity set.

Script Behind the Postcode Button on the DIN Screen

[&Assign LF99=""]              *** Postcode

[&Assign LF01 = ""]            *** Company Name/Building Name/Sub-Name

[&Assign LF02 = ""]            *** Formatted Address

[&Assign LF03 = ""]            *** Locality

[&Assign LF04 = ""]            *** Town

[&Assign LF05 = ""]            *** County

[&Assign LF06 = ""]            *** Postcode

[&Assign LF07 = ""]            *** House Number

[&Assign LF08 = ""]            *** House/Building Name

[&Assign LF09 = ""]            *** Sub-Name

[&Assign LF10 = ""]            *** Organisation Name

[&Assign LF11 = ""]            *** PO Box

[&Assign LF12 = ""]            *** Street Name

[&Assign LF20=""]              *** Key

[&Assign LF21=""]              *** List

[&Assign LF30=""]              *** Longitude

[&Assign LF31=""]              *** Latitude

[&Assign LF32=""]

[&If DIN21=""]                 *** Postcode field on the screen is blank

   [&UserInput prompt="Enter Postcode",result=LF99]

[&Else]

   [&Assign LF99=DIN21]        *** Use postcode in postcode field on DIN screen if not blank

[&EndIf]

                               *** Run the Addressbook command returning all address information into local fields and the whole XML result into XL01

[&Addressbook postcode="LF99",result="LF01,LF02,LF03,LF04,LF05,LF06,LF07,LF08,LF09,LF10,LF11,LF12,LF13,LF14",output="XL01"]

                                *** Check for any errors with the command

[&If status(Addressbook) <> ""]

  [&Message "There is an issue with searching the for the Address please see your Administrator and provide the following details:"+status (Addressbook)]

  [&Quit]

[&EndIf]

[&If LF06<>""]                        ** Verify the Postcode Returned isn't blank and populate the DIN screen

  [&If LF09<>"" &And LF08<>""]

    [&Assign DIN15 = LF09 + ", "+LF08]

  [&Else]

    [&If LF09="" &And LF08<>""]

      [&Assign DIN15 = LF08]

    [&Else]

      [&If LF09<>"" &And LF08=""]

        [&Assign DIN15 = LF09]

      [&EndIf]

    [&EndIf]

  [&EndIf]

  

  [&Assign DIN16 = LF07]

  [&Assign DIN17 = LF12]

  [&Assign DIN18 = LF03]

  [&Assign DIN19 = LF04]

  [&Assign DIN20 = LF05]

  [&Assign DIN21 = LF06]

  [&Assign DIN05 = LF13]

  [&Assign DIN09 = LF14]

[&EndIf]

*** Parse Returned XML for Langitude,Latitude Elements and UDPRN Number

[&Select xl,type=XL01,element=AFDPostcodeEverywhere,first,parent]

[&DoWhile status(select)=""]

  [&Select xl,type=XL01,element=item value,first,parent]

  [&Assign LF30=XL01(element=Latitude,child)]

  [&Assign LF31=XL01(element=Longitude,child)]

  [&Assign LF32=XL01(element=UDPRN,child)]

[&EndDoWhile]

[&Message "The UDPRN Number which is used by Solicitors in Conveyancing Systems as a unique reference for the property :"+LF32]

[&Message "Google Map Location based on Langitude and Latitude"]

[&Open "http://maps.google.com/maps?q="+LF30+",+"+LF31]

12 Retrieve AFD address information and additional XML parameters

   [&Assign LF99=DIN21]        *** Use postcode in postcode field on DIN screen if not blank

      [&EndIf]

  [&EndIf]

  

  [&Assign DIN16 = LF07]
