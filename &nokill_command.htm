&NoKill

Syntax    

[&Nokill]

Description

Prevents a Schedule item being removed from the Schedule list once actioned.  This command is commonly used to set regular file reviews or re-schedule a process.

There are no parameters for this command.

Example Scripts

1 Re-scheduling an action (1)

The Review File script checks whether files have been reviewed. If today’s date is later than a date held in database field REV07, the &NoKill command prevents the schedule being automatically removed by the Review File script (GEN-SO25). The review of the file is then scheduled to occur again on today's date plus two weeks.

[&If REV07<Date]

   [&Assign REV07 = Date+2(Weeks)]

   [&Schedule GEN-S025,"Review file",date-due = REV07]

   [&Nokill]

[&EndIf]

2 Re-scheduling an action (2)

In this example rather than checking a database field, a question is asked to trigger the re-scheduling.

[&If 'Meeting Date has passed. Do you wish to re-schedule reminder...'(Question) = "Yes"]

   [&Schedule GEN-SO16,"REMINDER: Meeting Due",date-due = today+5(working days)]

   [&Nokill]

[&Endif]
