&Quit

Syntax    

[&Quit]

Example Scripts

Description

Allows a script or document to be ended before it has fully executed.  &Quit can be used within the &Route command (where the system checks to see if a file may move from one milestone to another), or in the Open File script to check, for example, if the user has the relevant permissions to view the file.

Parameter

no-status

When using the &Quit command in a system script, this message will display: Document quit command. This message will not display in a non-system script. Including the no-status parameter with the &Quit command in system scripts, will enable the script to quit without the message displaying.

Another use of the no-status parameter within non-system scripts is shown below in Example 3, where the no-status parameter allows all processing to continue in a script that has called another script containing a &Quit command.

Example Scripts

1 &Quit used with milestones

This example shows a file progressing from the Instructions Received milestone to the Issue milestone. If the first condition is true, the message is output and the &Quit command ends the script. However, if it is false, the file is progressed to the Issue milestone from the Instructions Received milestone.

[&If MPR05<"01/01/1980"]

   [&Message "The LBA has not been sent"]

   [&Quit]

[&Else]

   [&If MPR09 = "No"]

      [&Message "We do not yet have authority to issue"]

      [&Quit]

   [&EndIf]

[&EndIf]

2 &Quit used with the Open Case script

This command may also be used in the Open File script, where the file will not be opened. This allows a developer to apply security to individual files using conditional statements in the Open File script. SCY06 is a field on a user-definable screen which determines whether or not the file is secure: if it is secure and the user is not the File Holder, access will be denied.

[&If SCY06 = "Yes" &And US04<>MT11]

   [&Message "You are not authorised to open this file"]

   [&Quit]

[&EndIf]

3 &Quit used with the no-status parameter

This is an example of the &Quit command with and without the no-status parameter.

Without no-status parameter

 Script A

[&Call SCRIPTB]

[&Message “Test1"]

Script B

[&Message “Test2"]

[&Quit]

Script A immediately calls Script B, outputs the message “Test2" and quits processing. Therefore processing does not return to Script A to output the message “Test1".

With no-status parameter

Script A

[&Call SCRIPTB]

[&Message “Test1"]

Script B

[&Message “Test2"]

[&Quit no-status]

This time, Script A immediately calls Script B, outputs the message "Test2" and returns the processing to Script A to output the message "Test1". Including the no-status parameter with the &Quit command has enabled the processing in the scripts to continue running to the end.
