&Message

Syntax    

[&Message "<MessageText>"]

Example Scripts

Description

Displays a message to the user during the production of a document or running a script. The message is displayed in a message box with an OK button to dismiss the message. &Message commands can be a combination of text and data codes.

Basic &Message commands produce a simple text expression in a dialog box, such as a prompt or reminder. For example:

'Please select from the following list' or 'Please enter the address in full'.

Parameters

next-action={text_expression}

The next–action parameter is used to end the script run at this point. Any code after the &Message command is ignored and the script specified by the text expression in the next-action parameter is run. See example 7.

Text expressions are used to name the required script and can be constructed from any combination of the following:

  • "constant text", such as a script name
  • 'User input'
  • Database fields
  • LF, TF, PI fields. The temporary field specified will contain the value entered by the user.

Note: For more information on stateless scripting options go to no-wait and next-action Parameters, Visualfiles Browser Environment.

next-action-inputs={expression};{expression}...

List of semi-colon delimited items to be passed as PI fields into the next-action script.

no-wait

Ends the script run at this point.

When a script that includes the no-wait parameter is run and there is no user response to a screen prompt, the no-wait parameter is executed which ends the script, ignoring all subsequent code. This results in agents that would otherwise be tied to the unused application, being accessible to other system users. See example 8.

title = "<TextString>"

This parameter adds a user-defined title to the message box window.

Example Scripts

1 Producing a basic message

[&Message "Please enter contact details"]

2 Using data codes in a message

This code will incorporate the value of DC10 (Issue Date) into the message.

[&Message "Warning - Proceedings issued on " + DC10]

3 Capitalising text in a message

This code will capitalise the message text.

[&Message "proceedings issued by " + DC12:caps]

4 Displaying the results of a calculation in a message

This code will multiply the two temporary fields together and display the product in the message box.

[&Message TF01*TF02]

5 Adding a title to a message box and incorporating a message

The code will add a title, Check Paper, to the message box.

[&Message "This Form must be Printed on Yellow Paper",title = "Check Paper"]

6 Message prompt format display in dialog

Use the code below to display values in a column.

[&Message TF01 + "<CR>" + TF02 + "<CR>" + TF03]

or

[&Message LF01+"
"+LF02+"
"+LF03+"
"+LF04+"
"+LF05]

Use the code below to display values in a line.

[&Message LF01+""+LF02+""+LF03+""+LF04+""+LF05]

7 next-action parameter

The next -action parameter ends the script run at this point. Then runs the specified script (for example, AfterMessage) when the user clicks the OK button in the message dialog it is attached to. So in the script below [&Assign TF01] will not be executed:

[&Message "TEST1 of next action script",next-action="AfterMessage"]

[&Assign TF01]

Note: No variables are available when the script is run, TF, LF etc.

8 no-wait parameter

The no-wait parameter ends the script run at this point. In the script below [&Assign TF01] will not be executed:

[&Message "Message nowait",no-wait]

[&Assign TF01]
