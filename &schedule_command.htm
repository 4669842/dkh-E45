&Schedule

Syntax    

[&Schedule]

Example Scripts

Description

Creates an item in the schedule of a file. The schedule item can be a simple text reminder or can be used to run a document or script due on a specified date.

Add-ins

When scheduling an item using the group parameter where the item to be scheduled and the group it is being scheduled for, are on separate add-ins, the add-in code must prefix the group code, see example 6.

Parameters

allgroup

Action the schedule item against all members of a group. Can only be used in conjunction with the group (groups) parameter.

Note: When this parameter is used a warning message may appear if the option, Fields to display when selecting items from a group, is not specified in the Group Menu Properties dialog, in the Database screen. For more information please refer to Developing Group Folders.

category = <CategoryCode>

Allocate a category to the schedule item (used to filter information in both the To-Do List and the History Screen).

code = <RelationName.EN56>

Create a schedule for each item identified by a relation within an &ForEach loop. EN56 is an internal read-only code assigned by Visualfiles and is used as a unique identifier for any new entity.

comments=

Create schedule with comments attached. See example 10. See Schedule Codes AD33 Comments.

Comments can be added but not deleted (the syntax [&Assign AD33=””] would do nothing). A value has to be assigned to AD33 and this is added to the end of existing comments on the schedule.

date-due = <DateDue>

Specify the date the schedule item will run.

The date-due may be defined in terms of the system date or a date specified in a database field. Specified as before/after a period of time; for example,  seven days from the date inserted. If no date-due is specified, the schedule item defaults to the date inserted.

Note: The date due must be a working day.

document

Enables history documents to be attached when the schedule is created. The document parameter needs a file name attaching to it; this can be assigned directly, or via TF/LF fields or via HS24 or HV11. See example 9.

This parameter enables History documents to be attached to scheduled items, either manually or via scripting, which means that end users can review and update the documents directly from within the To-Do list.

Note: The user access right Attach History Documents needs to be selected to enable you to use this parameter and attach history documents to schedule items in the Schedule Screen.

escalate

Allocate the schedule item to the File Holder's supervisor.

fee-earner = "<FileHolderName>"

Assign the File Holder to whom the schedule item is allocated. The default is to assign the schedule item to the File Holder for the file.

group = "<GroupCode>"

Assigns schedule item to previously selected member of a group.

To select a member of a group, view the group screen from the database tree or present the user with a choice using the &Select command immediately before the &Schedule command. If a member of a group is not selected, the user will be presented with an error message.

group = "<RelationCode>"

Schedule an item to each iteration of a relation.

When the script is actioned, a dialog will display, enabling users to select the entry against which the schedule item will be attached.

post-tray = <PostTrayName>

Assign a Document Review into which the schedule item will be stored after action.

priority = <PriorityLevel>

Assign a priority to the schedule item (Normal, High, Critical).

This adds a schedule item with the corresponding priority colour and allows the user to filter the to-do list according to the assigned priority.  If not specified, the schedule item will be assigned the priority of normal.

PY-Record = <PaymentRecordNumber>

Link the schedule item to a payment.

robot = <RobotName>

Allocate the schedule item to a robot user.

schedule-code = <ScheduleCode>

Allocate a code to the schedule item.

Assign a unique code to the schedule item. If required, the schedule-code may then be referenced by &Kill commands to remove the schedule item.

time-due

Specifies the time the schedule item is to be run.

user-data1 = "<UserData1>"

Add user data (AD30) to the schedule item (optional). See example 11.

user-data2 = "<UserData2>"

Add user data (AD31) to the schedule item (optional). See example 11.

"<String>"

Describe the schedule item (this text is included in the To-Do list and the Schedule).

If no text is entered, the description of the document or script being actioned will show instead.

Example Scripts

1 Date-due parameter

Schedule the item, S-RECD-1, to run one working day after the date inserted.

[&Schedule S-RECD-1,"New Support Call",Date-due = today+1(Working Days)]

Schedule the item, L-Cl-LT, to run 4 days after the date assigned to CRD08.

[&Schedule L-Cl-LT,"Write to Client",Date-due = CRD08+4(Days)]

Schedule the prompt "Review File", with no associated action, for 1 week after the date assigned to TF01.

[&Schedule ,"Review File",Date-due = TF01+1(Weeks)]

2 Date-due parameter, working day

[&Schedule ,"Test",date-due=today+6(Days),work-date]

3 Priority parameter

Schedule the item, S-RECD-1, to run, with a priority of high.

[&Schedule S-RECD-1,"New Support Call",Priority = High]

4 Escalate parameter

Schedule the message Completion Overdue one working date after the date held in COM01 to the supervisor of the File's File Holder.

[&Schedule,"Completion Overdue", Date-due = COM01+1(Working days),Escalate]

5 Group parameter

Schedule the document, D-BOO-CON, to be produced for user-selected members of a group called CONT.

[&Schedule D-BOO-CON,"Produce confirmation for contact",Group = "CONT"]

6 Group parameter with add-in

In this example the schedule is created from the Sale Instructions (SALEINSTAA) add-in for an iteration of the adult occupier group screen (GAOC).

The adult occupier group screen is on the Sale add-in (SALEAA).

[&Schedule L-ADO-REQCONS,Adult occupier - request consent,

7 allgroup parameter

Schedule the document, D-BOO-CON, to be produced for each member of a group called CONT.

[&Schedule D-BOO-CON,"Produce confirmation for contact",Group = "CONT",Allgroup]

8 Post-tray parameter

[&Schedule ,post-tray=default]

9 Document parameter

[&Schedule CASETEST,"Test Hist Doc2",date-due=today+7(Days),document=HS24,schedule-code="TEST"]

10 Create schedule with comments attached

[&Schedule COMMENTTEST,"test",date-due=today+1(days),priority=Normal,comments="Additional comments"]

11 Add user data

[&Schedule DEED-22392603,"test34343443",date-due=today+0(days),work-date,user-data1="A",user-data2="B"]

Developers' Tip

Avoid situations where a document name is the same as the schedule note description; the results may be unpredictable.
