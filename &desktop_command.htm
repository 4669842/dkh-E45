&Desktop

Syntax    

[&Desktop item = "<ItemType>",code = "<ItemCode>"]

Example Scripts

Description

Enables a user to manually add items to a Desktop folder when the System Default, Auto Save Items to Desktop, has been disabled.  

When this option is enabled, Entities, Roles, File types, Files and Entity Types are automatically saved to the Desktop on close. Disabling this option and using the &Desktop command gives the user direct control over which items are saved to the Desktop at the expense of extra scripting.

All values for parameters for the &Desktop command must be of the format 'text'.

Parameters

code = "<ItemCode>"

Code for the item to be saved to the Desktop.

delete

Delete a Desktop item for the current user.

deleteall

Delete a Desktop item for all users.

duplicate = "<Allow>"

Set this parameter to Yes to enable duplicate Desktop items to be saved. The default is No.

folder = "<FolderName>"

Name of the folder to which the item should be saved. If this parameter is not specified, the default is to save to the Desktop.

group = "<GroupCode>"

Name of the group to which the item should be saved.  If the group specified does not exist, the default is to save to the Desktop.

item = "<ItemType>"

Choose from: Case, MType, Entity, Entity-Type, Role, System Script, and Icon.

The value Icon does not work for the current user. The Icon value will only work if either the user or template parameters are included in the command.

same-folder = "<Allow>"

Defaulting to No, changing this parameter to Yes allows duplicate items to be saved to the same folder as long as the duplicate parameter is also set to Yes.

template ="<TemplateName>"

Update the Desktop of all users using this template. This parameter will only work if the users using this template are not logged on. To update the Desktops of users who have a particular template assigned, use the value of US01 for the template record. Where a template parameter is specified, any User parameter is ignored.

user = "<UserId>"

Update the Desktop of this user. This parameter will only work if the user is not logged on.

Example Scripts

1 Saving a file to the Desktop using the file internal code

[&Desktop Item = "Case", Code = MT30]

2 Prompts the user to choose which folder to save to

[&Desktop Item = "Entity", Code = "EN56", group = "UserGroup" , folder = 'Save to'(choose "Folder One":"Folder Two":"Folder Three")]

3 Allowing duplicates

The value for a parameter can be taken from a database field. Where the database field, UPF01, holds the user preference for whether duplicates should be allowed (Yes or No), this example will allow duplicate items across different folders, conditional on that preference.

[&Desktop Item = "Entity", Code = "EN56", duplicate = UPF01]

4 Allowing duplicates in the same folder

Allows multiple instances of the file to be saved to the My Folder folder in the groupNew Group.

[&Desktop Item = "Case", Code = MT30, Group = "New Group", Folder = "My Folder", Same-Folder = "Yes", Duplicate = "Yes"]

5 Delete a Desktop item

[&Desktop Item="CASE",CODE=MT30,delete]

6 Use a text file to feed values to the &Desktop command

A comma-delimited text file, containing the data required by the &Desktop command, can be used with to update user desktops automatically. If the field structure in the file was: User, template, item type, item code, group, and folder, the syntax could be:

[&if if02 <> ""]

   [&Desktop Item="IF03", Code="IF04"(text), template=IF02, Group="IF05", Folder="if06", duplicate="yes", same-folder="yes"]

[&else]

   [&Desktop Item="IF03", Code="IF04"(text), user=IF01, Group="IF05", Folder="if06", duplicate="yes", same-folder="yes"]

[&endif]

7 Add an icon to a toolbar

To add an icon to the toolbar of a user (or template), the syntax would be:

[&Desktop Item="Icon", Code="Create New Party", user = "PMarsh",  Group="Desktop", duplicate="yes", same-folder="yes"]
