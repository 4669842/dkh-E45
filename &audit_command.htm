&Audit

Syntax    

[{&Audit action =} "[create|amend|delete|admin]"]

Example Scripts

Description

Adds audit entries as well as performing administration actions such as deleting and/or exporting to a text file audit entries.

Parameters

action = "[create|amend|delete|admin]"

The action of the audit. The user can assign an action. If the action is admin the functionality to delete and/or export audit entries to a text file is available. If not, the user can only add an audit entry in conjunction with the type and code parameters.

audit-type = "[matter|entity|role|history|ledger|link|document|budget|user|schedule|audit]"

The audit type of the audit. The parameter can be used only if the action is admin.

code = <IntCode>

The internal code of the item being audited. This can be MT30 – file code; EN56 – entity code; HS26 – history ID; LG22 – ledger number. This is used in conjunction with the action and type parameter.

delimiter = "<Delimiter>"

The delimiter to be used in the output file. This parameter is optional and can be used only if the action is admin.

field-code = "<DBFieldCode>"

The database field that the audit is recording. This parameter can be used only if the rec-type is field.

file = "<FileName>"

The output file to export the audit record to if export has been selected as a value for the perform parameter. This parameter can be used only if the action is admin.

filetype = "<text>"

Specifies the file type for the output file. The audit can only be exported to a text file. The parameter is optional and can be used only if the action is admin.

header = "yes"

Used to export the column headers as well as the audit data. This option can also be selected when exporting audit records from the Audit Viewer.

perform = "[delete|export|export&delete]"

The action to be performed when the command is executed.  Delete will delete the audit record. Export will export the audit record to the text file specified. Export & delete will export the audit record to the specified text file and then delete the audit record. The parameter can be used only if the action is admin.

rec-action = "[amend|delete|create]".

The record action of the audit. This parameter can be used only if the action is admin.

rec-type = "[record|field|link]"

The record type of the audit. The parameter can be used only if the action is admin.

type = "[matter|entity|history|ledgertype]"

The type of audit: used in conjunction with the action and code parameters.

Example Scripts

1 Add audit entry for file

The script could be placed in the File Open script to add an audit entry every time the file is accessed.

[&Audit action = "File opened",type = Matter,code = MT30]

2 Add audit entry for history

The script will add a history item and an Audit entry.

[&History "Order Authorised: Finance Order Number - "+PSU13]

[&Audit action = "Create history",type = History,code = HS26]

3 Add audit for entity field amended

The script is attached to a field and will add an Audit entry when the field is amended.

[&Audit action = "Amend name",type = Entity,code = EN56]

4 Add audit entry for ledger

The script will add a Ledger item and an Audit entry.

[&Post ISSU,noconfirm]

[&Audit action = "Add ledger",type = Ledger,code = LG22]

5 Export an entity audit to a text file

The script will add an Audit entry and export it to a text file.

[&Audit action = "Admin",perform = "Export",audit-type = "Entity",rec-type = "Record",rec-action = "Create",file = "C:\AUDITDELETE.txt,delimiter = ,,

6 Export an entity audit to a text file and delete the audit entry

The script will add an Audit entry, export it to a text file, then delete the audit entry.

[&Audit action = "Admin",perform = "Export & Delete",audit-type = "Entity",rec-type = "Field",rec-action = "Amend",field-code = "EN02",file = "C:\AUDITDELETE.txt,delimiter = ,,

Developers' Tips

  • If the action is admin it is possible to run the script non-silent or silent.  If running it non-silent, a window displaying the status of the records being processed and, if Export & Delete or Delete has been specified, will prompt for confirmation before deletion.  After the script has executed, a message will be displayed showing statistical information.  The option is available from the Command Builder on the Admin tab.
  • To export and/or delete audit entries the user privilege Allow Audit Deletion must be enabled.
