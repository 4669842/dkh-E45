Email Server Commands

Syntax    

Command = <CommandName>

Script = <ScriptName>

Description

All email server commands are entered in the subject field of the incoming email.

Quotation marks are not necessary unless the values for parameters contain spaces.

Parameters

command = get

Retrieve file(s) from the attachments directory and return to the calling email address without running a script.

command = get parameter options include:

return-file = <FileName1.txt>

Name(s) of the file(s) to be returned.

subject = "<String>"

Text for the Subject field of the email.

command = <logfile>

Return a log file as an attachment.

command = <script>

Remotely run a system script and return the result to the calling email address.

command = script parameter options include:

script = <{VisualfilesSystemScript|message}>

Script can be either a system script or the contents of an email message field.

script-output = <{attachment|message}>

Return the result of the script to the calling email address as either an attachment or an email message.

The expression script-output = attachment will always return as script.txt. In the example above, the text 'You are logged in as Visualfiles user [US01]' is returned as the email message.

return-file = <FileName1,FileName2,FileName...>

Return a file, generated by a script, to the user.

subject = "<String>"

Specify the subject field text of the returned email. If not specified, the text 'Visualfiles Email Server - Script Completed' is used.

command = test

Will return an email to the calling email address to confirm that the email server is running - no parameters are required.

command = verification

Specify verification for the sending address.

The email server will generate a unique code which will be returned to the calling email address. This code must then be specified as a parameter of the next command sent from the email address to the Visualfiles server.

The email server will not process any commands for a user with verification switched on who does not include a verification code.

Example Scripts

1 Command verification

Send an email to the server with the command command = verification in the Subject field. The email server will reply with a verification code. Copy/paste the verification from the server into the subject field of a new email and enter the command details. Send the email to the server to be processed.

2 Run a Script with Email Verification Enabled

  1. Send an email to the server with the subject of Command=verification.

  2. Wait for the email server to reply with a verification code, for example: verification=irjflbhreoivbwofkrg.

  3. Copy and paste the verification code from the returned email into the subject line of a new email.

  4. Enter the command details, for example:

Command = script,script = test,verification = irjflbhreoivbwofkrg.

  1. Send the email back to the email server to be processed.
