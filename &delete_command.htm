&Delete

Syntax    

[&Delete <GroupCode>]

Example Scripts

Description

Deletes the most recently selected instance of a group screen and all associated schedule items.

Group screens from a linked matters (linked Sale and Purchase Applications [&Delete P_GSAM] or [Delete

Note: A_ is the linked case prefix to identify which case type the group screen ABCD belongs to.

Parameters

AD

Delete Schedule items.

HS

Delete history items.

LG

Delete Ledger items.

RC

Delete Recoverable Cost items

History-No ="<History Number>"

Delete the history item referenced by the history ID (HS26). See example 4.

Ledger-No ="<Ledger Number>"

Delete the ledger item referenced by the Ledger No (LG22). See example 5.

Record-No ="<Record Number>"

Delete the recoverable costs item referenced by the Record Number (RC09). See example 6.

Schedule-No ="<Schedule Number>"

Delete the schedule item referenced by the Internal Schedule Number (AD16). See example 7.

<GroupCode>

Specify the Group that will have an instance deleted. If this parameter is omitted, the command has no effect.

Example Scripts

1 Deleting an instance

Executing this command will delete the most recently-viewed instance of the group, GRPS.

[&Delete GRPS]

2 Deleting a particular instance

Using an &ForEach command to loop through the group GRPS, the user has the option of deleting each instance based on the information shown by the group screen, SCR.

[&ForEach GRPS]

   [&Show SCR]

      [&If ‘Delete Instance?’(Question) = "Yes"]

         [&Delete GRPS]

      [&EndIf]

[&EndFor]

3 Deleting from a system script

In this example, the &Select command produces a grid from which the user can select the instance to be deleted.

Note: When working from a system script, it is necessary to select a file using the &SelectFile command before the group screen can be accessed from the file.

[&SelectFile code = "FIL1",ref = "MyRef"]

[&With MyRef]

   [&Message "Select instance for deletion"]

   [&Select GRPS]

   [&Delete GRPS]

[&EndWith]

4 History-No

[&Delete HS, history-no=HS26]

Note: The history item needs to be found before the &Delete command is run. This can be done using the &Grid command to display a list of history entries for selection which returns the history number.

5 Ledger-No

[&Delete LG, ledger-no=LG22]

Note: The ledger item needs to be found before the &Delete command is run. This can be done using the &Grid command to display a list of ledger entries for selection which returns the ledger number.

6 Record-No

[&Delete RC, record-no=RC09]

[&Delete RC, record-no=RC09]

The recoverable cost item will have to have been found prior to the &Delete command being run. This could be done using the &Grid command to display a list of recoverable cost entries for selection which returns the record number.

Note: Disbursements can be deleted using the &Delete LG option.

7 Schedule-No

[&Delete AD, schedule-no=AD16]

Note: The schedule item needs to be found before the &Delete is run. This can be done using the &Grid command to display a list of schedule entries for selection which returns the schedule number.

Developers' Tips

The &Delete command only deletes an instance of a group where that instance has been selected since the file was opened. If a script containing the &Delete command is run immediately on opening the file, there is no currently selected instance, and, consequently, no deletion.
