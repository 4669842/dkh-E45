&OutputTo

Syntax    

[&OutputTo Name = <OutputName>,File = <Filename>]

Example Scripts

Description

This command is defined and used in the Data Extract Module to define the output file of the extract.

The data extract will currently output to Microsoft Access, Microsoft Excel, standard text documents, XML documents or other databases linked through ODBC/Microsoft Jet Engine.

&OutputTo and Visualfiles Browser

&OutputTo is supported in Visualfiles Browser but its use is restricted because scripts run on the server not the client PC; this means that output files are saved to the server not the client.

Parameters

access = <{Create|Append}>

Set whether the output will create a file, append to a file or update a file. The default value is Append.

file =

The name of the file to create to or to which data will be appended will vary depending on the connection type.

For an Access database: a fully-qualified UNC path to the Access Database.

\\servername\sharename\databasename.mdb

Example: \\sls-dhartley\warehouse\warehouse.mdb

For an ODBC connection: a collection of parameters to pass to DSN.

DSNname;UserName;Password;DBName;DBType;DBServerName

Example: Warehouse;ReportUser;v1sualf1les;Reports;SQL Server;sls-dhartley

An ODBC connection method requires an ODBC Data Source to be set up on the Visual Exchange client machine. This can be configured via Control Panel in most Windows environments.

length = <length>

Specifies the length of the character or decimal field.

log=

Used to create a log file and in a specified directory. See example 7.

log="path\directory_name"

Name = <Output>

A unique identifier for the output file.

name = <FileName>

The name of the file to create or to which data will be appended it will vary depending on the connection type. The filename specified may be a script element such as a temporary field.

no-validate

Specifies whether table schema validation should be carried out.

scale = <scale>

Specifies the number of decimal places for a decimal data-type.

stylesheet = <{No|Create|<StylesheetName>}>

Set whether a stylesheet will be used. The default value is No.

titles = <{fldno|flddesc|<UserDefinedDescription>}>

Field descriptions format. If this parameter is not specified, the default is (Field Number) Field Description.  Field Number or Field Description alone can be specified with fldno and flddesc respectively, or they can be used in combination with direct text input.

type = <{Access|Excel|Text|XML|ODBC|Notes}>

The type of file to be output to: Access, Excel, Text, XML, ODBC. The default value is Text.

The Notes data type enables you to output large amounts of data to a single field.
In Access, it creates a MEMO data type column; in SQL, it creates a TEXT data type column. This works with both ADO and DAO output methods.

warehouse = <{Yes|No}>

Specifies that the script is of type Data Warehouse.

Example Scripts

1 Define a text file as the output destination

This example shows how the first line of a data extract script can define a text file as the output destination using only the two compulsory parameters.

[&OutputTo Name="Example",File="C:\output.txt",access="Create"]

2 Combines the &OutPutTo command to create an Excel spreadsheet

This example shows how the &OutPut command is used in conjunction with the &OutputTo command to create an Excel spreadsheet containing all Private entities in the system, with name and entity code in labelled columns.

[&OutputTo Name="Example",File="C:\Example.xls",Type="excel",Access="Create"]

[&ForEach PRIV]

   [&Output Name="Example",Table="SheetName",Access="Create",EN02{Name="Entity Name"},EN56{Name="Entity Code"}]

[&EndFor]

3 Output XML file with stylesheet

This example demonstrates the use of the OutputTo command in outputting an XML file with the fundamental details of all individuals.  An accompanying stylesheet is specified to control how the xml file is displayed.

[&OutputTo Name="ENXML",File=TF01,Type="XML",Access="Create",stylesheet="Type='Text/xsl' Href='enstyle.xsl'"]

[&ForEach EN,EN06="INDI"]

   [&Output Name="ENXML",Table="Entity",Header="Yes",EN56{Name="EntityCode"},EN02{Name="Name"},EN03{Name="Description"},EN04{Name="ShortName"},EN07{Name="EntityType"},EN30{Name="Surname"},EN31{Name="Forename"},EN32{Name="Title"}]

[&EndFor]

[&Message "Open "+TF01]

4 Update the fields in the table

This example appends three generic fields and one file-specific field from all Support files to an Access file, database.mdb. The &OutputTo command is used to update the fields in the table Visualfiles Report Data.

[&Assign TF01=TIME]

[&OutputTo Name="Calls",File="C:\database.mdb",Type="access",Access="Append"]

[&ForEach MT,MT06="SUPP",Filter=all,NoIncludeAll]

   [&Output Name="Calls",Table="Visualfiles Report Data",Access="Update",MT05{Name="File_Code",Unique=yes,Primary=yes,Index=yes},FIL14{Name="Status",Type="char"},MT08{Name="Milestone",Type="char"},MT11{Name="Analyst",Type="char"}]

[&EndFor]

[&Message "Extract started at "+TF01+". Finished at "+TIME]

7 Testing for errors using Status system variable

[&Assign LF01='Enter Matter Ref']

 [&Assign LF02="a Desc"]

[&Assign LF03="01/02/2s001"]

[&Assign LF04="1"]

[&Assign LF05="1"]

[&Assign LF06="1"]

[&Assign LF07="7"]

[&Outputto name="Warehouse",warehouse=yes, log="c:\sql-log",file="Warehouse;sa;;DataWarehouse;Sql Server;localhost",type="ODBC",access="update",LOG="C:\sql-log\"]

[&Message  status(outputto)]

 [&Output name="Warehouse" ,warehouse=yes,log="c:\sql-log",table="test_tbl",access="update",LF01{name="MatRef",type="char",unique=yes,primary=yes,index=yes,length=50},LF02{name="MatDesc",type="char",length=20},LF03{name="mydate",type="date"},LF04{name="bigint",type="int"},LF05{name="MYDEC",type="char",length=20},LF06{name="MYDECa",type="char",length=20},LF06{name="MYDECb",type="char",length=20}]

[&Message status(output)]

This script outputs details to a test table and outputs any errors to the screen using &Message and to a log file in c:\SQL-LOG. Log files are named LNVF-DEBUG-99-99-9999.log

Some ways of generating an error include:

  • Entering invalid data in a field, for example, a date of 01/012a.2001

  • Trying to update a database that does not exist

  • Trying to append a row where a row with that key already exists in the database

The same error in the log file would be:

15:41:34 | LNVF Error Number: LNVF00003| Source: Microsoft Cursor Engine| Number: -2147217887| Description: Multiple-step operation generated errors. Check each status value.:| Further Details: Error writing data to field LF03 (mydate) Value = 01/02/2s001

Visualfiles error number

Error source

Error number

Error Description

Visualfiles details

Developers' Notes

As an alternative to hard-coding the parameters, you can also use TF fields for the following parameters:

  • Name
  • Access
  • File
  • Type
  • Titles
  • XML-SERVER-DOC
  • Stylesheet

Test if the &OutputTo Command Worked

The system variable Status, is used to return the status of the last execution of selected commands including the &OutputTo command.

Error trapping can be reported in two ways:

  • Use status(outputto) in scripts. See example 7 above.
  • Specify the parameter log=. See the log= parameter above.

Note: Messages are displayed on screen if an error is unexpected and therefore is not being trapped, or if the message requires user interaction.
