&Select

Syntax    

[&Select {<GroupCode>|<SystemCode>}]

Example Scripts

Description

Selects a component of an application type on which to perform an action.

&Select automatically selects either the first, next or last occurrence of a data item. This is essential in documents and scripts set up to be actioned without user input by the To-Do batch facility.

&Select can also be used with system codes:

  • &Select HS

  • &Select LG

  • &Select AA

    &Select AA supports first, next, last and rec-no=<rec-no>.

    Where <rec-no> is an expression that resolves to a valid AA11 value e.g:

    [&Select AA, rec-no=LF01]

  • &Select PY

    The &Select PY command is used to select Payment items.

  • &Select DB

    The &Select DB command is used to select Debt items.

  • &Select SL

    The &Select SL command is used to select SIM Log items.

  • &Select XL

Parameters

:first

Select the first iteration of a group or system code.

:last

Select the last iteration of a group or system code.

:lookup

Select the iteration of the group specified, pointed to by the currently selected lookup pointer field.

<GroupCode>:lookup

:new

Create and select a new iteration of a specified group.

<GroupCode>:new

:next

Select the next iteration of a group or system code.

Note: Forwards navigation through payment (&Select PY) and ledger (&Select LG) groups is not currently possible. For example [&Select PY:next] focuses on the ‘last’ entry in the group so no navigation between items is possible.

:prev

Select the previous iteration of a group or system code.

Note: Backwards navigation through payment (&Select PY) and ledger (&Select LG) groups is not currently possible. For example [&Select PY:prev] focuses on the ‘last’ entry in the group so no navigation between items is possible.

field1 = "<Field>"

The first field to be displayed.

field2 = "<Field>"

The second field to be displayed.

Group

Use this parameter to confirm that the item to be selected is a group code. This avoids possible conflicts with Entity Types on the system that share the same code as the file level Group.

The group parameter can be appended to a &Select command after the group code is entered or after the :new parameter. See examples 5 & 6.

label1 = "<TextString>"

The label for field1.

label2 = "<TextString>"

The label for field2.

type = <payment-type>

Used to specify the payment type i.e. payments direct to Client and Ledger items. This parameter overrides the default system behaviour in which new payments default to payment of the solicitor, resulting in the temporary creation of a client receipt ledger posting. See example 7. title = "<TextString>"

User-defined title of the screen displaying group details.

<GroupCode>

Specify the group screen to be selected.

<GroupCode>:lookup = <LookupPointerField>

Select the iteration of the group specified (pointed to by the currently selected lookup pointer field).

<GroupCode>:new

Create and select a new iteration of a specified group.

<SystemCode>

Specify the system screen to be selected.

Example Scripts

1 Linked Files

The &Select PY command is used to select Payment items.

[&Select P_PMOR:last]

Note: For linked files the group code must include the prefix letter.

2 Customising the Display of a Group

When using the &Select command to find a group the user can set the title of this dialog, determine which fields are displayed and set their labels. All the additional parameters are optional but quotation marks around values are mandatory.

[&Select ABCD, title = "Solicitor Details", field1 = "ABC01", field2 = "ABC02", label1 = "Name", label2 = "Address"]

3 New Parameter

In this example a new iteration in a group is selected by using the parameter :new. This code creates and selects a new iteration of the group EVIS.

[&Select EVIS:new]

4 Assigning Values

This code will select the occurrence/iteration of group EVIS, pointed to by the currently selected lookup pointer field SDM01.

[&Select EVIS:lookup = SDM01]

5 Group parameter

Used to confirm that the group code selected is a group code to avoid conflict; see group parameter.

[&Select PROP, Group]

6 Group and :new parameter

Used to confirm that the group code selected is a group code to avoid conflict; see group parameter.

[&Select PROP:new,Group]

7 type= parameter

[&Select PY, new, type="Client"]

Developers' Tips

If the first, next or last parameters are not included in the syntax, the &Select command selects the last referenced screen, either a previous &Select command or viewing in the GUI.
