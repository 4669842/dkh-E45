&Debug

Syntax    

[&Debug]

There are no parameters for this command.

Description

Processes a script step-by-step and is useful for monitoring the execution of a script.

&Debug causes the program to pause before each subsequent line of code and tell the developer which line is about to execute.  At each step, the developer can choose whether to continue with the debug, or whether to continue the normal execution of the script.

Example Scripts

1 Debugging a whole script

This example shows the &Debug command being used before a number of &Schedule commands to track which one is producing an error. In this case, the third command has the non-existent parameter, due-date, where the developer clearly intended to use the date-due parameter, and so the debugging will produce the error message Schedule command: unknown parameters ("Run Script three",Due-Date = today)".

[&Debug]

[&Schedule S-APP-SCRPT1,"Run script one"]

[&Schedule S-APP-SCRPT2,"Run script two",Group = GRPS]

[&Schedule S-APP-SCRPT3,"Run script three",Due-Date = today]

2 Selective debugging

In this example a &ForEach loop has been added to an existing script to debug the code required.

[&Assign TF1 = DBF1*DBF2]

[&Message "Result"+TF1,title = "Calculation"]

[&Debug]

[&ForEach GRPS]

   [&Message GRF1,title = "Edit Group Instance"]

   [&Show GRF]

[&EndFor]
