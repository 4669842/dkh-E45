Development > Commands Reference > Commands > &Webfield

&Webfield

Syntax    

[{&Webfield}]

[{code|action}]

[[value|item|target|property]...]

Example Scripts

Description

This command enables you to Set and Get information from a form using the web page object type.

To Set the check box or a multi-select object value, the items within the object must be declared selected, for example, true or false, see examples 4 & 5.

Note: This command is not supported in Visualfiles Browser.

Data Retrieval from Web Page

The &Webfield command can also be used to retrieve the text or HTML values from an embedded web page for a whole document, or on a form by form basis. See examples 6 to 8 below for more information on how to do this.

The source of a web page includes on-screen objects that have a name/id and a type. These objects include fill-in fields, buttons, images and hidden fields which  contain data that is not intended for display to the user. Use the &Webfield command to extract data from any object on a web page including hidden fields. Use the action = get parameter to retrieve the data required; see example 19.

Use the &Webfield command to navigate the web page, move back, forward, home or refresh the page. See example 9.

Extract URL

It is possible to extract the current URL from an embedded web page and to assign it to a TF or LF field; see example 21.

Parameters

action = "<actionname>"

Set or  Get information from a form.
Click indicates a mouse click to select.
Navigate loads a Webpage/form using the values forward, backward, home, refresh and/or URL.

code = "<codename>"

Temporary field assigned to the Webfield object on the database screen.

File = "<directory|filename>"

Enables the text of a web page to be output to a file.

The data returned from the web page can be returned directly to a text file on disk rather than a Visualfiles field, which later gets sent in 32k chunks to a text file. This removes much of the overhead and complex scripting currently required to generate the insertion file.

Form =

This parameter replaces the old method of specifying ITEM=form:frame:item. See example 14.

Note: The old method using ITEM=form:frame:item, is still supported but we recommend using this parameter or the Frame parameter.

Frame =

This parameter replaces the old method of specifying ITEM=form:frame:item. See example 14.

Note: The old method using ITEM=form:frame:item, is still supported but we recommend using this parameter or the Form parameter.

item = "<itemname>"

The ID or name if the field on the form the information is to be set or retrieved.

Note: If the item name contains colons use the no-item-path parameter to prevent them from being interpreted as specifying the form:frame path to the item see Form and Frame parameters.

no-item-path

By default the item= parameter interprets values containing colons as representing a form:frame:item path. Add the no-item-path parameter to override this behaviour for item names that contain colons.

property = "<TEXT|HTML>"

Defines the type data to be retrieved, either text or HTML.

Script =

Traps the onclick and onsubmit events.

Tabchild =

Enable data extract or set data on a frame that is part of a series of tabs.

Tabparent =

Enable data extract from a web page frame that is embedded within other frames.

Tag =

Use this parameter to access tags by ID, e.g. Tag="SPAN" , Item="LBLFIRSTNAME" this locates the first SPAN with this ID. If no Item is specified the first occurrence is retrieved.

 Use the Tag-Occurrence parameter where no ID exists.

Tag-occurrence =

Use this parameter to access tags by their occurrence, e.g. search for the third occurrence of the Table tag.

If no Tag-Occurrence is specified the first occurrence is retrieved.

target = "<targetname>"

Defines the field to retrieve a value to.

value = "<valuename>"

The value to set the specified item to.

To get a list of checked items the command is:

[&Webfield  code = "TSS01",action = "GET",item = "icecream,target = LF01]

Where TSS01 is the assigned field to the web page object and ‘Icecream’ is the name of the check box (defined in the HTML form).

6 Retrieve text on an embedded web page

This script demonstrates how to retrieve text or HTML for either the whole document or just and individual form.

[&Webfield code ="WEB01",action="GET",item="",property="TEXT",target=LF01]

The parameter item is left blank and the parameter property is added with the value of "TEXT". To get the HTML for the entire page you would do the same but specify the property as "HTML".

7 Retrieve text for a specific form on the web page

This script demonstrates how to retrieve text or HTML from a specified form.

[&Webfield code ="WEB01",action="GET",item="myformname",property="TEXT",target=LF01]

The parameter item is the name of the form and the parameter property is added with the value of "TEXT". To get the HTML for the entire page repeat this procedure but specify the property as "HTML".

Note: If the value of the text or the HTML retrieved is greater than 32000 characters this exceeds the limit for a character variable in Progress, so the amount of data returned is trimmed to 31000 characters for safety. If the amount of data available is greater than 31000 characters then the command status will be set to "MORE". This means the user can test status(webfield). If its value is equal to "MORE" this indicates that there is more data to retrieve.

Use the following code to retrieve the second and subsequent batches of information. Add a second entry to the property parameter to specify which set of data to retrieve:

[&Webfield code ="WEB01",action="GET",item="",property="TEXT:2",target=LF01] would retrieve bytes 31001 to 62000.

To loop through and retrieve all data available use the following code:

[&Webfield code ="WEB01",action="GET",item="",property="TEXT",target=LF01]

[&Assign LF02 = 1]

[&DoWhile status(webfield) = "MORE"]

[&Assign LF02 = LF02 + 1]

[&Webfield code ="WEB01",action="GET",item="",property="TEXT:" + LF02 ,target=LF01]

[&EndDoWhile]

This will continue to retrieve the data until status(webfield) no longer = "MORE".

8 Extract all HTML or text, and data from a specified form

The sample script below demonstrates all of the features mentioned in examples 6 and 7. It extracts all the HTML or text and the data from a form called "samplesecondform":

[& Assign TF01 = 'Get all text for text from a particular form'(choose "all":"form")]

[&If TF01 = "all"]

[&Assign TF02 = ""]

[&Else]

[&Assign TF02 = "samplesecondform"]

[&EndIf]

[&Assign TF04 = 'Select text or html'(choose "TEXT":"HTML")]

[&If TF04 = "TEXT"] [&Assign TF05 = "C:\Text-Document.doc"] [&EndIf]

[&If TF04 = "HTML"] [&Assign TF05 = "C:\Html-Document.html"] [&EndIf]

[&Webfield code ="WEB01",action="GET",item=TF02,property=TF04,target=LF01]

[&If status(Webfield) <> "MORE"]

[&Out TF05:LF01,append]

[&Else]

[&Assign LF02 = 1]

/** Check to see if we exceed 32,000 **/

[&DoWhile status(webfield) = "MORE"]

[& Message LF01]]

[&Out TF05:LF01,append]

[&Assign LF02 = LF02 + 1]

[&Assign LF03 = TF04 + ":" + LF02]

[&Webfield code ="WEB01",action="GET",item="",property=LF03 ,target=LF01]

[&EndDoWhile]

[&Out TF05:LF01,append]

[&EndIf]

9 Navigate the web page

When you use the action="navigate" parameter you can specify a URL in the value field to move to, or you can specify Back, Forward, Home, Refresh to move the current page as a normal Internet Explorer frame would move.

You can not retrieve the URL unless it is contained in the source for the web page, in which case you could extract it from there.

[&Webfield code="tst01", action="Navigate", value="Back"]

[&Webfield code="tst01", action="Navigate", value="Forward"]

[&Webfield code="tst01", action="Navigate", value="Home"]

[&Webfield code="tst01", action="Navigate", value="Refresh"]

[&Webfield code="tst01", action="Navigate", value=www.google.co.uk]

10 Searching for text using a form and a frame

[&Webfield code ="TSS01",action="GET",Form="results_docview_DocumentForm:Frame",property="TEXT",file=LF03]

Note: FORM:FRAME and the frame do not need to be the whole name just the start.

11 Searching for text where the frame lives on a series of tabs

Searching for text where the frame lives on a series of tabs and we need to ensure we get the document from the correct tab.

[&Webfield code ="TSS01" ,action="GET" ,item="results_docview_DocumentForm:fr_resultscontent" ,property="TEXT" ,file=LF03 ,tabparent=""framemaintop"" ,tabchild=""tabnumber""]

Note: Tabparent parameter, the parent frame that contains the tabs.
Tabchild parameter, the child frame of each tab.

12 View screen script navigating to a page

View screen script navigating to a page. Setting "on click" and "on submit" events and pre-setting a field.

[&Webfield code="TSS01" , action="NAVIGATE" , value="HTTP://WWW.LEXISNEXIS.COM/HOTTOPICS/KYCID/"]
[&Message "I'm in the view screen script again"]
[&Webfield code="TSS01" , action="onclick", item="searchform:search" , script="onclick" ]
[&Webfield code="TSS01" , action="onsubmit", item="searchform:search" , script="onsubmit" ]
[&Assign LF01 = status(webfield)]
[&Assign LF02 = 0]

[&DoWhile LF01 <> ""]
[&Assign LF02 = LF02 + 1]
[&Webfield code="TSS01" , action="onclick", item="searchform:search" , script="onclick" ]
[&If status(webfield) = "" ]
[&Webfield code="TSS01" , action="onsubmit", item="searchform:search" , script=""onsubmit" ]
[&EndIf]
[&Assign LF01 = status(webfield)]
[&If LF02 >= 20] [&Assign LF01 = ""] [&EndIf]
[&EndDoWhile]
[&Webfield code="TSS01" , action="SET" , property="" , item="searchform:search:companytext" , value = "VISUALFILES"]
