&Bill

Syntax    

[{&Bill}[bill-no = <bill-number-expression>]|[selected]]

Example Scripts

Description

Creates a bill for a file or an entity with the role of Client. Used with the billing and statements functions, &Bill can be included in a document or a script.

All postings for time, fees and disbursements in the Ledger or History are calculated and the user is prompted to update the balances. Where a user selects Yes, an entry is made in History and a bill number is assigned. On the next billing cycle, only unbilled items are included in the calculation and are then marked asBilled.

Use the ,selected parameter to help make billing more flexible through scripting, particularly for SOS linked sites where the Billing Module cannot be used. Normally when &Bill is run on a case it bills all outstanding unbilled time and ledger items. The ,selected parameter enables you to choose via scripting specific items to be billed. It works by testing values in billing status fields HS27 and LG20 respectively.

Note: This command is not supported in Visualfiles Browser.

Parameters

bill-no = <bill-number-expression>

Enables a bill number to be specified for a bill rather than the default behaviour of taking the next value from Visualfiles' internal bill number sequence.

The value of the parameter, bill-no, must resolve to an integer (whole number): textual or mixed values (for example, 5063C) will cause the command to be ignored. The expression can be a number, an input prompt, a database field or a database field related to the file.

,selected

Enables you to choose the specific items to be billed. With this parameter applied, only items where HS27 or LG20 are set to “R” (“Requested”) are picked up. Typically you would run a script (e.g. using a grid to enable individual selection of transactions) to assign “R” into the relevant field for the relevant transactions prior to running &Bill. Transactions can be deselected by setting the status back to “ “ or “X” for exclude, prior to running the &Bill command. Please note that once the bill has been run it is too late to apply this parameter.

Example Scripts

1 Produce a statement

This script will produce a statement of all unbilled items from History (HS) and Ledger (LG) including: Instruction Fee, Exchange Fee, Completion Fee, Land Registry Search Fee, and Miscellaneous Client Payment.

[&ForEach HS,HS18 = u][HS01] [HS07][EndFor]

[&ForEach LG,LG13 = u][HS01] [LG01][EndFor]

[&Bill]

2 Select items

This script gives the choice of selecting top bill ledger only, time only or all:

[&Assign TF01='Select what to bill'(Choose "Ledger only":"Time items only":"All items")]

[&If TF01 &Begins "L"]

   [&ForEach lg,LG13="U"&And LG15="Yes",noincludeall]  **For each unbilled ledger items flagged as posted in accounts

      [&Assign LG20="R"] **assign billing status to "R" for "Requested"

   [&EndFor]

   [&Bill selected]

[&Else]

   [&If TF01 &Begins "T"]

      [&ForEach hs,HS13="yes"&And HS18="U",noincludeall] ** For each unbilled history time item

         [&Assign HS27="R"] **assign billing status to "R" for "Requested"

      [&EndFor]

      [&Bill selected]

   [&Else]

      [&Bill]

   [&EndIf]

[&EndIf]

3 Display unbilled ledger and history time items

 Display unbilled ledger and history time items in grids enabling selection, then bill all the selected items.

[&Grid action = "delete"]

[&Grid action = "create"]

[&Grid action = "create-col",name = "Description",format = "X(40)",type = "char"]

[&Grid action = "create-col",name = "Date",format = "X(12)",type = "char"]

[&Grid action = "create-col",name = "Amount",format = "X(12)",type = "char"]

[&Grid action = "create-col",name = "Internal Code",format = "X(8)",type = "char",key,hidden]

[&Grid action = "create-col",name = "Selected",format = "X(8)",type = "char",selected,hidden] **This will contain YES if selected

 

[&ForEach lg,LG13="U"&And LG15="Yes",noincludeall]  **For each unbilled ledger items flagged as posted in accounts

  [&Grid action = "create-row"]

  [&Assign GR01 = LG05]

  [&Assign GR02 = LG04]

  [&Assign GR03 = LG01]

  [&Assign GR04 = LG26]

**Nothing is allocated to the selected flag field GR05

[&EndFor]

 

[&Grid action = "display",title = "Select Ledger items to bill",button1 = "Cancel",button2 = "Select",default = "button2",response = TF06,result = TF07,rows = 6,multi-select]

**At this point Visualfiles will assign GR05 = YES for each row that was selected using control clicking.

[&If TF06 = "Button2"]

[&ForEach GR]

[&If GR05 = "YES"] if the flag is set to yes

**This loop will now cycle through each item that was selected.

   [&Assign LG20="R"] **assign billing status to "R" for "Requested"

[&EndIf]

[&EndFor]

[&EndIf]
