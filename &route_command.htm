&Route

Syntax    

[&Route <RouteCode>]

Example Scripts

Description

&Route is used at the end of a script to move onto the next milestone.  Once the &Route command is encountered, the script associated with that route is executed.  If a Quit command is encountered within the route script, then the file is returned to the previous milestone.

If the route script is run without encountering an &Quit command, the script attached to the next milestone is run.

Parameter

<RouteCode>

Represents a route between the current milestone and a destination milestone. If the RouteCode does not start at the current milestone, the user will receive a message stating that the route specified cannot be followed.

Example Scripts

1 <RouteCode> parameter

This example simply moves a file along the route ENQUIRY-BOOKING, where ENQUIRY is the current milestone for the file, and the enquiry is to become a BOOKING.

[&Route ENQUIRY-BOOKING]

2 <RouteCode> parameter

Similar to the example above, this code shows that a previous milestone can be revisited by using a route that runs in the opposite direction.

[&Route BOOKING-ENQUIRY]

Developers' Tips

It may be beneficial to implement a coding standard that consistently names routes according to the two milestones they connect. This aids memory and understanding.
