Development > Commands Reference > Commands > &RM8

&RM8

Syntax    

[{&RM8} ObjectType="< >",Action="< >"]

Example Scripts

Description

This command is used to manipulate specific objects within RM8 using Visualfiles Business languageVBL.

All commands take an ObjectType parameter which defines the object in RM8 that will be manipulated, and an Action parameter which defines what will be done with that object.

[&RM8 ObjectType="CaseSearchCriteria",Action="Create"]

Allows the user to define the scope of the History document search within RM8. At least one of the parameters must be supplied.

If this has not been specified within the Case Open Script, then the History Screen on the file will not allow Content searching within RM8; see example 8.

Documents can be finalised in RM8 from the file History screen to stop documents being edited using the Finalise Document Script; see example 10.

Parameters

,ClassificationURI="<value>"

Defines the history document classification scope.

,FolderURI="<value>"

Defines the history document folder scope.

,ClientCode="<value>"

Defines the history document clientcode scope.

,MatterCode="<value>"

Defines the history document mattercode scope.

[&RM8 ObjectType="CaseSearchCriteria",Action="Reset"]

Allows the user to clear the current History document search scope. See example 9.

[&RM8 ObjectType="Classification",Action="Query"]

Allows you to search for one or more classification objects within RM8. The classification object ID is needed either to associate a classification with a record when storing a document, or to provide a parent ID when creating a new classification to maintain the structure correctly. See example 4.

The details on the matching classifications are returned within a grid to allow the developer to further select and manipulate the results.

Parameters

,Topmost

Returns only 'Top Level' classifications within RM8.

,Code=<code>

Searches for a classification with this code. Using wild cards is allowed to find all matches with a specific start to the code (e.g. QA* ).

,Uri=<URI>

Returns a specific classification from RM8 by the unique identifier, or an empty grid if the classification does not exist.

,ParentCode=<code>

Returns classifications from RM8 where the parent classification matches the specified code (i.e. they are *direct* children of the parent classification). This allows for the traversal of the classification tree to find the desired classification.

[&RM8 ObjectType="Classification",Action="Create"]

Allows you to create a classification within RM8. You can create top level classifications or child classifications. When a classification is created, the unique identifier (URI) for the classification is returned to Visualfiles and can be stored in a user defined field. See example 3.

Parameters

,Toplevel

Creates a top level classification. Creating a top level requires a unique value for LevelNo.

,LevelNo="<value>"

Supplies the unique level number for the creation of a top level classification object. This level number is used as the basis for the numbering scheme for any sub classifications. Required for top level creates.

,ClassificationName="<value>

Unique name for the new classification (mandatory).

,ClassificationParent="<value>"

URI of the parent classification to associate the newly created child classification (required for child creates).

,ReturnField=<value>

Scripting variable which is updated with the URI for the newly created classification.

ObjectType="Folder",Action="Query"

Allows you to query the available folders within RM8, to allow for the selection of a folder to store a document within.

The details of the matching folders are returned within a grid to allow the developer to manipulate the results if required.

Parameters

,Uri=<value>

Searches for a folder with a specific URI within RM8 and returns a single folder.

,Code="<value>"

Searches for a folder with a specific folder name. Wild cards are supported but the RM8 search finds the word within the Name so it can find folders with the folder name out of Position.

ObjectType="Folder",Action="Create"

Allows you to create a folder within RM8 and return the URI of this folder to be stored for future use. See example 1.

Parameters

,Name="<Value>"

Name for the new folder (mandatory).

,ReturnField=<Value>

Scripting variable which is updated with the URI for the new folder.

ObjectType="Record",Action="Select"

Allows for the selection of a record from RM8 prior to either querying or updating the record fields. A successful selection of a record is required before any Access or Update methods are required. See example 2.

Parameters

HistoryNumber="<Value>"

Select the record in RM8 using the Visualfiles HistoryNumber for this record (one of HistoryNumber or URI must be supplied; if both are supplied URI takes precedence).

Note: This is the .hst field History Number.

URI="<value>"

Select the record in RM8 Using the RM8 Unique Identifier.

ObjectType="Record",Action="Access" ,Field =" "

The Access parameter is further split by the Field parameter, which is used to determine which field on the record is to be accessed. See example 5.

Parameters

Classification

Allows access to Classification Fields. On the record the classification is not held directly, but is held as a reference, so all fields of a Classification are not accessible. At least one Field must be present. When a record has no classification applied, the fields are set to "No Record Classification".

NameField="<value>"

Scripting variable that is updated with the Name of the Classification of the record.

URIField="<value>"

Scripting variable that is updated with the URI of the Classification of the record.

Notes

Allows access the record Notes.

NotesField="<value>"

Scripting variable that is updated with the Notes from the record.

Title

Allows access to the record Title.

TitleField="<value>

TitleField="<value> Scripting variable that is updated with the Title of the record.

Author

Allows access to Author Fields. On the record the Author is not held directly, but is held as a reference, so all fields of a Location are not accessible. At least one Field needs to be present. When a record has no author applied, the fields are set to "No Author Specified".

NameField="<value>"

Scripting variable that is updated with the Formatted name of the Author of the record.

URIField="<value>"

Scripting variable that is updated with the URI of the Author of the record.

Folder

Allows access to Container fields. On the record the Container is not held directly, but is held as a reference, so all fields of a record are not accessible. At least one field needs to be present. When a record has no author applied, the fields are set to "No Folder Specified".

NameField="<value>"

Scripting variable that is updated with the Formatted name of the Container of the record.

URIField="<value>"

Scripting variable that is updated with the URI of the Container of the record.

Addressee

Allows access to Addressee Fields. On the record the Addressee is not held directly, but is held as a reference, so all fields of a record are not accessible. At least one field needs to be present. When a record has no author applied, the fields are set to "No Addressee Specified".

NameField="<value>"

Scripting variable that is updated with the Formatted name of the Addressee of the record.

URIField="<value>"

Scripting variable that is updated with the URI of the Addressee of the record.

AdditionalField

Allows access to Additional Fields on the record. Only String and DateTime values are supported.

FieldName="<value>"

Name of the Additional Field to access.

Note: This is the field name not the Title, the two can be different.

FieldValue="<value>"

Scripting variable that is updated with value of the selected AdditionalField on the record.

RecordType

Allows access to the URI of the RecordType for this record. When not set the field is updated with "No RecordType URI specified.", otherwise the field is updated with the URI of the Record Type for this record.

RecordTypeURI="<value>"

Scripting variable that is updated with the URI of the RecordType for the record.

RecordNumber

Allows access to the Number of this record. If a record is created, this field is blank. If a new version is created this field contains the record number for the *new* version. When overwriting a document, this field contains the number of the current record.

NumberField="<value>"

Scripting variable that is updated with the Record Number of the current record. See example 12.

ObjectType="Record",Action="Create"

This creates a record object as the current record to update. The record is not actually created in RM8, this is merely the in memory representation of the record before the record is created by uploading the document.

This allows the record creation to set profile fields, change the record type and setup metadata within the Document Import Script, before the record is created in RM8 by the upload of the document. See example 7.

ObjectType="Record",Action="Release"

This releases the current record object being accessed by scripting without submitting any changes through to RM8. This is useful for aborting any script changes to the record object.

ObjectType="Record",Action="Update"

The Update parameter is further split by the Field parameter, which is used to determine which field on the record to update.

Parameters

Classification

Allows you to set the Classification on a record.

ClassificationURI="<value>"

URI of the Classification to associate the record with.

Notes

Allows you to set the Notes of the record,

Notes="<value>"

New notes for the record object.

Title

Allows you to set the Title of the record.

Title="<value>"

New title for the record object.

Author

Allows you to set the Author of the record. If the Author does not exist within RM8, a new Location is created.

Author="<value>"

Name of the Author to change the record's author to.

Folder

Allows you to set the Container of the record. This should be a folder object.

FolderURI="<value>"

URI of the folder to associate the record to.

Addressee

Allows you to set the Addressee of the record. If the Addressee does not exist within RM8, a new Location is created.

Addressee="<value>"

Name of the Addressee to change the record's addressee to.

AdditionalField

Allows you to set the Additional Fields on a record. Only String fields are supported.

FieldName="<value>"

Name of the Additional field to Update.

FieldValue="<value>"

Value to set to the Additional Field.

RecordType

Allows you to set the RecordType. Expects the URI for the Record Type to be supplied. Can only be used after an Initial Create of the record object; once it is set, it cannot be changed.

RecordTypeURI="<value>"

URI of the Record Type to use for this record.

ObjectType="Record",Action="Save"

Perform the update of the record to RM8, committing any changes to Records Manager. This allows the scripter to control how many round trips to RM8 are made, and perform error trapping at opportune points in the logic.

Performing the Save command whether successful or not, clears the currently selected record; this forces the scripter to reload the record object from RM8 in order to reflect any changes made by the system automatically (e.g. Auto classification, security policies, etc.).

Parameters

DetailedError="<value>"

Scripting field to be updated with the detailed error messages from RM8. If an error occurs when saving the record to RM8, this field is updated with the detailed error message supplied by RM8.

UriField="<value>"

Scripting field that is updated with the URI of the created/updated record.

ObjectType="Record",Action="Finalise"

Used to finalise the currently selected record in RM8. This action also marks the history item in Visualfiles as a record. See example 11.

By default all other versions of the document are marked as 'Inactive' within RM8, making them available for disposal.

Parameters

Inactive-Versions="NO"

If this field is provided and set to "NO" then previous versions of the document are not marked as 'Inactive', which allows earlier versions to be retained. See example 10 & 11.

ObjectType="Record",Action="MarkActive"

Used to mark a record that is currently 'Inactive' in RM8 as active again. Once called it releases the record in memory, so the record would need to be reselected if further actions are required.

Has no effect on a record that is already active.

ObjectType="Record",Action="MarkInactive"

Used to mark a record 'Inactive' in RM8, and set the InactiveDate to Today. Once called it releases the record in memory, so the record would need to be reselected if further actions are required.

Has no effect on a record that is already inactive.

ObjectType="Record",Action="Reprofile"

Reprofiles the selected record, i.e. the Records Manager profile script is rerun on this document. Afterwards the record needs to be saved to commit the changes to RM8.

10 Finalise Document Script

Documents can be finalised in RM8 from the file History screen to stop documents being edited.

[&RM8 objecttype="Record",action="finalise"]

[&UserInput 'Enter the URI of the document to Finalise', result=LF01]
