&Conflict

Syntax    

[&Conflict {contains|begins|word},...,count = <FieldName>]

[&Conflict {contains|begins|word},...,list = <FieldName>]

Example Scripts

Description

Searches entity types to identify those containing data matching the search criteria.

Parameters

begins = "<string>"

Filter the search to include only entities that begin with a specific string of characters.

This parameter enables fast searches against entity types.

button<X> = "<ButtonText>"

Set the text of button1, displayed in the Finder. The &Conflict command can display a dialog box with up to five buttons: the buttons are named button1, button2, button3, button4 and button5. To have one of these buttons show on a screen, say button3, add button3 = "button 3 text", to your command.

contains = "<string>"

Filter the search to include only entities that contain a specific string of characters.

This parameter does not work in SQL.

Note: Using this parameter to search with can be slow. If this is the case, we recommend using the begins and word parameters for faster searches.

count = <FieldName>

Identify the number of matching entities and return this number to a field.

default = "<ButtonNumber>"

Assign a button as the default. The default button has the focus applied and its value will be assigned if a user simply presses Return on a screen. If the default parameter is not used, focus will apply to button1. Valid values are 1, 2, 3, 4, and 5, representing each of the five buttons.

fetch = <NumericValue>

Limit the number of values returned to a specified number.

list = <FieldName>

Identify the field to which the matching entity will be returned.

Where there is more than one matching entity, the list will be presented to the user to select entity codes to assign to the field.

nodisplay

Prevent the Finder being displayed if 1 or 0 conflicts are found.

response = <Field>

Specify the location to which the user response should be returned.  If the user double-clicks their selection or highlights a finder entry and presses Return, the response will equal GO.  If the user clicks the Close window button in the top right-hand corner of the dialog, or presses Escape, the response will equal CANCEL.  If any of the user-defined buttons are clicked, the response will equal BUTTONX.

role = "<Role1;Role2;... >"

Return only entities of a certain role. Multiple types may be specified in a list separated by semi-colons.

title = "<FinderText>"

Set the text of the title displayed in the Finder.

type = "<EType1;EType2;...>"

Return only entities of a certain type. Multiple types may be specified in a list separated by semi-colons.

Note: The &Conflict command only searches on the entity name, EN02.

word = "<string>"

Filter the search to include only entities that contain a specific word.

This parameter enables fast searches against entity types.

Using the Returned Value

[&Conflict contains = "smith",list = TF01]

The value returned from this search will be assigned to TF01. If there is no conflict, TF01 will contain the value 0 (zero). If there is a conflict, TF01 will contain the internal code of the selected entity.

Example Scripts

1 Creating a new case

[&Assign TF01 = "Visualfiles"]

[&Assign TF03 = 0]

[&Conflict Contains = TF01,Type = "orga", Role = "client;introducer",Fetch = 9,List = TF02,Count = TF03]

[&If TF03 = 0]

   ***Entity does not yet exist so it is OK to create it.***

[&Else]

   ***Select existing Visualfiles entry, currently contained in the TF02 code.***

[&EndIf]

   ***Create the new file.

2 Default focus

[&Assign TF01 = 'Please Enter Search String Containing Name of the Person']

[&Conflict contains = TF01,type = "INDI",count = TF02,list = TF03,default = "2",fetch = "all"]

Note:  Button2 will have the focus when the conflict dialog box is initially displayed.

3 Using a temporary field for a role list

A temporary field can be constructed to contain a list of roles such that this list can be used in the role parameter.  In this example, the conflict command would return individuals taking on the role of secretary, chair or administrator.

[&Assign TF01 = "SECRETARY;CHAIRMAN;ADMINISTRATOR"]

[&Conflict type = "INDI",count = TF03,role = TF01]

Developers' Tips

  • If no fetch limit is set, the entire database is searched. If the &Conflict command is being used to establish whether a conflict exists, set the value of fetch to 1.
  • If a fetch value has been set, it will determines the maximum value that can be assigned using the count parameter, even if other matches exist. If fetch is set to 1, then the value assigned by the count parameter can only be 0 or 1.
  • The internal code, EN56, is stored as a decimal value (number) but the output from the &Conflict command is of type text: when an EN56 code is returned from an &Conflict search, it is returned in type text. Consequently, when testing, it is necessary to test against a value of type text to avoid a <mismatched data types> error.  Alternatively, the text value can be converted to type number with an &Assign command: [&Assign TF01 = TF01:number].
  • If 1 or 0 conflicts are found when the nodisplay parameter is used, the conflicting entity is not displayed.
