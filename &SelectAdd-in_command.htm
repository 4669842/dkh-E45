&SelectAdd-in

Syntax    

[{&SelectAdd-in}]

Example Scripts

Description

Selects the Add-in referenced in the command.

Parameters

case-ref

If you are creating an Add-in for a case that is not the current owner.

code

Value of database field.

field

An indexed database field.

int-code

The internal code of the Add-In. MT30.

ref

Where an Add-in has been selected or created with reference.

type

Add-in code.

Example Scripts

1 Select an Add-in for a file

This example selects the money laundering add-in giving it a reference of myMONEYL.

[&SelectAdd-In type="MONEYLAUNDERINGAA",ref="myMONEYL"]

2 int-code parameter

This example selects the M2 add-in from a file where the int-code (MT30) matches, and gives it a reference of myM2.

[&SelectAdd-In type="MATTERMNGTAA",int-code=763,ref="myM2"]

3 Using the case-ref parameter

This example selects the M2 add-in for all of the files that are linked to the client, outputting the number of files that have the add-in attached:

[&Assign LF01=0] ** Files with M2 Add-in

[&Assign LF02=0] ** Total Files

[&ForEach CLIENT_FILE]

   [&SelectFile code=CLIENT_FILE.MT05,ref="myFILE"]

   [&SelectAdd-In type="MATTERMNGTAA",case-ref="myFILE",ref="myM2"]

   [&If status(SelectAdd-In)=""]

     [&Assign LF01=LF01+1]

   [&EndIf]

   [&Assign LF02=LF02+1]

[&EndFor]

[&Message LF01+"/"+LF02]

4 Using field and code parameters

This example selects the M2 add-in which has a field code of OPY22 that contains the value “Complete".

[&SelectAdd-In type="MATTERMNGTAA",field="OPY22",code="Complete"]
