&PutDocument

Syntax    

[&PutDocument Document=

<OriginalHistoryDocumentName>, Directory=<DirectoryPathOfHistoryDocument>, Filename=<NameOfNewDocument>, Action=<VersionDescription>, Master]

Example Scripts

Description

Creates a new History version of a specific History document and can be used in conjunction with the &GetDocument command.

&PutDocument is used to create additional versions of a History document even when that document is not flagged for multiple versions.

This command also enables you to automatically import and save images into the Imaging Inbox. See examples 1-4.

Note: This command is not supported in Visualfiles Browser.

Parameters

action = <VersionDescription>

Document version description. If not specified, history version action will be 'Created <DD/MM/YY HH:MM:SS>'

delete

Use this parameter to delete imported images. See example 12.

directory = <DirectoryPathOfHistoryDocument>

Location of History Document. If directory is not specified, the command will look in the current directory.

document = <History-ID_HS26>

The value of the internal code of the History ID, HS26.

filename = <NameOfNewDocument>

Name for new document.

filetype

List of file types when importing images from a named directory.

image

Save Item to the Imaging Inbox. To use this parameter you must have the Import Images Access Right selected.

import

Import items (documents and images of all formats) to the import queue Document Import.

You need to select the Desktop Document Import User Access Right to use this parameter. See example 13.

master

Make this version the master version. Overrides the existing master version.

user

Optional user Id for saving an image into a specific Imaging Inbox.

Example Scripts  

1 Load a single file

[&PutDocument filename="c:\temp\image.jpg", image]

2 Load all files from a named directory

[&PutDocument directory="c:\temp", image]

3 Load all tif and jpg files from a named directory

[&PutDocument directory="c:\temp", filetype="tif,jpg", image]

4 Use the “user=“ option to specify a user’s Imaging Inbox and save images

[&PutDocument directory="c:\temp", filetype="tif,jpg", user="vfile", image]

5 filetype parameter

filetype = <FileTypeList>

6 user parameter

user = <UserId>

7 Load a single image

[&PutDocument filename="c:\temp\image.jpg", image]

8 Load all files from a named directory

[&PutDocument directory="c:\temp", image]

9 Load all tif and jpg files from a named directory

[&PutDocument directory="c:\temp", filetype="tif,jpg", image]

10 Specify a certain user’s imaging inbox to save images

[&PutDocument directory="c:\temp", filetype="tif,jpg", user="vfile", image]

11 Create a new History version of a specific History document

[&PutDocument document={history-ID} ,  Filename="Myfile.doc"]

 

[&PutDocument document={history-ID} ,  Filename="Myfile.doc" , directory="c:\junk"]

 

[&PutDocument document={history-ID} ,  Filename="Myfile.doc" , directory="c:\junk",action="my new description"]

12 delete parameter

[&PutDocument directory=TF01, filetype="tif,jpg", user=TF02, image,delete]

13 import parameter

[&PutDocument filename="c:\tmp\mydoc.docx", import]

Developers' Tips

To use the &PutDocument command with the image parameter, you must have the Import Images Access Right selected.

To use the &PutDocument command without the image parameter you must have the Import Document Access Right selected.

This command is compatible with Document Management systems.
