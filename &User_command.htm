&User

Syntax    

[{&User type="amend"|"create" ,user-id="<user-id>" ,name="<name>" ,template= " template" ,password= "password"}]

[supervisor|fee-earner|mail-folder|system-relations|level|email|mail-profile|charge|cost|file-holder|tab-folder-display|no-progress-warn|auto-refresh|auto-refresh-minutes|web-user|timesheet|network-id|status]...]

Example Scripts

Description

Enables user records to be created via scripting.

Mandatory fields are type, user-id, name, template and password. All others parameters are optional.

The values of all the parameters can be set using local fields, for example, LF01.

All parameters used create the values specified in the syntax, with the exception of template, system-relations and level.

Parameters

account-file-holder="<name>"

File holders with an accounting package integration installed (e.g. SOS).

auto-refresh="<Y or N>"

Specify "Y" to enable auto refresh and control the refresh rate for the current file.

auto-refresh-minutes="<60>"

When auto-refresh is enabled, you can set a time period up to 60 minutes.

charge=""<00.00>"

Set the charge out rate per hour for the file holder's time. This monetary unit is set to 2 decimal places, for example, "100.00" per hour.

cost="<00.00>"

Cost rate per hour of the users time (the difference between the charge out rate and the cost rate is profit).

email="<email@address>"

Specify the users email address.

ext="<extension_number>"

Specify the users phone extension number.

ext-fee="<extension_number>"

Specify the fee earners phone extension number.

fee-earner="<fee_earner_name>"

If the user is a fee earner, enter their name.

file-holder="<Y or N>"

Specify ”Y” to make the user a file holder. Enter “N” to denote a regular system user.

job-title="<job_title_name>"

Specify the users job title.

level="<Level_name>"

Specify the required Access Level for the user. If a level is specified it must exist on the system e.g. "ADMIN".

mail-folder="<mail_folder_name>"

Specify the name of the mail folder for the user.

mail-profile="<mail_folder_name>"

Enter the name of the mail profile for the user.

name="<name>"

Enter user’s name.

network-id="<Windows_ID>"

Enter the user's network login ID details.

no-progress-warn="<Y or N>"

Specify ”Y” to suppress Progress warning messages, for more information go to http://knowledgebase.progress.com/articles/Article/P79795.

password="<password>"

Enter the user’s password.

Note: When a password is assigned using the password parameter it cannot be amended using &User type="amend".

ref="<user_reference>"

Specify the user reference which can be up to 12 characters long. The user reference is a user-defined free form string.

status="<LF01>"

This parameter sends any errors returned to LF01 for example, enabling developers to resolve or code around errors.

If the status parameter is not specified and the command errors, the script terminates and any code after the &User command is not executed.

supervisor="<escalation_fee_earner_name>"

Enter the escalation fee earner's name. This is the file holder who is allocated when the file is ready for escalation, who is usually more senior than the current file holder.

system-relations="<Y or N>"

Set up system-relations by specifying “Y” to activate; specify “N” to deactivate. If system-relations are not setup and active on the system, this parameter is ignored.

tab-folder-display="<Y or N>"

Note: This parameter is not supported in Visualfiles 3.0 and later.

Specify “Y” to replace the navigation bar buttons with a tab display, or specify “N” to revert to the button display.

template="<template_name>"

Specify the required user template. The template specified must exist on the system.

timesheet="<Y or N>"

Specify "Y" to switch on the user's Time Sheet which the the user can access from the Desktop.

Note: Take care when activating the Time Sheet option because selecting this option may mean that documents with time that would usually be recorded straight into the file history, no longer appear there. Instead these items are listed in the relevant time sheet ready for posting later. Extra development work may also be needed in order to post a separate history item for the document and the time elements. Please test this option in a development and test environments before making changes in live.

type="amend"

Update existing user records. See example 1.

type="create"

Create user records. See example 2.

update-levels

Use this parameter when changing user levels in the hierarchy, in order to ensure that links between users and files are retained when they are moved. The command goes through the file history and schedule and updates the levels and restores the links.

user-id="<UserID>"

Enter the user ID.

web-user="<Y or N>"

Specify "Y" if the user needs access to Visualfiles Online or Visualfiles Browser.

Example Scripts

1 User-defined in script

[&Assign LF01="User Test 55"]

[&User type="Create",user-id = "Test55",name = "LF01",level" = "ADMIN",email = "test55@test.com",template ="Default",mail-profile ="OUTLOOK",charge ="40.00",cost ="56.00",file-holder = "Y",tab-folder-display = "Y",no-progress-warn = "Y",auto-refresh = "Y",auto-refresh-minutes =  "60",web-user = "NO",timesheet = "Y",password = "TEST",system-relations = "Y",fee-earner = "Nick",mail-folder = "gggg"]
