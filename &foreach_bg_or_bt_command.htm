&ForEach BG, BI or BT

Syntax    

[&ForEach BG]

[&ForEach BI]

[&ForEach BT]

Example Scripts

Description

&ForEach BG

When using the &ForEach BG command, users can utilise a filter, a sort and a group-by parameter to reduce the number of records returned.

&ForEach BI

&ForEach BI provides individual detail on each ledger or history item on a bill.

&ForEach BT

&ForEach BT will make only one record available per Bill. This contains all Totals for the Bill: any subtotalling will be handled by BG codes.

Parameters

SORT = <BG01>

Sort all records to show in Source Type order (BG01).

FILTER = History

or

FILTER = Ledger

Allows the &ForEach command to be filtered to show only History or Ledger items.

GROUP-BY = <BG01>

This allows groups of BI fields to be subtotalled: returning a BG record where all BI records with the same Source Type (BG01) have been added together.

Example Scripts

1 Group together all BI records

This script would group together all BI records having the same Source Type and Fee-earner and subsequently, the BG fields relating to totals will produce a subtotal for all these records.

[&FOREACH BG,GROUP-BY = BG01+BG09]

2 Find and sort BI records

This script would find all BI records of type Ledger and group them by VAT rate. They will then be sorted and displayed in folder code order.

[&FOREACH BG SORT = BG02,FILTER = LEDGER,GROUP-BY = BG16]

3 Find and sort BI records

This script would find all BI records having the same Source Type and Vat rate. They will then be sorted and displayed in folder code order.

[&ForEach BG, SORT=BG01, GROUP-BY = BG01+BG16]

4 Find BI records

This script would find all BI records with the same Source Type code as the code assigned to the temporary field, TF01.

[&Assign TF01 = "1234"]

[&ForEach BI,BI01=TF01]
