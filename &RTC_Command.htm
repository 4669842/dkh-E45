&RTC Command

Syntax    

[{&RTC action="Start"|"Stop"|"Create"|"Delete"|"Post"|"On"|"Off"}]

[{&RTC action="Start"}[int-code=< >|, activity=< >|, create|, description=< >|, notes=< >|,prompt]...]

[{&RTC action="Stop"}[int-code=< >|, activity=< >|,prompt]...]

[{&RTC action="Create"}[int-code=< >|, activity=< >|, description=< >|, notes=< >|, start|,prompt]...]

[{&RTC action="Delete"}[int-code=< >|, activity=< >|, all|, active|,prompt]...]

[{&RTC action="Post"}[int-code=< >|, activity=< >|, description=< >|, notes=< >|, all|, active|,prompt]...]

[{&RTC action="On"|"Off"}]

Example Scripts

Description

The &RTC command enables timers within a case to be controlled through scripting. Timers are used to record time against a file and post it to History for charging to the client. Before real time can be captured a new timer must be created and attached to an existing file. There can only ever be one timer running at a time.

Within the &RTC command there are options that can be specified that enable developers to change the behaviour of the selection list that opens when choosing a timer. These include the title buttons, columns and user response.

Note: This command is not supported in Visualfiles Browser.

Pick Lists

When more than one timer matches the search criteria, a pick list is displayed from which users can select the required timers. The pick list only displays inactive timers.

When only one timer matches the search criteria, it is automatically selected. If you prefer to display the pick list when only one valid result is available, specify the ,prompt on your command.

Parameters

"Start"

Used to select an existing timer and start it. There are several options available which can be included with this parameter to fine tune the search and start the required timer.

When more than one timer matches the search criteria, a pick list is shown and the selected timer is started. When no matching timers are found, the command does not start any timer, unless the ,create option is used. This creates and starts a new timer. Any values specified in the int-code, activity, description and notes options will be copied into the new timer.

When using the create option, the description and notes options can be specified to pre-assign values to the new timer. See examples.

Note: Do not use the "Start" parameter in scripts run from a robot or other automated process unless you are certain that the pick list will not open.

You can be specific about the timer that starts by passing in various options.

"Start" parameter options include:

int-code=

Used to find and start a timer already assigned to a specified case. When there are no matching timers and the create option is defined, a new timer is created and assigned to the case.

When matching timers exist and the create option is included in the command, create has no effect.

The int-code corresponds to the case internal-code MT30.

activity=

Used to find and start a timer already assigned to the specified activity type. When there are no matching timers and the create parameter is specified, a new timer is created and assigned to the activity code.

create

Used to create and start a new timer if no matching timers are found. Any values specified on the int-code, activity, description and notes parameters are copied to the new timer. If an existing timer is found, this option has no effect.

description=

Used with the create option (above). This parameter takes effect when there are no matching timers to start. The value assigned to the description parameter is copied to the description field of the new timer.

notes=

Used with the create option (above). This parameter takes effect when there are no matching timers to start. The value assigned to the notes parameter is copied to the notes field of the new timer.

"Stop"

Used to select and stop an existing timer. You can be specific about selecting the timer that stops by including the int-code= and activity= options. Both these options can be specified to make the search more precise. If neither of these options are specified the active timer, irrespective of the case it is assigned to or its activity code, is stopped. See examples.

Note: Since there can only ever be one active timer running, this parameter never result in a pick list being displayed.

"Stop" parameter options include:

int-code=

Used to find and stop a timer already assigned to the specified case. When there are no matching timers for the case specified, the command fails.

activity=

Used to find and stop a timer already assigned to the specified activity type. When there are no matching timers for the activity code specified, the command fails.

"Create"

Used to create a new timer and optionally start it. You can define the timer that is created by passing in various options. User-defined text can be pre-assigned to the new timer using the description and notes parameters. See examples.

"Create" parameter options include:

int-code=

Used to assign the new timer directly to a case by specifying its internal number MT30. If this option is not specified the timer is created but it is not be assigned to a case.

activity=

Used to automatically assign the new timer to an activity code by specifying specify the code on the activity option. If option is not specified the timer is created without an activity code.

description=

Used to automatically assign a description to the new timer by specifying the text using this option. If this option is not specified the timer is created without a description

notes=

Used to automatically assign some notes by specifying the text. If this option is not specified the timer is created without notes.

start

Used to make a new timer start automatically; which will automatically stop the currently running timer. If this option is not specified the timer is created, but it does not start, and any other active timer remains active.

"Delete"

Used to delete a timer. You can make the search more specific by passing in the int-code and activity options to locate the timer to delete. If there are no matching timers the command fails. If there is only one matching timer it is deleted. If more than one timer matches the search criteria, a pick list is shown from which the timer to be deleted can be chosen. See examples.

"Delete" parameter options include:

int-code=

Used to find and delete a timer already assigned to a specific case. When there are no matching timers for the case specified, the command fails.

activity=

Used to find and delete a timer already assigned to the specified activity code. When there are no matching timers for the activity code specified, the command fails.

all

Used to delete all the timers that match the search criteria.

active

Used to delete the active timer that matches the search criteria.

"Post"

Used to post the time from an existing timer to History or to Time Sheets if it is switched on by selecting the Use Time Sheet option in User Properties. You can specify the timer to post by passing in the int-code and the activity options. If there are no matching timers, the command fails. If there is only one matching timer, its time is posted. If there is more than one timer matching the criteria, a pick list is displayed and the selected timer is posted.

For a timer entry to be posted it must have a valid file code and activity code. When there are multiple timers matching the search criteria, a pick list is shown. It only list items that have a valid case and activity codes.

In line with the manual process of posting timers, time values are rounded up to their standard unit as defined for the activity code. See examples.

"Post" parameter options include:

int-code=

Used to find and post a timer already assigned to the specified case. When there are no matching timers for the case specified the command fails.

activity=

Used to find and post a timer already assigned to the specified activity code. When there are no matching timers for the activity code specified the command fails.

description=

Used to assign a description to the resulting history item when posting a timer.

notes=

Used to assign extra notes to the resulting history item when posting a timer.

all

Used to post all the timers that match the search criteria.

active

Used to post the active timer that matches the search criteria.

"On"

Performs the Power On function in the Real Time Capture window.

"Off"

Performs the Power Off function in the Real Time Capture window.

,prompt

Use this command to display the pick-list dialog box when only one valid result is available. Normally when only one timer matches the search criteria, the single result is automatically picked and used as the timer for the action. By including ,prompt in the syntax, the pick-list dialog opens with the single result displayed.

TITLE=

Represents the title that opens in the window of the selection list. If you want to change the default title to something else use this parameter to specify the text.

BUTTON1=

Represents the standard Close button. If you want to change the text from Close to something else, use this parameter to specify the text.

BUTTON2=

Represents the Select Timer button. If you want to change the text from Select Timer to something else, use this parameter and specify the text.

BUTTON3=

Represents an additional button. If you want an extra button to been shown, then specify this parameter with the text you want to show inside that button.

BUTTON4=

Represents an additional button. If you want an extra button to show, then specify this parameter with the text you want inside that button.

BUTTON5=

Represents an additional button. If you want an extra button to show, then specify this parameter with the text you want inside that button.

BUTTON-ORDER=

Determines the order in which buttons 1 to 5 show. The default order for the two standard buttons is 2,1 - use a comma delimited list of numbers to specify the order, for example 5,4,3,2,1.

COLUMN1=

Represents the text that shows in column 1 of the selection list. If you want to replace the default text then specify this option along with your text.

COLUMN2=

Represents the text that shows in column 2 of the selection list. If you want to replace the default text then specify this option along with your text.

COLUMN3=

Represents the text that shows in column 3 of the selection list. If you want to replace the default text then specify this option along with your text.

RESPONSE=

Represents the user's response to the selection list. It contains CANCEL if the user pressed escape, the Close (X) button or button 1, SELECT if the user pressed F2 or button 2, BUTTON3 if the user pressed button 3, BUTTON4 if the user pressed button 4, BUTTON5 if the user pressed button 5.

Example Scripts

1 Start existing timer and create/start timer

[&RTC action="start"]

This command is used to start a timer.

If there is only one timer then it is started irrespective of the case it is assigned to.

If more than one timer is defined, a pick list is displayed for the user to select which timer to start.

This command fails if there are no timers. Add the create option to resolve this, see below:

[&RTC action="start", create]

This command attempts to start a timer, but if it does not exist, creates and starts a new one.

The timer created is not assigned to a case, or given an activity code, description or notes.

If one timer already exists then it is started irrespective of the case it is assigned to and no new timer is created.

If more than one timer is defined, a pick list is displayed for the user to select the timer to start. No new timer is started.

2 Start existing timer specified in MT30, create/start timer

[&RTC action="start", int-code=MT30]

This command is used to start a timer assigned to a case specified in MT30.

If there is only one timer assigned to the specified case in MT30 then it is started.

If more than one timer is assigned to the case specified in MT30, a pick list is displayed listing all timers assigned to the case for the user to select and start.

If there are no timers assigned to the case specified in MT30 the command fails. Add the create option to resolve this, see below:

[&RTC action="start", int-code=MT30, create]

This command attempts to start a timer assigned to a case specified in MT30, but if it does not exist, creates and starts a new one.

If there are no timers already assigned to the case specified in MT30 a new timer is created and assigned to it.

If there is only one timer assigned to the case specified in MT30 then it is started and a new timer is not created.

If there is more than one timer assigned to the case specified in MT30, a pick list is displayed of all inactive timers assigned to case MT30, for the user to select and start. No new timer is created.

3 Start/create timer, assign activity code and description

[&RTC action="start", int-code=MT30, activity="ABCD", create, description="New Timer"]

This command creates a new timer, assigns it to the case specified in MT30, gives it an activity code, for example, ABCD and a description, for example, New Timer then starts it.

If there is one timer assigned to the case specified in MT30 which shares the activity code that is not already active, then it is started. A new timer is not created.

If there is more than one inactive timer assigned to the case specified in MT30 which shares the activity code , a pick list is shown with all these timers displayed for the user to select and start. A new timer is not created.

4 Stop timer

[&RTC action="start", int-code=MT30, activity="ABCD", create, description="New Timer"]

This command creates a new timer, assigns it to the case specified in MT30, gives it an activity code, for example, ABCD and a description, for example, New Timer then starts it.

If there is one timer assigned to the case specified in MT30 which shares the activity code that is not already active, then it is started. A new timer is not created.

If there is more than one inactive timer assigned to the case specified in MT30 which shares the activity code , a pick list is shown with all these timers displayed for the user to select and start. A new timer is not created.

[&RTC action="stop"]

This command is used to stop the current active timer irrespective of the case or activity code it is assigned to.

If there is no active timer, the command fails.

5 Stop specified timer

[&RTC action="stop", int-code=MT30]

This command is used to stop an active timer running against the case specified in MT30.

If there is no active timer the command fails.

6 Stop timer running against specified activity type

[&RTC action="stop", activity="ABCD"]

This command is used to stop any active timer running against the specified activity type irrespective of the case.

If there is no active timer for the specified activity type the command fails.

7 Stop timer running against specified case & activity type

[&RTC action="stop", int-code=MT30, activity="ABCD"]

This command is used to stop any active timer running against the specified case with the specified activity code. If there is no active timer for the specified case and activity code the command fails.

8 Create a timer, create/start timer

[&RTC action="create"]

This command creates a new timer.

It is not assigned to a case and has no description, notes or activity code. The new timer is not started and any other active timer remains active.

[&RTC action="create", start]

This command creates and starts a timer.

It is not assigned to a case and has no description, notes or activity code. When the new timer starts the current active timer is stopped.

9 Create a timer assigned to a case with a description and activity code, create/start timer

[&RTC action="create", int-code=MT30, description="New Timer", activity="ABCD"]

This command creates a new timer assigned to the case defined in MT30. It has the description New Timer and is assigned the activity code ABCD.

The new timer is not started so if there is an active timer it remains running.

[&RTC action="create", int-code=MT30, description="New Timer", activity="ABCD", start]

This command creates and starts a timer.

The new timer is started and the active timer is automatically stopped.

10 Delete timer, delete active timer, delete all timers

[&RTC action="delete"]

This command is used to delete a timer.

If there is only one timer it is deleted. If there is more than one timer, a pick list is displayed for the user to select the timer to delete.

If there are no timers the command fails.

[&RTC action="delete", active]

This command is used to delete the active timer irrespective of its case code and activity.

If there are no active timers the command fails.

[&RTC action="delete", all]

This command is used to delete all timers.

11 Delete a timer specified in MT30, delete all specified timers

[&RTC action="delete", int-code=MT30]

This command is used to delete a case specified in MT30.

If there is only one timer for the case specified in MT30 it is deleted.

If there is more than one timer for the case specified in MT30, a pick list is displayed for the user to select which timer to delete.

If there are no timers for the case specified in MT30 the command will fail.

[&RTC action="delete", int-code=MT30, all]

This command deletes all timers assigned to the case defined in MT30.

12 Delete timers specified in MT30 with defined activity code, delete specified active timers

[&RTC action="delete", int-code=MT30, activity="ABCD"]

This command is used to delete the timer specified in MT30 with a defined activity code, for example, ABCD.

If there is only one timer for the case specified in MT30 with the defined activity it is deleted.

If there is more than one timer for the case, a pick list is displayed for the user to select which timer to delete.

If there are no matching timers for the case the command fails.

[&RTC action="delete", int-code=MT30, activity="ABCD", active]

This command is used to delete the active timer for the case specified in MT30 and assigned to a defined activity code.

If there are no matching timers for the case the command fails.

13 Post timer, post all timers, post active timer

[&RTC action="post"]

Use this command to post a timer to History.

If there is only one timer with a valid case and activity code it is posted to History.

If there is more than one timer a pick list is displayed for the user to select the timer to post.

If there are no timers the command fails.

[&RTC action="post", all]

This command is used to post all timers that have a valid file code and an activity code assigned to them to History.

[&RTC action="post", active]

This command is to post the active timer to History, if it has a valid case code and activity code assigned.

If there are no active timers then the command fails.

14 Post timer specified in MT30

[&RTC action="post", int-code=MT30]

This command is used to post timers assigned to the case specified in MT30. If there is only one timer assigned to the case it is posted provided it has a valid activity code. If there are multiple timers assigned to the case specified in MT30 a pick list of timers with valid activity codes is displayed from which the timer to post can be selected.

If there are no timers assigned to the case defined in MT30 the command will fail.

15 Post timer specified in MT30 with activity code defined, post active, post all

[&RTC action="post", int-code=MT30, activity="ABCD"]

This command is used to post timers assigned to the case defined in MT30 with the activity code specified, for example, ABCD. If there is only one timer assigned to the case with the activity code ABCD it is posted. If there are multiple timers assigned to the case specified in MT30 with activity code ABCD a pick list displayed from where the timer to post can be selected.

If there are no timers assigned to the case defined in MT30 with the activity code ABCD the command fails.

[&RTC action="post", int-code=MT30, activity="ABCD", active]

This command is to post an active timer for the case specified in MT30 with activity code assigned, for example, "ABCD". If there is no matching active timer, the command fails.

[&RTC action="post", int-code=MT30, activity="ABCD", all]

This command is used to post all timers for the case specified in MT30 with activity code ABCD. If there are no matching timers, the command fails.

Developers' Tips

The standard Real Time Capture open/close case behaviour can be removed by adding the system option 912 Real Time Capture in System Parameters. For more information see Real Time Capture.
