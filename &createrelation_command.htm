&CreateRelation

Syntax    

[&CreateRelation type = "<RelationCode>"]

[&CreateRelation relation-field = "<DBCode>"]

Example Scripts

Description

Creates new relations between specific entities.

Parameters

from-ref = "<EntityRef>"

Reference the source item. If not specified, this will be the default owner of the script.

noscript

Prevent the Create File script being run when a case is being created.

ref = "<TextString>"

Give the located target a reference.

When specified, the ref parameter provides a quick reference to the selected relation/target. This reference can be used throughout the remainder of the script.

relation-field = "<DBCode>"

Where a screen has a field of type relation, the relation displayed in the field may be created via the relation-field parameter.

Tip: In order to push an updated relation field out into the Visualfiles Integration Framework you need to update the screen field after the relation command by assigning the field to itself.
  [&CreateRelation relation-field="ABC01"
 [&assign ABC01 = ABC01]

to-ref = "<{EntityRef|RelationString}>"

Reference the target item. If not specified, the user will be prompted to select a target for the relation.

type = "<RelationCode>"

Type of relation to create or select.

Example Scripts

1 Ref parameter

This code will create a relation between Insurance and Claim. The target of this relation will then be given the reference CltIns. This entity may then be later referenced using CltIns.

[&CreateRelation type = "INSURANCE_CLAIM",ref = "CltIns"]

2 To-ref parameter

The entity of role CUSTOMER is linked to the entity of role CONTACT which is related to a support file.  The relation SUPPORT_CUSTOMER will be created from a support file to the entity of role CUSTOMER.

For example, a new support file has been created from the entity record John Smith. John is a contact employed by the entity record Visualfiles Ltd. This code will create a relation between the support file and Visualfiles Ltd.

[&CreateRelation type = "SUPPORT_CUSTOMER",to-ref = "SUPPORT_CONTACT.CONTACT_CUSTOMER"]

3 From-ref parameter

This code will create the relation between Insurance and Claim from the entity previously referenced as Claimant to the entity previously referenced as CltIns.

[&CreateRelation type = "INSURANCE_CLAIM",from-ref = "Claimant",to-ref = "CltIns"]

4 From-ref parameter

The following code creates the relation from the field ENG01 (Engineer Name) which is held as the reference FirstEng and it is made to reference a specific target which, in this case, is Contact.

[&CreateRelation relation-field = "ENG01",ref = "FirstEng",to-ref = "Contact"]

Developers' Tips

The &CreateRelation command does not work when using Visual Exchange. Instead, select a role and from within that role use the &Select MT command.
