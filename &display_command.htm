&Display

Syntax    

[&Display <DatabaseScreenCode>]

Example Scripts

Description

Provides an easy way to move between screens in the database menu, effectively changing the focus of the database window.  

Unlike the &Show command, &Display defaults to the screen specified in the database tab rather than in a separate dialog box.

The &Display command can be used in a button script and may be placed anywhere in that script but its effect will only be seen when the script has finished.

The &Display command can also be used in OPEN-CASE and OPEN-ENTITY scripts. If the user opens a case/entity using &Open and wants to ignore any &Display commands in the OPEN-CASE/ENTITY script then they can add the new override-display parameter to the &Open command.

&Display can be used in a field script and from a row-action script in a grid.

&Display can also be used with Quick Steps icons in a file.

Parameters

add-in = "<Add-in name>"

Specify the code of the Add-in which contains the screen to be displayed. See example 5.

group = "<GroupCode>"

Display a database screen from the specified group.

Nodelete

Enables you to prevent users from deleting a group screen. This parameter is only used with group screens.

When a group screen is displayed using [&Display ,nodelete], the group screen is displayed but the Delete button is disabled. See example 7.

owner

Used to reference a file level data from an Add-in. See example 6.

role = "<RoleCode>"

To display a screen from a role, the role parameter must be specified.

screen=<expression>

Enables you to specify the screen to show. See example 8.

<Database>

Display the Database screen.

<DatabaseScreenCode>

Specify the screen code for the screen to be displayed.

<Document>

Display the Action screen.

<Entities>

Display the Entities screen.

<FileCoverCode>

Display the File Cover screen.

<History>

Display the History screen.

<Ledger>

Display the Ledger screen.

<Milestone>

Display the Milestone screen.

<Schedule>

Display the Schedule screen.

Example Scripts

1 Displaying a screen

This script will display the screen, PDI, in the database window. PDI is located on a file and is not a group screen.

[&Display PDI]

2 Displaying a group screen

This script will display the screen, CIF, in the database window. CIF is located on a file but is a group screen within the group with the code INFO.

[&Display CIF, group="INFO"]

3 Displaying a Screen on Role

This script will display the screen, CUS, in the database window. In this example CUS is located on a role with a role code of CUSTOMER.

Note: This script will only work if run from an Entity with the role of Customer.

[&Display CUS, role = "CUSTOMER"]

4 Displaying a Group Screen on a Role

This script illustrates how examples 2 & 3 can be combined to display a group screen from a role in the database window. In this example GES is a group screen within a group code of GTRE which is located on a role called GUEST.

[&Display GES,role = "GUEST",group = "GTRE"]

5 Add-in parameter, call Add-in from file

When the screen EST, is on an Add-in called from a file.

[&Display EST, add-in="SALEAA"]

6 Owner parameter, call screen from Add-in

When the screen LET, is on the file and the screen is called from the Add-in.

[&Display LET, owner]

7 Nodelete parameter

[&Display ABC, group="ABCD",nodelete]

8 screen parameter

[&Assign LF01="ABC"]

[&Display screen=LF01]

is equivalent to:

[&Display ABC]

Developers' tips

Only one &Display command can be used in a script: where there are multiple &Display commands, only the last will be actioned.
