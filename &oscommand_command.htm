&OSCommand

Syntax    

[&OSCommand "<DOSCommand String>"]

Example Scripts

Description

Performs DOS commands within Visualfiles.

DOS commands can be assigned to fields to be run within scripts.

The &OSCommand can be used to run a batch file and be used with fields (see example 3).

&OSCommand and Visualfiles Browser

&OSCommand is supported in Visualfiles Browser but its use is restricted because scripts run on the server not the client PC, which means they behave differently. Using the &OSCommand command in Visualfiles GUI, it is possible to launch applications and do other things that interact with the user, since scripts are executed on the client PC; however, this use is not possible in Visualfiles Browser.

Parameter

interactive

Use this parameter to assist with debugging.

When this parameter is included, a command window is displayed when the script is run, enabling any errors, messages, etc., that are produced to be viewed.

start <DOSCommandString>

Used to open an application from within the OSCommand. See example 2.

Example Scripts

1 Change directory, copy, and delete

[&OSCommand "cd c:\"]

[&OSCommand "XCOPYc:\SolDataTransfer\FileOutput c:\SolDataTransfer\OutputBackup"]

[&OSCommand "DELc:\SolDataTransfer\FileOutput\*."]

2 Opening an application from the &OSCommand

[&OSCommand "Start Iexplore.exe http://www.google.com/"]

3 OSCommand and &Assign

[&Assign TF02="Start excel.exe c:\example3.xls"]

[&OSCommand TF02]

4 Run a batch file

[&OSCommand "c:\temp\myscript.bat"]

Runs the myscript.bat batch file in the c:\temp folder.

Developers' Tips

The &OSCommand will fail and an error message is returned if the command line is too long.

On MS-Windows the actual limit is 80+/- characters. The limit with Progress 9.1C on Windows 98 is 122 characters. The limit with Progress 9.1D or later, OpenEdge 10.0A or later, on Windows 32 bit is 200 - 230 characters.

This issue can be resolved by creating a batch file using the &Out command and call it using &OSCommand. Take care over the naming of the batch file to avoid other processes trying to write to the file at the same time.

The other alternative is to upgrade to OpenEdge 10.1A or later where the limit has been increased to 2048 characters.
