&Access-Mode

Syntax    

[{&Access-Mode}[get|set|display-override|result|mt-int-code|user-id|value]...]

Example Scripts

Description

Has several functions related to Access Plan Maintenance including:

  • returning the security for the specified file and user
  • temporarily over-riding security
  • displaying the file over-ride tree

Note: This command is not supported in Visualfiles Browser.

Parameters

display-override

Enables the script to display the current 'override' settings for a File, and change them. These override settings are security settings for a case which override the system defaults, and allow the user to set which users have read-write, read-only or no access to the file.

get

Retrieve details of the security setting for a user and case.

Used to get the current security settings for a file, and determine whether a specified user has access to it — what access level does user X have to case Y?).

mt-int-code = FileInternalCode

The int-code of the file being checked with get (not used for set).

result = <FieldToStoreResult>

Where the return value of get is stored, for example, RESULT=TF01 (not used for set).

set

Ignore the system settings, and set the security access for the case currently being opened to the value set in the value parameter.

Only intended for use in an Open-Case script, and sets the security setting for the case for that session only, regardless of the current user.

user-id = "<UserID>"

The user being checked with get (not used for set).

value = {<RW|RO>}

Used with set to specify the access level to change to - see Example 2 (not used for get).

Example Scripts

1 Return the access mode

Returns the access mode (Read Write RW/Read Only

[&Access-Mode get,result = TF01,mt-int-code = TF02,User-id = TF03]

2 Temporarily override security

Allows security to be temporarily overridden. TF01 should be RW or

[&Access-Mode set,value = TF01]

3 Display file override tree view

Displays the file override tree view for the file specified. If mt-int-code is omitted, the current file is used. This functionality is intended to provide scripted prompts to a user (who does not have access rights to modify user access settings) to modify the access parameters of their own file.

[&Access-Mode display-override, mt-int-code = TF01]
