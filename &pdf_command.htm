&PDF

Syntax    

[{&PDF Type=[Convert|Merge]}]

Example Scripts

Description

Enables the conversion of documents into a PDF document, and merging of documents into one .PDF file via scripting.

Note: Active PDF must be installed on the same machine as the AppServer before you can use the &PDF command.

Note: The PDFC PDF Conversion system module must be installed before you can use the &PDF command.
This system module requires a new licence key. Please contact your LexisNexis Account Manager for further details.

Note: This command is not supported in Visualfiles Browser.

Error Trapping

The Status system variable is available on this command, which enables error messages to be captured and assigned to LF fields. Setting the Status option to a temporary field passes timeout errors to the field that may occur during ActivePDF conversion, and the script continues to process. If no status option is supplied the functionality works as before and displays a message terminating the script. See example 7.

Parameters

Type = "Convert"

Specifies that documents are being converted to .PDF.

"Convert" parameter options include:

Debug

This option displays the status of the conversion.

Filelist = "<list of file names>"

The name of the files to be converted.

History

If a case is selected, applying this option adds the .PDF file to the case history.

ID=<field>

Passes the History ID of the newly created history item. If more than one file has been specified then it passes a comma separated list containing all the history IDs.

Image

Allows the ‘Height’ and ‘Width’ of the image to be specified (in pixels), giving more control over the conversion of images.

Location

The location where the converted files are placed.

If you want to create the .PDF file in the case

Do not use this parameter in the same command string as the History parameter. The Location parameter will not work if they are used together.

Status="<TF01>"

When the ActivePDF conversion times out, an ActivePDF timeout message is returned reporting the timeout.

The Status parameter is used to remove this core error message and assign it to an LF field. The Status option passes the timeout error to the assigned temporary field, for example, error or blank (if completed successfully) and the field and the script continue to process. If the status option is not included an ActivePDF timeout message is displayed and  the script is terminated. See example 7.

Title

This option enables a History Title to be specified onto the selected case history, providing that 'history' has also been specified.

Type = "Merge"

Specifies that documents are to be merged into one .PDF

"Merge" parameter options include:

Bookmark="<list of document references>"

The bookmark parameter allows you to reference each merged document as a bookmark in the PDF.

Each document is given their own bookmark within the PDF. These can be structured to allow some documents to be nested within others.

When you merge documents into a single PDF, you have to specify each file using the FileList parameter:

FileList="\\bbweb\pdf-temp\UserRequirements.doc,\\bbweb\pdf-temp\Specification.doc"

In the Bookmark parameter, the first document is given the reference of Doc1, the second Doc2 and so on.

So in the FileList example above, UserRequirements.doc is referenced as Doc1, andSpecification.doc would be Doc2.

You can also force additional structure by adding separators between the document references:

  • ^ =change the level of the document. You need to enter the name of the level after the separator, then add a forward slash (/) to add documents under that level.

  • / Documents within that level

  • , Return to level one.

Structure examples

Example 1

Bookmark="Doc1,Doc2,Doc3"

Structures the all documents in the bookmarks to a single level:

Doc3

Debug

This option displays the status of the Merge.

Filelist = "<list of file names>"

The list of the files to be appended to one .PDF (comma separated).

The order of the files in the merged document will be the same as the list. Document types other than .PDF can be specified.

The command will convert the document before adding to the .PDF

Filename="<filename>"

The filename of the newly generated file.  If not specified, the Filename will be first filename in the list with an Mappended on the end.

Filesize=<temporaryfield>

Returns the file size in bytes of the converted PDF documents. If more than one document is specified then all file sizes are returned separated by a comma.

History

If a case is selected, applying this option will add the created Document to case History.

Location

The location of the appended file.  If the Location is not specified, the merged file will be placed in the user's temporary directory on the server.

NoEdit

Prevents the PDF from being edited.

NoPrint

Prevents the PDF from being printed.

OwnerPassword=password

Allows you to add a password to the PDF file. This will prevent the PDF from being edited unless the password is entered.

The password must not be longer than 32 characters, and should not contain control characters.

Title

Enables a History Title to be specified onto the selected case history, providing that ’history’ has been specified.

UserPassword=password

Allows you to add a password to the PDF file. This will prevent users from opening the PDF unless they enter the password.

The password must not be longer than 32 characters, and should not contain A special, non-printing character that begins, modifies, or ends a function, event, operation or control operation. The ASCII character set defines 32 control characters. control characters.

Example Scripts

1 Convert documents using location modifier

[&PDF Type=CONVERT, FILELIST =\\dev-jy\C\hip2.doc,LOCATION =\\

2 Convert command for images

[&PDF Type="CONVERT", FILELIST ="\\dev-jy\C\hip2.jpg",LOCATION=

"\\dev-jy\C\Activepdfout", HEIGHT=500, WIDTH=550, image, history, debug]

On the convert option of the PDF Command, there is also an option of ‘Image’.  Specifying ‘Image’ will allow the ‘Height’ and ‘Width’ of the image to be specified (in pixels). This gives more control over the conversion of images.

The convert command will convert images without the image option, but the resulting PDF my not be what is required.

3 Convert and merge documents

[&PDF Type="Convert",filelist="\\vfqa-sbell\vishist\mergeme1.doc",location ="\\vfqa-sbell\pdf\",filename = "Converted.pdf"]

4 Convert document and add to the History

[&PDF Type="CONVERT", FILELIST ="\\dev-jy\C\hip2.doc",LOCATION =\\dev-jy\C\Activepdfout,ID=TF01,history,debug]

5 Security

[&PDF Type="CONVERT", FILELIST ="\\dev-jy\C\hip2.doc",LOCATION =\\dev-jy\C\Activepdfout,ID=TF01,history,debug]

6 Bookmarks

[&PDF Type="Merge",FILELIST="\\bbweb\pdf-temp\UserRequirements.doc,\\bbweb\pdf-temp\Specification.doc",BOOKMARK="Doc1,Doc2",LOCATION="\\bbweb\pdf-temp\",HISTORY,FILENAME="NewProductInformation.pdf"]

7 Status Parameter

[&Assign TF01 =""]

[&PDF type="Convert", filelist="C:\test.doc",status="TF01"]

[&Message TF01]

Developers' Tips

As the conversion/merge is done via the AppServer – the filenames must have the full network path and the server must have access to this location.
