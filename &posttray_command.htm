&PostTray

Syntax    

[&PostTray <PostTray>

Example Scripts

Description

Enables name and priority to be assigned from within a document.  &PostTray should only be used within documents: not in scripts.

Parameters

confirm

Presents the user with a dialog box to confirm the PostTray and Priority. The confirm parameter is ignored where the document has been produced by a robot, as no user interaction is allowed for the robot.

posttray = <PostTrayName>

Name of the PostTray to assign the document to.

priority = <PriorityLevel>

The priority attached to the document: 1, 2, 3.

Example Script

1 Displays a document review

When this document is actioned, a dialog box will display showing the Document Review named REVW with a priority level of 2. The user can then choose whether to accept or override these values. All parameters for this command are optional.

If the confirm parameter is not included, the dialog box will not be displayed and the document will be sent to the Document Review, REVW, with a priority of 2. If the posttray or priority parameters are not included in the command, it will default to the relevant values recorded in the Action screen. If the document is actioned from a robot user, the confirm parameter will be ignored and any values in the command will be assigned automatically.

[&POSTTRAY posttray = REVW,priority = 2,confirm]
