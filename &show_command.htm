&Show

Syntax    

[{&Show <ScreenCode}]

Example Scripts

Description

Displays a database screen to a user during the production of a document or form or when running a script. This enables the user to verify data before it is entered into a document. In a script, &Show can present a screen for data entry.

&Show may be used with the following system screens: Ledger (LG); Payment (PY); Debt Invoices (DB); History (HS); Client Debt Params (FI); Fee Scale and Interest (DI), Schedule (AD), Recoverable Costs (RC) and Payment and Arrangement Details (PA).

&Show can be used to display the main Visualfiles To-Do list screen from a system script attached to a button.

The &Show command can also be used in conjunction with the AL and AC codes to display the SOS Practice Manager Ledger screen. SOS Practice Manager screens that can be used with &Show are:

AC  Set Matter Charge Band

AL  View Matter Ledger Screen

PM  View Matter Details

VF  Generate Unique File Number

CH  View CDS11 History

Note: The codes used to display SOS Practice Manager screens are documented in the screen-variant section below.

Billing Module codes BI, BG & BT can also be used with &Show.

The &Show command cannot show system screens such as MT or MX as they are set for textual representation not graphical representation.

Visualfiles recommend avoiding the use of the &Show command and using the &Display Command instead, unless the no-wait or the next-action parameters are incorporated. This is because while the &Show command is running, a broker is tied up for the length of time that the screen is shown, however this is resolved with the inclusion of the no-wait and the next-action parameters. However, there are situations when it is not possible to use &Display, for example, the user does not have the case open, including when a process is run from the To-Do List.

The cancel and ok parameters can be used in button definitions to ensure that one of them will be triggered regardless of how the screen is exited. When this occurs a script associated with the button can be run.

Document Import

The &Show command can be used to display the Document Import screen from a script using the following syntax:

[&Show Docimport]

Parameters

:Database Field

<:FieldNo.1,FieldNo2,>

Specify the fields to be highlighted when the screen is displayed. For linked case types, the prefix letter must be included.

button

<UDScreen>,button="<buttonlabel>";"<scriptname>"

The button parameter enables developers to add buttons to a user-defined screen. Define the button label and run the script associated with the button when selected by the user.

The button label can be specified and the script associated with it runs when the button is selected. See examples 7 & 13.

Note: When you assign a script to a button and the button is selected, processing stops in the current &Show script and the button script is run; this is the same behaviour as the next-action parameter. For more information please refer to Stateless Scripting Options.

cancel

When this parameter is used as part of a dialog button definition, the button is triggered when Alt + F4 or the X (Close) button are used to exit the dialog. This ensures that the button is triggered regardless of how the dialog is exited. See example 8.

highlight-fields=<expression>

Enables you to specify the list of screen fields to highlight. Comma delimited list of fields to highlight, user-defined screens only. See example 12.

next-action={text_expression}

Used to end the script run at this point. Any code after the &Show command is ignored and the script specified by the text expression in the next-action parameter is run. See example 6.

Text expressions are used to name the required script and can be constructed from any combination of the following:

  • "constant text", such as a script name
  • 'User input'
  • Database fields
  • LF, TF, PI fields. The temporary field specified will contain the value entered by the user.

Note: For more information on stateless scripting options go to no-wait and next-action Parameters, Visualfiles Browser Environment.

next-action-inputs={expression};{expression}...

List of semi-colon delimited items to be passed as PI fields into the next-action script.

no-cancel

Used to hide the Cancel system button and prevent the user cancelling out of a dialog.

Note: When the no-cancel parameter is used it is possible for end users to exit incomplete mandatory dialogs and screens using the Esc button, if the hide screen scripts that are used include the &If command instead of using a &DoWhile loop after a screen is displayed.
No-cancel and &DoWhile (recommended)End users are unable to exit a screen using the escape button using &DoWhile.
[&Show BAA,no-cancel]
[&DoWhile BAA04=""]
   [&Message "All highlighted fields MUST be completed",title="Attention"]
   [&Show BAA,no-cancel]
[&EndDoWhile]
 
No-cancel and &If (avoid)End users can exit a screen using the escape button using &If.
[&Show BAA,no-cancel] **Hide screen script on BAA screen
[&If BAA04=""]
   [&Message "All highlighted fields MUST be completed",title="Attention"]
   [&Show BAA,no-cancel]
[&EndIf]
 
We recommend that using &DoWhile with [&Show <CODE>, no-cancel] and avoiding the use of &If with this parameter.

Nodelete

Enables you to prevent users from deleting a group screen. This parameter is only used with group screens.

When a group screen is displayed using [&Show ,nodelete], the group screen is displayed but the Delete button is disabled. See example 11.

no-help

Used to hide the Help system button from the dialog.

no-navigate

View the current iteration of a screen: no previous iterations are displayed nor can iterations be added or deleted. Used only with Group screens, this parameter determines whether the Navigation, Add and Delete buttons are displayed.

no-wait

Ends the script run at this point.

When a script that includes the no-wait parameter is run and there is no user response to a screen prompt, the parameter is executed which ends the script, ignoring all subsequent code. This results in agents that would otherwise be tied to the unused application, being accessible to other system users. See example 5.

Note: The no-wait parameter does not work for system screens in Visualfiles Browser, e.g. Show HS, Show LG. Currently, the stateless Visualfiles Browser functionality only works with user definable screens, e.g. [&Show SCR,no-wait].

OK

When this parameter is used as part of a dialog button definition the button is triggered when the Return/Enter key is used to exit the dialog. This ensures that the button is triggered regardless of how the dialog is exited. See example 9.

screen=<expression>

Enables you to specify the screen to show. See example 12.

screen-variant="<code>"

Enables you to display SOS Practice Manager screens. See example 15 for information on the SOS PM screens.

show-list

If screen appears as a list in the database, show the screen list. Without this option the screen currently selected from the list will be shown. See example 10.

size-to-fit

Shrinks the dialog size to fit its contents. See example 14.

This parameter is used in conjunction with Temporary Field screens.

Title="<screen title>"

Set the text of the title displayed in the screen.

Note: This parameter cannot be used with hard coded screens such as HS and LG.

<ScreenCode>

Specify the screen to be displayed.

Example Scripts

1 Using field numbers

To display the sale property screen in a Conveyancing case, the codesPRO07andPRO08represent purchase price and deposit respectively.

This code displays the screen with the purchase price and deposit highlighted:

[&Show S_PRO:7,8]

2 &Show command with Schedule screen

The Schedule screen may be called from a script using the &Show command.

[&Show AD]

3 Using the no-cancel parameter

Display the database screen COU and remove the Cancel button from the screen.

[&Show COU,no-cancel]

4 Remove Cancel button and highlight fields

Display the database screen COU, highlight the first, second and fourth fields and remove the Cancel button from the screen.

[&Show COU:1,2,4,no-cancel]

5 no-wait parameter

The no-wait parameter ends the script run at this point. So in the script below [&Assign TF01] will not be executed:

[&Show NAS,no-wait]

[&Assign TF01]

Note: NAS is a user-defined screen.

6 next-action parameter

The next-action parameter ends the script run at this point. Then runs the script AFTERMESSAGE when the user clicks OK in the message dialog. So in the script below [&Assign TF01] will not be executed:

[&Show NAS,next-action="AfterMessage"]

[&Assign TF01]

Note: No variables are available when the script is run, TF, LF, etc. NAS is a user-defined screen.

7 button parameter applying labels and attaching scripts

A script can be defined for the Cancel button, which runs when the screen is closed using the Cancel button or the X Close button.

This command creates a screen that includes a Cancel button but no Help button. NAS is a user-defined screen. NEXTACTION and AFTERMESSAGE are scripts.

[&Show NAS,button="Press me";"NEXTACTION",button="Another button";"AFTERMESSAGE";CANCEL,no-help]

Note: No variables are available when the script is run, TF, LF etc. NAS is a user-defined screen.

Note: When you assign a script to a button and the button is selected, processing stops in the current &Show script and the button script is run; this is the same behaviour as the next-action parameter.

8 cancel parameter

When this parameter is used as part of a dialog button definition, the button is triggered when Alt + F4 or the X (Close) button are used to exit the dialog.

[&Show ABC,button= "Cancel" ; S-Test ;cancel]

Note: ABC is a user-defined screen.

9 OK and no-help parameters

When this parameter is used as part of a dialog button definition the button is triggered when the Return/Enter key is used to exit the dialog.

[&Show ABC ,button= "OK" ; S-Test2 ;ok,no-help]

Note: ABC is a user-defined screen.

10 show-list parameter

[&Show ABC]

Shows the first or currently selected iteration of the screen.

[&Show ABC, show-list]

Shows the list of screens created, as it appears in the database.

11 Nodelete parameter

[&Show ABCD, nodelete]

12 screen parameter

[&Show screen="ABC", highlight-fields="1,4,7"]

or

[&Assign LF01="ABC"][&Assign LF02="1,4,7"]

[&Show screen=LF01,highlight-fields=LF02]

are the equivalent of

[&Show ABC:"1,4,7"]

13 button parameter; run button script

Note: S-TEST-OK is a script.

[&Show INP,button= OK ; S-TEST-OK ,no-cancel,no-help,no-navigate]

[&Message "After show"]

S-TEST-OK

[&Message "Testing OK"]

When the first script is run and the OK button is selected, the S-TEST-OK script is run and the “Testing OK” message is displayed.

The &Message command is not run because processing stops when the button is clicked and the script runs; this is the same behaviour as the next-action parameter. For more information please refer to Stateless Scripting Options.

14 Size-to-fit parameter

[&Show ABC ,size-to-fit]

15 Show SOS PM screens - screen-variant parameter

[&Show PM, screen-variant="BILL_LIST"]

"BILL_LIST": displays the Practice Manager matter bills list.

[&Show PM, screen-variant="LA"]

"LA": displays the Practice Manager matter enquiry LA tab.

[&Show PM, screen-variant="RELS"]

"RELS": displays the Practice Manager matter enquiry Relations tab.

Note: These codes can be used with either [&Show AL ... ] or [&Show PM ... ] the same screens are displayed when AL or PM is used.
