&Select HS

Syntax    

[&Select HS,History-no = <history id>]

Example Scripts

Description

The &Select HS command is used to select history items.

Parameters

history-no = <history id>

Selects the history item identified by the value stated.

no-load-file

This parameter is only applicable to systems with the Billing Module.

Ensures that the billing file remains the focus of the script rather than the originating file.

:first

Select the first iteration of a group or system code.

:last

Select the last iteration of a group or system code.

:next

Select the next iteration of a group or system code.

:prev

Select the previous iteration of a group or system code.

Example Scripts

1 Select a history item

This code selects a history item identified by the value stated.

Assign LF02 the history ID (HS26) of the history item you want to select. For example, could be as a result of a grid selection.

[&Select HS, history-no=LF02]

2 Select a history item, HS34

Select history items using HS34, History ID value, e.g.:

[&Select HS, history-no=<history-number>,no-load-file]

or

[&Select HS, history-no=<history-number>]

Where <history-number> is an expression that evaluates to the value of HS34 for the history item to be selected.

Note: Unless the no-load-file parameter is used, the file that the history belongs to is automatically selected. For this reason it is recommended that the no-load-file parameter is always used within an application script or when a matter has been pre-selected via scripting.

3 Select a history item no-load-file

The information in this example is only applicable to systems with a Billing Module.

This code selects a history item from the billing item that refers to it. Ensures that the billing file remains the focus of the script rather than the originating file.

A history item is selected from the billing item that refers to it using the field code BI05 for each bill item.

[&Select HS,History-no = BI05,no-load-file]

4 Displays HS01 value for first history record

Displays HS01 value for first history record (by date/time created) since there are no more records before first.

[&Select hs:first]

[&Select hs:prev]

[&Message HS01]

5 Display HS01 value for last history record

Displays HS01 value for last history record (by date/time created) since there are no more records after last.

[&Select HS, history-no=PI01]

[&Select hs:next]

[&Message HS01]
