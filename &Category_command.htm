&Category

Syntax    

[&Category type = <ApplicationTypeCode>]

Example Scripts

Description

This command enables the creation and maintenance of a category hierarchy from within scripting. Categories can be added, amended, moved and deleted. Category references can also be added.

The &Category command also enables the case-create script to build a default hierarchy for each new case.

Note: Category creation is done by default, so there is no create parameter on the &Category command.

Parameters

amend

Notifies that the category description is to be amended.

delete

Notifies that the category, sub-levels and references are to be deleted.

description = "<description>"

The description of the category to be created/associated.

move

Moves the category, sub-levels and references to another.

move-to = "<move-to>"

Represents the category to move the selected category to.

newdescription = "<newdescription>"

The new description of the category.

ownercode = "<ownercode>"

Represents the reference number for the owner; i.e. if the type is H then this field contains the history ID number. The history number to create a reference for (HS26).

parentcode = "<parentcode>"

The internal file details code for the owner (MT30). Typically this is the internal code for the entity or matter.

remove

Removes the nominated reference from this matter.

type = "<typecode>"

The type of category to be added; H = history items.

Example Scripts

1 Adding a category.

[&Assign LF01 = "Emails\Personal;Emails\Work"]

[&Category type="H",description=LF01,parentcode=MT30]

[&Message status(category)]

Note: The full path must be specified when using this command since multiple categories of the same description may exist.

2 Amending a category.

[&Assign LF01 = "Emails\Work"]

[&Assign LF02 = "Office"]

[&Category type="H",description=LF01,parentcode=MT30,Newdescription=LF02,amend]

[&Message status(category)]

Note: The full path must be specified when using this command since multiple categories of the same description may exist.

3 Deleting a category.

[&Assign LF01 = "Emails\Work"]

[&Category type="H",description=LF01,parentcode=MT30,delete]

[&Message status(category)]

Note: The full path must be specified when using this command since multiple categories of the same description may exist.

4 Moving a category.

[&Assign LF01 = "Emails\Personal"]

[&Assign LF02 = "Correspondance"]

[&Category type="H",description=LF01,parentcode=MT30,Move-to=LF02,move]

[&Message status(category)]

Note: The full path must be specified when using this command since multiple categories of the same description may exist.

5 Adding a category and a reference.

[&Assign LF01 = "Emails\Personal\Home"]

[&Category type="H",description=LF01,parentcode=MT30,ownercode=HS26]

[&Message status(category)]

Note: The full path must be specified when using this command since multiple categories of the same description may exist.

6 Removing a Reference.

[&Assign LF01 = "Emails\Personal;Emails\Work"]

[&Category type="H",description=LF01,parentcode=MT30,ownercode=hs26,remove]

[&Message status(category)]

Note: The full path must be specified when using this command since multiple categories of the same description may exist.
