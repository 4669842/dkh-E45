&Mail

Syntax    

[{&Mail to = Recipient}]

Example Scripts

Description

Sends an email and can be used within scripts and documents.

By default emails are generated in text format. In order for Outlook to recognize hyperlinks emails need to be generated in HTML format, this is achieved by adding the HTML parameter to the &Mail command and add the necessary html code to define a hyperlink. See example 10.

Note: Some of the parameters are only available as part of the standard interface to Outlook or Lotus Notes and cannot be used with GroupWise.

Outlook 2000/2003 Scripts

If you have &Mail scripts designed for use in earlier versions of Outlook, formatting such as <CR> carriage returns, or Plain and RTF formats, may not work correctly when the scripts are used in Outlook 2007 or later. You may also experience a loss of functionality when sending RTF format emails as HTML in Outlook 2007, which was not available in earlier versions.

There are workarounds to resolve this loss of functionality:

  • If &Mail command scripts do not have a mail format defined, for example ,RTF, add this parameter to the code. Alternatively, set the default format in Outlook to RTF.

  • If <CR> formatting is not working correctly, add ,HTML to the &Mail command and insert a <BR> tag in the message text where line breaks are needed.

Parameters

amend

Launches the Email Message dialog, enabling changes to be made to the message before it is sent.

attach

Causes the document in which the command is embedded to be included in the email message as an attachment.

bcc = "< bccRecipients>"

Specifies the BCC (blind carbon copy) recipients of emails.

Note: Cannot be used with GroupWise.

cc = "<ccRecipients>"

Specifies the CC (carbon copy) recipients of emails.

Note: Cannot be used with GroupWise.

delivery

Request an automatic delivery receipt.

Note: Cannot be used with GroupWise.

encrypt

Set the encryption flag on the email before sending.

Note: Cannot be used with GroupWise.

file = "<path&filename1, ...>"

Add file attachments to an email. This parameter defines the filename and path. Multiple attachments are comma-delimited. See example 4.

from = "<name>"

Specifies the From field value.

Enables the From field value to be amended so that for example, a generic email address can be specified rather than an individual, i.e. "HelpDesk@Client.co.uk".

HTML

Converts output to HTML format. For more information see Email Signatures.

Note: Cannot be used with GroupWise.

importance = "<high>"

importance = "<low>"

Assign a level of importance.

The following parameter is only available as part of the standard interface to Lotus Notes.

message = TextString

The value of this parameter is displayed in the body of the email.

no-wait

The no-wait parameter is for use in Visualfiles Browser only to ensure that scripts continue after the mail is generated rather than locking up an agent. Ensure that the &Mail command is the last interaction in a script otherwise there is the potential for two scripts to be running simultaneously which can cause problems.

nofileopen

Suppress the Do you want to open this file? prompt.

Note: Cannot be used with GroupWise.

nohistoryprompt

Remove the confirmation of the history narrative when saving an email to history.

Note: Cannot be used with GroupWise.

Plain

Compose the email in plain text format.

read

Request an automatic read receipt.

Note: Cannot be used with GroupWise.

RTF

Compose the email in rich text format (RTF).

save

Saves the email to the History of the current file.

signature = ""

Used to remove the default signature.

subject = Subject

The value of this parameter is displayed in the Subject line of the mail.

to = Recipient

Email address of the recipient of the email.

version

Enables versioning of emails when they are saved to history.

Example Scripts

1 attach parameter

[&Mail to = "Nick",subject = "Lease for Review",message = "Please forward your comments by Friday",attach]

2 amend parameter

This example shows the use of temporary fields within the message parameter.

[&Mail to = "j.smith@visualfiles.com",subject = "Email Confirmation",message = "The meeting will go ahead on " +TF01,amend]

3 save parameter

The field CON05 holds an email address which is then used with the to parameter of the &Mail command.

[&Mail to = "CON05",subject = "Meeting",message = "The meeting is confirmed",save]

4 file parameter

Attaching the file questionnaire.docto an email.

[&Mail to = "a.taylor@visualfiles.com,subject = Please complete the attached questionnaire,message = Complete and return back to me by next Monday,file = c:\my

Attach multiple files within the &Mail command using temporary fields:

[&Assign TF01="C:\my documents\doc1.doc,C:\my documents\doc2.doc"]

[&Mail to = "test.user@lexisnexis.co.uk",subject = "Please complete the description",message = "Test",file = TF01]

Attach two temporary fields storing a file location in each:

[&Assign TF01="C:\my documents\doc1.doc"]

[&Assign TF02="C:\my documents\doc2.doc"]

[&Assign TF03=TF01+","+TF02]

[&Mail to = "test.user@lexisnexis.co.uk",subject = "Please complete the description",message = "Test",file = TF03]

5 bcc parameter

In this example, the email to Fred will also be sent to Bert and Mary but Bert will not know that Mary received the email as well.

[&Mail to = "fred",cc = "bert,bcc =

6 read parameter

This example requests an automatic read receipt and assigns the value of the temporary variable TF02 to the message parameter.

[&Mail to = "fred",read,subject = "sale of house",message = "TF02"]

7 importance parameter

The field EMP09 holds an employee's email address which is used with the to parameter of the &Mail command.

[&Mail to = EMP09,importance = "high",subject = "Sale of house",message = "Money has been credited to your account"]

8 encryption parameter (Lotus Notes only)

[&Mail encrypt,to = "fred",importance = "low",subject = "sale of house",message = "money has been credited to your account"]

9 HTML parameter

This code is used to convert output to HTML format.

[&Mail to="support@visualfiles.com", subject="test", message=TF02+TF01,HTML]

10 HTML parameter, define a hyperlink
