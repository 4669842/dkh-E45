&DoWhile

Syntax    

[{&DoWhile <subject> <test> <object>}

[&And{<subject> <test> <object>}|&Or{<subject> <test> <object>}|&Xor{<subject> <test> <object>}]...]

<Text or further Visualfiles commands>

[{&EndDoWhile}]

Example Scripts

Description

Creates an iterative loop running a section of script between the &DoWhile and &EndDoWhile commands. The script will continue to loop to the beginning of the &DoWhile expression as long as the defined logical test (<subject> <test> <object>) is met.

Each &DoWhile command must have a corresponding &EndDoWhile command.

The &DoWhile construct consists of a block of code and a condition. The condition is first evaluated and if the condition is true, the code within the block is executed. This repeats until the condition becomes false. Because &DoWhile loops check the condition before the block is executed, the control structure is also known as a pre-test loop.

Note: The default setting is noincludeall which is the default setting on a &DoWhile loop.

For more information on nesting iterative loops see Nested statements in documents.

Parameters

<object>

The <object> of the test can be any type of user input: direct text/number/date input, a database field, a temporary field, a global field, an input parameter or an output parameter.

The <subject> and <object> must be the same type: text must be compared to text, numbers to numbers, date to date.

(Question)Where there is not a specific database field to test against, a user can be prompted for a Yes/No response using the question modifier.

<subject>

The subject to be tested can be any type of user input: direct text/number/date input, a database field, a temporary field, a global field, an input parameter or an output parameter.

The <subject> and <object> must be the same type: text must be compared to text, numbers to numbers, date to date.

<test>

The <test> used may be any of the following:

=

<>

>

<=

>=

&Contains

&Begins

&IsOneOf

Equal to

Not equal to

Greater than

Less than or equal to

Greater than or equal to

Tests if text in the Subject contains the Object value.

Tests if text in the Subject begins with the Object value.

Tests if text in the Subject is one of the list of values.

The following commands can be used to join tests:

&And

&Or

&Xor

Condition is met if both tests are true.

Condition is met if either test is true.

Condition is met if only one test is true.

&EndDoWhile

Terminates the &DoWhile command.

Example Scripts

1 Repeat the loop whilst TF01 does not equal 10

[&Assign TF01 = 0]

[&DoWhile TF01<10]

   [&Assign TF01 = TF01+1]

   [&Message "This message will open 10 times!"]

[&EndDoWhile]

2 Force the entry of a valid date

In this example, the screen ACC holds accident details and the field ACC01 the date of the accident. A warning message is displayed until a date before 01/01/1900 is inserted into the field ACC01.

[&Show ACC:1]

[&DoWhile ACC01<"01/01/1900"]

   [&Message "Please enter the accident date"]

   [&Show ACC:1]

[&EndDoWhile]

3 Force the entry of a valid selection using &And join.

In this example, the screen DET holds car insurance quote details. Field DET02 is the number of points the driver has and DET16 is a text selection field where the insurance type is selected. If six or more points are selected in DET02 and fully comprehensive insurance is selected, a message is displayed to tell the user this option is not available.

[&Show DET:2,16]

[&DoWhile DET02> = 6 &And DET16 = "Fully Comprehensive"]

   [&Message "Fully Comprehensive is not offered to drivers with 6 points or more"]

   [&Show DET:16]

[&EndDoWhile]

4 Extended iterative statement

This example searches for customer names repeatedly while the value of TF01 is assigned the value Yes. If the user selects No, the expression &DoWhile TF01 = Yes will fail and the search will end.

[&Assign TF01 = "Yes"]

[&Assign TF02 = ""]

[&DoWhile TF01 = "Yes"]

   [&Assign TF02 = 'Enter Name of Customer'(text,default = TF02)]

   [&Assign TF03 = ""]

   [&Conflict contains = GF01, type = "COMM",role = "CUSTOMER", list = TF03, title = "Customers matching word "+TF02]

   [&If TF03>"0"]

      [&Select Entity, int-code = TF03, ref = "Entity"]

      [&Open ref = "ENTITY"]

      [&Assign TF01 = "No"]

   [&Else]

      [&Assign TF01 = 'No Matches Found - Do you wish to search again?'(question)]

   [&Endif]

[&EndDoWhile]

Developers' Tips

When testing a check box, avoid using ABC01 <>No, as it may return items where the check box's default value is blank or where the check box has not been amended.
