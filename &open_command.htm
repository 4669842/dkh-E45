&Open

Syntax    

[{&Open}]

Example Scripts

Description

The &Open command is used to open:

  • A document/file, held on the file system but external to Visualfiles.
  • A web page, for example, when linking to the SearchFlow web site (only using Internet Explorer).
  • Visualfiles files and entities.
  • The initial screen displayed when a Visualfiles file or entity is opened.
  • A history item.
  • Data in an excel spreadsheet.

Specifying the Initial Screen

The &Open command is used to specify the screen that is initially displayed when a case or entity is opened using the following parameters:

  • Add-in="<add-in-code>"
  • Group="<group-code>"
  • Role="<role-code>"
  • Screen="<screen-code>"|"<prefix-screen-code>"

When opening a case or entity a nominated entity, role, file or add-in screen can be displayed rather than the file cover. This functionality helps applications work more efficiently in Visualfiles Browser, reduces the number of prompts displayed to users and eliminates the need for the user to manually navigate to the relevant screen.

If the information supplied on the &Open command is invalid and the nominated screen cannot be found, the entity or file opens as normal but displays the standard file cover. Make use of the Status, System Variable for this command to indicate if the screen is not found.

Override-display Parameter and the &Display command  

The &Display command can be used in Open-Case and Open-Entity scripts. If the user needs to ignore the &Display commands that are used in these scripts, they can do so by adding the override-display parameter to the &Open command. See example 9.

Parameters

add-in = "<add-in code>"

Used to display an add-in screen when opening a file by specifying the code of the add-in concerned.

This option is invalid if you are opening an entity, or if the screen= parameter is not present on the command. See examples 8, 9, 15, 17, 18.

case-menu

Display a web page or document in the case menu window with a navigation menu: Forward, Back, Home, Refresh or to call another URL.

Note: These scripting options are also available within the Visualfiles user interface, Additional Tabs option, when you are Editing Application Details, Editing Entity Type Details, Editing Role Details and Editing Add-in Details.

There are three options related to the case-menu parameter that are used when opening a web page into a File tab (see examples 19):

no-navigation

no-popups

silent

Prevents the display of an address bar and forward/backward buttons on the web viewer page.

Prevents links on web pages from opening in a new window.

Stops the browser from displaying dialog boxes, e.g. script errors on the web page.

display = "<{window|dialog|auto}>"

The display parameter enables the developer to specify whether the file should be displayed in a window or a dialog box.

By default, files are opened in a dialog box. Consequently, no other Visualfiles dialogs or windows may be opened until that dialog has been closed. However, if the file is opened in a window, other Visualfiles windows may be accessed while the window is still open.

The value "auto" will open a file/entity in a window if the command is run from a window or in a dialog if the command is run from a dialog.

doc-view

Opens a History Item dialog in Document view.

excel

Opens data in an excel spreadsheet.

file = "<{FilePath|URL}>"

Specify the location and name of the file.

It is good practise to use the file = parameter when opening external files and web pages. In this way, these commands may be differentiated from &Open commands that are being used to open Visualfiles files and entities. So, although [&Open file = "c:\temp\document.doc"] and [&Open "c:\temp\document.doc"] essentially behave in the same way, it is recommended that the former be used for opening external files.

group = "<group-code>"

Used to display a group screen when opening a file or entity by specifying the code of the group concerned. The group code is a four character string.

This parameter is invalid if the screen= parameter is not present on the command. See examples 16-18.

history = <HS>

Points to a history item using the history system code, e.g. HS26.

main-menu

Display a web page or document in the Visualfiles window rather than in a web browser.

A navigation menu is displayed to enable page Forward or Back, Home, Refresh or to enter another URL.

open-document

Open the specified document.

openwith

Prompt the user to select which application should be used to open the file. The openwith parameter should not be used when attempting to open web pages.

Where a PC cannot recognise a file extension, this parameter displays the Open With … dialog box for the user to select the appropriate application. Where it can determine the application, the Open With … dialog will not open, even if openwith is included in the coding.

options = "autofilter"

Enables autofilters in the resulting spreadsheet for sorting of the excel data.

override-display

Use this parameter to ignore any &Display commands that may be included in the OPEN-CASE and OPEN-ENTITY scripts when a case or entity is opened.

Note: When an OPEN-CASE or OPEN-ENTITY script includes an &Display command and the override-display parameter is not used, it is the &Display command screen that is displayed rather than the screen displayed with the &Open command.

popup-height = <popup-height;"

Determines the height of any popup called from an embedded webpage.

popup-width = <popup-width;"

Determines the width of any popup called from an embedded webpage.

print

Prints an open document. Some third-party applications do not support this feature. The print parameter should not be used when opening web pages.

When documents are produced from scripts, the documents will be sent to the Document Review specified by the script. Otherwise, the document will be sent to its default Document Review (provided one has been set up).

ref = "<EntityReference>"

Opens an entity reference.

Unlike the ref parameter for other commands, the ref parameter for the &Open command finds a reference that has been specified previously in the script. See example 6.

role = "<role-code>"

Used to display a role screen when opening an entity by specifying the code of the role concerned.

This parameter is invalid if you are opening a file, or if the screen= parameter is not present on the command. See example 10.

screen = "<screen-code>"|"<prefix_screen-code>"

Used to specify the screen to be displayed.

This parameter is mandatory when switching the screen to display at startup. If it is not included then any Role, Add-In and Group options coded on the command are ignored.

When specifying this parameter the screen code must represent either a three letter screen code, or, for screens on a linked case type, the code must start with the prefix for the linked case type then an underscore followed by a three letter screen code. See examples 9 - 17

time-view

Opens a History Item dialog on the Time tab.

Example Scripts

1 Open a File or Website

The following command will open the word document, Test.

[&Open file = "c:\program files\my documents\test.doc"]  

The following command will open the Visualfiles website.

[&Open file = "www.visualfiles.com"]

2 Open an entity

This asks the user to select an entity with the role Introducer and open each of the clients they introduced.

[&Select ENTITY,role = "introducer"]

[&ForEach int_client]

   [&With int_client]

      [&Open]

   [&EndWith]

[&EndFor]

3 Controlling whether the entity opens in a window or a dialog box

The display parameter may be set to window (opens the entity in a window), dialog (opens the entity in a dialog) or auto (opens the entity in either a window, if the &Open command was run from a window, or a dialog, if the &Open command was run from a dialog).

[&SelectENTITY,type = "indi"]
[&Open display = "auto"]

4 openwith and print Parameters

The openwith parameter displays an open dialog box if it is unable to distinguish which program to use to open the file. The print parameter will then print test.doc.

[&Open "c:\temp\test.doc",openwith,print]

5 Opening a document in time-view and in document-view

This script selects the last history item, opens it in Time View and then again in Doc-View

[&Select HS:last]

[&OPEN HISTORY = HS26,TIME-VIEW]

[&OPEN HISTORY = HS26,DOC-VIEW]

6 Using the ref parameter

Here all News Items related to the Commercial entity are shown in the finder using the &SelectRelation command. The selected item is given the reference myNews.  This reference is used with ref parameter in the &Open command to open the item previously chosen.

[&SelectRelation type = "NEWS_COMM",ref = "mynews"]

[&Open ref = "mynews"]

7 Create an .spr file using &Finder and open in Excel, then auto format and add filter options

[&Finder file-only, type="AGEN",WORD="banking",output="file",filename="C:\dbtest.spr",dbfields="MAT11"]

[&Open excel="c:\dbtest.spr", options="autofilter"]

8 Call add-in from file

When the screen EST, is on an Add-in called from a file.

[&Open EST, add-in="SALEAA"]

9 override-display parameter

Script used for opening a case and ignoring any &Display commands in an Open-Case script:

[&Open MyCase, screen="ABC", add-in="Prospectaa", override-display]

10 Open an entity including &Display

Script used for opening an entity and allowing &Display commands in Open-Entity script to work:

[&Open MyEntity, screen="ABC", role="ALLCLIENTS"]

Note: This functionality also works with linked cases. When referencing a linked case screen the linked prefix must be added to the screen e.g.: S_ABC.

11 Screen parameter

[&SelectEntity int-code=LF01, ref="myCLIENT"]

[&With myCLIENT]

   [&Open screen="ADD"]

[&EndWith]

Note: ADD is the screen code.

12 Screen parameter

This code opens the entity or file and defaults to the screen with code ABC located at entity-type or application level.

If this screen does not exist at that level then the item opens normally and displays the standard file cover.

[&Open screen="ABC"]

13 Screen parameter, prefix_screen-code

This code opens the file and defaults to the screen with code ABC that is located on the linked case type with the prefix A.

[&Open screen="A_ABC"]

If the file you are opening is not linked to another case type with prefix A or the screen with code ABC cannot be found on the linked case type then the file opens as normal but displays the standard file cover.

If you attempt to open an entity using this syntax then the entity opens and displays the normal file cover, because the syntax is invalid for opening entities.

14 Screen and role parameters

This code opens the entity and defaults to the screen with code ABC that lives on the role with code INDI.

[&Open screen="ABC", role="INDI"]

If you attempt to open an entity using this syntax but the entity does not have that role, then the entity opens as normal and displays the standard file cover.

If you attempt to open a file using this syntax then the file opens and displays the normal file cover, because the syntax is invalid for opening files.

15 Screen and add-in parameters

This code opens the file and defaults to the screen with code ABC that lives on the add-in (case plan) with code CCRT.

[&Open screen="ABC", add-in="CCRT"]

If you attempt to open a file using this syntax but the file does not have that add-in (case plan) then the file opens as normal but displays the standard file cover.

If you attempt to open an entity using this syntax then the entity opens and displays the normal file cover, because the syntax is invalid for opening entities.

16 Screen and group parameters

This code opens the entity or file and defaults to the screen with code ABC located inside the group CLAI.

[&Open screen="ABC", group="CLAI"]

If you attempt to open a file or entity using this syntax but the item does not have that group or the screen cannot be found inside that group, then the item opens as normal and displays the standard file cover.

17 Screen, add-in and group parameters

This code opens the file and defaults to the screen with code ABC that is located inside the group WITN in the add-in (case plan) with code CCRT.

[&Open screen="ABC", add-in="CCRT", group = "WITN"]

If the file you are opening does not have the add-in (case plan) CCRT or the group WITN cannot be found inside add-in CCRT, or the screen ABC cannot be found inside group WITN, then the file opens as normal but displays the standard file cover.

If you attempt to open an entity using this syntax then the entity opens and displays the normal file cover, because the syntax is invalid for opening entities.

18 Open file screen parameter excluded

[&Open add-in="CCRT", group = "WITN"]

If you attempt to open an entity or file using this syntax then the item opens and displays the normal file cover, because the syntax is not specifying a default screen.

19 case-menu parameter options

[&Open file="www.lexisnexis.co.uk", case-menu, no-navigation, no-popups, silent]
