&Out

Syntax    

[&Out "<PathName>"]

Example Scripts

Description

The &Out command is used to send information contained in fields or from direct input to a new or existing output file.

The values for <PathName> and <FileContent> must be separated by a colon.

Note: If you using the Data Extract Module the &OutPutTo command can be used to define the output file of the extract. Data extract can be output to Microsoft Access, Microsoft Excel, standard text documents, XML documents or other databases linked through ODBC/Microsoft Jet Engine.

Parameters

append

Append data to an existing output file.  If this parameter is omitted, a new file will be created or the existing file overwritten.

"<:FileContent>"

Specify the data to be stored in the output file.  If this parameter is omitted, the output file will contain only the filename.  Where multiple fields are to be output to a single file, they should be separated by an appropriate delimiter.

"<PathName>"

Specify the location of the file to which the data will be output.

Example Scripts

1 Transfer data and append to a file

This script will transfer the data contained in fields CL21, CX08 and CX09 and append them to the file C:/VFileData Transfer/CaseOutput, with commas separating the fields.

[&Out "C:/VFileData Transfer/CaseOutput":CL21+","+CX08+","+CX09,append]

2  Appending data to a file

This example will append comma separated data from fields UDF01, UDF02 and UDF03 to the file C:\VFileData Transfer\Output.

[&Out "C:\VFileData Transfer\Output":UDF01+","+UDF02+","+UDF03,append]

3 Asking the user to select a file location

In this example, the fields UDF01, UDF02 and UDF03 are separated by pipes and appended to a user selected file.

[&Out 'Select location'(Filename):UDF01+"|"+UDF02+"|"+UDF03,append]

4 Storing a delimiter in advance

This example stores a comma as the delimiter to a temporary field before using it in outputting the three data fields UDF01, UDF02 and UDF03 to the file C:\VFileData Transfer\Output.

[&Assign TF01 = ","]

[&Out "C:\VFileData Transfer\Output":UDF01+TF01+UDF02+TF01+UDF03]

5 User selected delimiter and file location

This example stores a user specified delimiter in advance (to make typing easier) and uses it to separate the fields UDF01, UDF02, and UDF03 when appending them to the user selected file.

[&Assign TF01 = 'Specify delimiter']

[&Out 'Select location'(Filename):UDF01+TF01+UDF02+TF01+UDF03,append]

Developers' tips

Information is stored in plain text format regardless of the file extension specified. If the data is to be examined in a spreadsheet, each field should be separated by a delimiter, typically either a comma or a pipe, and stored to a Comma Separated Value.csv file.

The way in which Windows® addresses certain directories within a user profile means that the &Out command cannot save files in the following folders: My Documents, Desktop, Start Menu or Favorites. If a user attempts to do so, the error message Invalid filename: PathName will be displayed.
