&ForEach

Syntax    

[{&ForEach [<GroupScreen>|<RelationCode>|<RoleCode>|<EntityCode>]}]

<Text or further scripting, for example, tests, user input prompts and task commands>

[{&EndFor}]

Example Scripts

Description

An iterative command that selects each instance/iteration within a group or a relation for further processing by a script.  

Every &ForEach command must have a corresponding &EndFor command.

&ForEach can also be used with BG, BI and BT codes.

Nested &ForEach statements in documents.

&ForEach can be used with the following system screens:

AB - Address

AD - Schedule

DB - Debt Invoice

LG - Ledger

HS - History

HV - History Versions

MT - File Details

PY - Payment

RC - Recoverable Costs

Note: &ForEach MT cannot be used within a file.

Parameters

descending

Reverse the direction of the sort (the default is ascending: there is no 'ascending' parameter).

Filter = History  Enables the &ForEach command to be filtered to show only history items.

Filter = Ledger  Enables the &ForEach command to be filtered to show only Ledger items.

Filter = live  Ensures that only live matters are included in a test.

Filter = completed  Used to exclude current matters from a test.

Group-by = <BG01>

This allows groups of BI fields to be subtotalled: returning a BG record where all BI records with the same Source Type (BG01) have been added together.

Leave-field = <TF>

This parameter is used to specify a temporary field (TF) so that if it is set to any non-blank value it makes the &ForEach loop finish after the current iteration. See example 7 & 8.

noincludeall

Ensure that the information collected is not too large for the allocated memory available.

Note: When this parameter is used only one line is displayed in the resulting document. The parameter works by overwriting the previous line. Ensure that this parameter is removed to display all the required content.

Prefix =

This parameter enables &ForEach relations to work on linked cases.

On linked cases each of the two linked applications has a unique prefix letter which is used to identify the application the relation is on. See example 6.

Note: If the Prefix parameter is not used then the &ForEach command defaults to the application type of the parent case in the link.

sort

Manipulate the order in which the information is presented. See example 2.

Note: This parameter is used to point directly to a field not down a relation.

It is possible to do multiple sorts on &ForEach using the Data Extract wizard. When multiple entries are added to the sort field in the &ForEach command, the sort fields are displayed in the format "MT05+MT04" and are processed as "sort by MT05 then sort by MT04". The scripting syntax is:

"sort=MT05,sort=MT04"

&EndFor

Terminates the &ForEach command.

<EntityCode>

Specify the entity code to be selected.

<GroupScreen>

Specify the group screen to be selected.

<RelationCode>

Specify the relation code to be selected.

<RoleCode>

Specify the role code to be selected.

Example Scripts

1 Instances of a Relation

[&Assign TF01 = 0.00]

[&ForEach OTHERPARTY_OTHERPARTY]

   [&Assign TF01 = TF01+1]

[&EndFor]

[&Message "The number of other parties for this file is "+TF01]

2 &ForEach used with the sort Parameter

This example sorts the field in the default ascending order and then outputs each field entry as a message.

[&ForEach LG,sort = LG01]

   [&Message LG01]

[&EndFor]

3 &ForEach with the descending parameter

In this example, the field HS02 is sorted in descending order and then outputs each field entry as a message

[&ForEach HS,sort = HS02,descending]

   [&Message HS02]

[&EndFor]

4 Test the value of a field

This example selects each iteration within the group CLAI and checks to see if the field CLM23 = "No", if it does, the script also assigns CLM24 = "No".

[&ForEach CLAI,CLM23 = No]

   [&Assign CLM24 = "No"]

[&EndFor]

5 Loop through all entities in a database

In this example, the script will loop through all the entities in a database. Where the entity has the role of Expert, the entity ID will be appended to a text file.

[&ForEach COMM,noincludeall]

[&SelectRole type="EXPERT"]

  [&If Status(selectrole)=""]

  [&Out "c:\expert file.txt":EN56,append]

  [&Endif]

[&Endfor]

 

Or

[&ForEach EXPERT]

  [&Assign TF01=EN56:whole:text]

  [&Out "c:\tmp\experts.txt":TF01,append]

[&Endfor]

6 Prefix parameter

If you have linked Sales and Purchase where the prefixes are S and P respectively and you are trying to do a &ForEach on the Seller_Seller relation which is stored on the Purchase application the syntax is:

[&ForEach Seller_Seller,Prefix=P]

7 Leave-field parameter

At the start of the loop the temp field is set to blank. If this field is set to any non-blank value during the loop then no more iterations of the loop will be processed.

&ForEach MT

   &ForEach CRIT ***loop through all CRIT entities**

      ForEach CRIT_PROD,leave-field=LF01 **loop through PROD entities related to the CRIT entity**

         &If SomeCondition

            LF01="Done"

         &Endif

      &EndFor

 

      &ForEach CRIT_REGI,leave-field=LF01 **loop through REGI entities related to the CRIT entity**

         &If SomeCondition

            &LF01="Done"

         &Endif

      &Endfor

   &EndFor

&EndFor

8 Leave-field parameter

At the start of the &ForEach command it is automatically cleared and is then checked at the end of each iteration to see if has been set to any value.

If any non-blank value has been set then the for each will end at that point:

[&Assign LF02 = 1]

[&ForEach HS,leave-field=LF01]

 [&If LF02 = 5]

  [&Assign LF01 = "Limit reached"]

 [&Else]

  [&Assign LF02 = LF02 + 1]

 [&EndIf]

[&End]

Note: It does not matter what value the leave-field is set to. As long as it is not blank then ForEach loop will end.

9 &ForEach MT filter=live

This code is used to go through each live matter and assign the field GEN01 to be Monday. The noincludeall option is added on every &ForEach to ensure that the information collected does not exceed available memory.
The syntax filter=Live is specific to MT File Detail codes.

[&ForEach MT,filter=live,noincludeall]

    [&assign GEN01="Monday"]

[&EndFor]

10 &ForEach AB (Address Book) Codes

Include the following &ForEach AB (Address Book):

[&ForEach AB]

You can then message out the address book fields:

[&Message AB01]

There are two other screens related to address books:

  • Additional Details (ADG)

  • Additional Details for Address Book Type (ADD).

In effect ADG is a global screen for address book and ADD is specific to the address type.

Both of these screens can be accessed using the syntax:

[&Message AB99(ADG01)]

[&Message AB99(ADD01)]

Note: See also User-defined Address Book Fields.

11 Instance of a relation and interrogate the entity

This code displays the FIL01 field from the Outcome Entity for all outcomes that have a relation back to the receive complaint add-in.

[&ForEach AIOUTCOME_OUTCOME]

   [&With AIOUTCOME_OUTCOME]

      [&Message FIL01]

   [&EndWith]   

[&EndFor]

Developers' Tips

&ForEach statements can only be used in scripts and Microsoft Word documents: they do not work in any other word processing package. In Word documents, depending on the version in use, you may need to place each statement on a separate line.

GR codes (&Grid command) can be used within an &ForEach command:

[&ForEach GR]

   <some processing>

[&EndFor]
