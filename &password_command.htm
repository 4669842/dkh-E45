&Password

Syntax    

[{&Password "<Passwordtext>"}]

or

[{&Password}]

Example Scripts

Description

The &Password command enables you to set a password for each file which can then be used in any script. It is commonly set up in the Create File Script and used in the Open File Script to prevent unauthorised users from opening files, or in prescripts to limit access to documents or scripts.

The &Password command lets you assign a password to a file (stored in MX25 by default), either by specifying a password using, [&Password "<password>"], or by generating a password for you when no password is supplied.

When password rules are defined in the Password Configuration screen, any passwords generated consist of 16 alphanumeric characters that conform to the selected password rules. Where no parameters are specified, &Password generates a random string as a password.

Alternatively you can use the &Assign command to set the file password.

It is also possible to assign passwords to fields other than MX25.

Parameter

<AllInputTypes>

Specify the password.

No-cancel

Used with the prompt parameter to prevent users cancelling out of the password entry screen.

No-validate

Prevents password validation being applied. When this parameter is used with the prompt parameter, password rules (defined in the Password Configuration screen) are turned off.

When No-validate is used is used in the &Password command to generate passwords, the passwords generated revert to the old style five character password regardless of any password rules defined.

Old=<old_password>

Used with the prompt parameter to specify a password that has to be entered along with a new password.

Online-only=<{”yes"|"no"|"current”}>

Used to specify whether a file password is applied to all users or just Visualfiles Online users.

When this parameter is specified the Online users only check box appears on the change password dialog that is displayed when the prompt parameter is applied.

Prompt [,title=<screen title>][,old=<old password>][,no-cancel]

Used to display a dialog where users can change their password. The title of the dialog is specified using the title= parameter.

If an old password is supplied with the old= parameter, this must be entered before a new password can be set.

The no-cancel parameter forces the user to enter a password by removing the option of cancelling out of the password entry screen.

If the online-only parameter has been specified the option Online users only is displayed in the change password dialog that enables you to apply the file password to Visualfiles Online users only.

Result=<field_code>

This parameter allows the password to be stored in a field other than the default location of MX25.

Title=<Screen_Title>

Used in conjunction with the prompt parameter to specify a title for the password entry screen.

Example Scripts

1 Set MX25 to the value of temporary field LF01

[&Password LF01]

2 Generate a password and store it in LF01

[&Password result="LF01"]

3 Prompt user for a password and store it in LF01

[&Password prompt,result=LF01]

4 Prompt for a new file password displaying the File Code

[&Password prompt,title="Enter New Password for Case "+MT05,old=MX25]

Prompt for a new file password showing the File Code in the screen title and requiring the user to first enter the old file password.

5 use password style character blanking

User input prompts have a password option that allows them to use password style character blanking:

[&Assign LF01='Enter Password'(password)]

Password characters are displayed as bullets.

6 Define a random password to be remembered by the user

This code automatically generates a password and assign it to the field MT25.

[&Password][&Message "Your random password is "+MT25]

7 Define a password

This code can be included in a Create File script for an application and will set a password for each file of that type.

[&Password "Password"]

8 Using MT25

A password can also be set by using the &Assign command and setting the value of MT25.

[&Assign MT25 = "Password"]

9 Use MT25 in a script to prompt the user to enter a password access a file

MT25 can be referenced in an Open File script to prompt the user to enter a password and gain access to a file.

If the user types an incorrect password, the &Quit command will end the script.

Enter the password between the quotes at MX25.

[&Assign MX25="Password goes here"]

[&If MT25<>'Please enter file password']

[&Message "The password you entered was incorrect for this file"]

   [&If MT25<>'Please enter the correct password']

      [&Quit]

   [&EndIf]

[&EndIf]
