&DraftBill

Syntax    

[&DraftBill]

Example Scripts

Description

The &DraftBill command without accompanying parameters will open the Select Items For Bill dialog box.

Adds Ledger and Time items to a specific draft Bill file.

&DraftBill can be used in conjunction with additional parameters to filter Ledger and Time items to bill:

  • by start or end date

  • by period

  • select all items

  • remove all items

  • prevent the selection dialog from opening

Only items that are set as Ready to Bill or those items which are set as unauthorised, where the case is marked as Unauthorised, may be added to the draft bill.  This command does not process unauthorised items, where the case is marked as Ready to Bill, excluded Ledger and Time items, or items on cases where the case is marked as Excluded.

Note: The information on this page is only applicable to systems with an activated Billing Module, and where an application has been set with the application type of Billing Module. The bill case must also have been moved to the Draft Bill milestone.

Note: This command is not supported in Visualfiles Browser.

Parameters

bill-int-code = "<SourceFileInternalCode>"

Filters the items to include only those from a specified file, as identified by the file's internal code (MT30).

bill-relation="<RelationCode>"

Relation code between the Bill and the related entity.

end-date = "<EndDate>"

Filters the items listed to display only those that fall between specified start and end dates.

end-period = "<PeriodCode>"

Filters the items to display only those that fall between specified Periods. The start and end dates defined in the Period Codes automatically define the start date and end date.

entity-code="<EntityCode>"

The entity code of the related entity

entity-relation="<EntityRelationCode> "

The relation between the entity and the matter.

no-display

Prevents the selection screen from being displayed.

remove-all

Removes all the available billable items with the specified files and dates to the bill.

select-all

Adds all available billable items from the specified files and dates to the bill.

start-date = "<StartDate>"

Filters the items listed to display only those that fall between specified start and end dates.

start-period = "<PeriodCode>"

Filters the items to display only those that fall between specified Periods. The start and end dates defined in the Period Codes automatically define the start date and end date.

type = <Type>

Filters the items to include only those of a specified type: either Time or Ledger.

unique

File will not be billed if it is related to more than one entity with the relation code specified in entity-relation, an example may be where there are multiple clients linked to a case.

Example Scripts

1 start-period parameter and end-period parameter

 This code automatically retrieves from predefined period codes the start date of period code 2005M11 and end date of the period code 2005M12 and bill all available items between and inclusive of those dates.

[&DraftBill ,Start-period="2005M11",End-period="2005M12"]

2 bill-int-code parameter

Note: This allows only those available items for a specific bill case to be draft billed. This parameter is most often used outside the billing application type, for instance in a system script or case level script where a draft bill is being created, preventing loss of focus. In this instance a draft bill will be created for the bill case with the internal code (MT30) of 2365. The internal code (MT30) can also be assigned to a temporary field (TF) and then used within the code.

[&DraftBill ,bill-int-code="2365"]

Or

[&Assign TF01="2365"]

[&DraftBill ,bill-int-code=TF01]

3 type parameter

Note: This code will only include those of the type Ledger in the Select Items For Bill dialog box.

[&DraftBill type=ledger]

4 bill-int-code, bill-relation parameter, ,entity-code parameter and entity-relation parameter

Note: All the parameters must be used in conjunction with each other; the parameters can be populated by temporary fields (TF). This code will automatically define relations from the bill with the internal code of 2365 to those cases available for billing linked to the client who the bill is for. The draft bill will then be produced and will display all available billable items linked to those cases that have been related in the Select Items For Bill dialog box.

[&DraftBill bill-int-code="2365",bill-relation="BILLEDFILE_BILLEDFILE",entity-code="3346",entity-relation="INSTRUCTINGCLIENT_CLIENT "]

Note: The bill-relation is the relation that has been defined between the bill case and the case types which are to be billed, for instance, General. The entity code relates to the internal code of the entity which the cases are created from, for example, a commercial entity with the role of client. The entity-relation is the relation between the general cases to be billed and the entity from which these cases are created.

5 unique parameter

Note: This code prevents cases being billed where the entity relation has multiple occurrences, for example in this instance if there was more than one client linked to the general case the case would not be included in the bill.

[&DraftBill bill-int-code="2365",bill-relation="BILLEDFILE_BILLEDFILE",entity-code="3346",entity-relation="INSTRUCTINGCLIENT_CLIENT",unique]
