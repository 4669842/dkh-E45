&GetDocument

Syntax    

[&GetDocument document = <UniqueHistoryNumber>]

Example Scripts

Description

Enables a user to save a local copy of a History document, and is used to extract documents to the Briefcase.  Documents can be read from any location, including document management systems.

The &GetDocument command can be used with linked entity documents; it works with the Linked Document field on an entity.

Note: This command is not supported in Visualfiles Browser.

Parameters

directory = "<DirectoryPath>"

The directory in which the file will be returned. If no directory is specified, the file will be copied into the current working directory.

document = <UniqueHistoryNumber>

Identify the document to be copied by its unique History number (HS26).

Note: In a linked system unique History number is HS73.

filename = "<FileName>"

It is recommended that a meaningful filename be assigned to the file. If no filename is specified, the name of the original document will be used. The file extension is not required, it will be read from the system.

,overwrite

Overwrite any existing file with the same name. The command will fail if this parameter is not used and a file with the same name exists in the directory specified.

print

The print parameter enables you to select and print history documents through scripting. Print button functionality in the History screen has been hooked up to the &GetDocument command with the addition of the print parameter.

Use this parameter to print to your default printer. See examples 6.

result = <FieldName>

Return the full document path into a temporary field or database field.

version = "<Version>"

Specify the version to copy. If no version is specified, the master version will be copied. If no master version exists, the most recent version will be copied.

Example Scripts

1 version parameter

Retrieve the first version of the History document:

[&GetDocument document=1234,version="1"]

2 directory parameter

In this example TF01 holds the history document number (HS26) and TF02 holds the directory in which the document is to be stored.

[&GetDocument document=TF01,directory=TF02]

3 filename parameter

The document will be saved with the extension associated with the originating program, for example, Retrieved.doc

[&GetDocument document=1234,filename="Retrieved"]

4 result parameter

[&GetDocument document=HS26,result=ABC01]

Note: The full path of the document is saved to the field specified, for example: C:\tmp\1234.doc

5 overwrite parameter

In this example TF01 stores the history code for the document, and TF02 will hold “c:\tmp\HistDoc.doc" with an existing file of the same name being overwritten.

[&GetDocument document=TF01,directory="c:\tmp", filename="HistDoc",result=TF02,overwrite]

6 print parameter: print to default printer

Print to the user’s default printer:

[&GetDocument print, document=""]
