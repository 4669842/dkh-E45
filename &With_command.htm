&With

Syntax    

[&With <Target>]

   Further commands

[&EndWith]

Description

&With is used to temporarily change the default owner for part of a script.  It can be used with either a reference or a relation code to indicate the location or target of the new owner.  Once specified, the subsequent scripting will be run against the new default owner until the script encounters an &EndWith command.

The concept of 'default owner' is fundamental to scripting in Visualfiles. The default owner is the file, entity or role in which a script is located at a given point in time during the processing of the script. The default owner changes as the script moves through the elements in the system.

System scripts, by nature, have no default owner and will inherit ownership of the last accessed component as the script progresses.

For example, in very simple terms, say you have a  system script that starts at an entity then references a role and, finally, selects a file.

As the script is processed, and the role is referenced, ownership is transferred to that role and it becomes the default owner. Similarly, as the script is processed and the file is selected, the file will be the default owner.  

The default owner  for all other scripts will be the  file,  entity or role where the processing starts.

Example Scripts

1 Using the ref parameter

This example will select an instance of the entity type CONT and give it a reference of myCONTACT. This entity can then be selected again using &With myCONTACT. The field CON01 can be assigned the value Yes.

[&SelectEntity type = "CONT",ref = "myCONTACT"]

[&With myCONTACT]

   [&Assign CON01 = "Yes"]

[&EndWith]

2 Using a reference and an associated role

This example will select an instance of a relation called CUSTOMER_ANALYST and give it a reference of myANALYST. To manipulate a field on a role of analyst which exists for analyst entities, we can move from our CUSTOMER entity to the analyst role using .ANALYST after the reference and then assign a value to a field on a screen on the analyst role.

[&SelectRelation type = "CUSTOMER_ANALYST",ref = "myANALYST"]

[&With myANALYST.ANALYST]

   [&Assign ANA01 = "Assigned"]

[&EndWith]

3 Using a relation

This example loops through each instance of the relation CLIENT_FILE and then posts an &Schedule command for each related file.

[&ForEach CLIENT_FILE]

   [&With CLIENT_FILE]

      [&Schedule "Schedule to each File"]

   [&EndWith]

[&EndFor]
