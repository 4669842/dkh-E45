&UserInput

Syntax    

[{&UserInput Choices= Default= Format= next-action= next-action-inputs= No-cancel prompt= Result= type="text expression"|"supplier" help= help-label= tooltip=}]

Example Scripts

Description

The &UserInput command replicates the functionally of the existing user input specified with single quotes, for example, ‘Enter your name’.

This command enables standard user input functionality to be extended by using the scripting parameters defined below.

It also enables user input in a stateless environment using the next-action parameter.

Values Input Prompt

The values option is available as an input prompt and on the &UserInput command. Example:

[&Userinput Result=<Field Code>, Values=<value-list>]

Or,

'Prompt text'(values <value-list>)

Where <value-list> resolves to a colon separated list of label=value pairs.

Input Prompt Help Text

Make use of the help text input prompts to display a help button on the input screen and supply tooltips to the end user. Go to Input Prompt Help Text Parameters. See example 5.

Parameters

Choices={text_expression}  

Evaluating to a colon delimited list.

Note: If only one option is required insert a colon after the first choice to display one radio button, see example 4.

Default={text_expression}

Enter the default text you want shown.

Format={text_expression}

Default is no fixed format.

next-action={text_expression}

The next–action parameter is used to end the script run at this point. Any code after the &UserInput command is ignored and the script specified by the text expression in the next-action parameter is run. See example 2.

Text expressions are used to name the required script and can be constructed from any combination of the following:

  • "constant text", such as a script name

  • 'User input'

  • Database fields

  • LF, TF, PI fields. The temporary field specified will contain the value entered by the user.

Note: For more information on stateless scripting options go to no-wait and next-action Parameters, Visualfiles Browser Environment.

next-action-inputs={expression};{expression}...

List of semi-colon delimited items to be passed as PI fields into the next-action script.

No-cancel

Default is to show the Cancel button.

prompt={text_expression}

Default is blank.

Result={Temporary_Field}

This can be TF, LF, PI and will be the field that the user input is stored into.

Note: There is a Values input option available, see example 3.

Type={text_expression}

Evaluating to one of text, number, whole, date, question, choose, filename, time, timeofday or notes. The default is text.

Note: To format the output and to perform tests you need to insert additional code after the &UserInput command, e.g.: &Assign LF01=LF01:date or &Assign LF01=LF01:number.

type="supplier"

Access SOS supplier code lookup. See example 7.

Input Prompt Help Text Parameters

help=<text expression>:

When a value for the help parameter is specified, a help button appears on the input screen. Click the help button to display the specified text.

help-label=<text expression>:

The tooltip on the help button can be changed from its default of ‘Help’ by specifying the help-label= parameter.

tooltip=<text expression>:

When text is specified for the tooltip= parameter, this tooltip appears when the cursor hovers over the input field.

Tooltip and Help text can be displayed across more than one line by use of <cr>  within the specified text, e.g.:

tooltip="line1<cr>line2<cr>line3"

Example Scripts

1 prompt parameter

[&UserInput prompt="Enter your name",result=LF01]

[&Message "You entered:" + LF01]

2 next-action parameter

Any code after the next-action parameter is ignored.

The script specified with the next-action parameter is run after the &UserInput command.

The temporary field specified contains the value entered by the user.

[&UserInput prompt="Enter your age",type="whole",result=LF45,next-action="nextScript"]

[&Message "This line will never be reached"]

The script nextScript includes the command:

[&Message "Your age is:" + LF45]

3 Values input option

[&UserInput prompt="Please choose",result=LF01,values="First Option=A:Second Option=B:Third Option=C"]

[&Message LF01]

Note: The value list resolves to a colon separated list of label=value pairs.

See also Values Input Prompt.

4 Choices parameter

[&UserInput prompt="Select A Colour",choices="red:green:blue",default="red",result=TF01]

Note: If only one option is required insert a colon after the first choice to display one radio button, e.g.:
[&Assign LF03 ="choiceone:"]
[&UserInput prompt="Option 1",choices=LF03,result=LF05,no-cancel]

5 Input Prompt Help Text

[&Assign LF01='Enter details of ID provided'(tooltip="Click on Help for guidance",help="See ML Regs")]

 

[&Userinput prompt="Choose value", choices="choice1:choice2:choice3", result=LF01, help=LF99,help-label="Click here for &help on selecting"]

6 Type parameter - timeofday

Use the timeofday input prompt to input a time value for the time of day rather than a period of time.

The value entered must be between 0.00 and 23.59. It is entered in the format HH.MM.

[&Userinput prompt="Enter Time", type="timeofday", result=LF01]

7 Query SOS suppliers

[&UserInput type=”supplier”]

Calls the SOS Suppliers query, enabling users to select the supplier code in the Supplier dialog which is returned in a local field.
