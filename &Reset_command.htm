&Reset

Syntax    

[{&Reset <screen code>}, [group|owner|role]]

Example Scripts

Description

The &Reset command is used to reset user-defined screens back to their initial values.

Parameters

add-in = "<add-in_name>"

Specify the code of the add-in which contains the screen to be reset.

group = "<GroupCode>"

Display a database screen from the specified group.

owner

Used to reference a file level data from an Add-in. See example 2.

role = "<RoleCode>"

To display a screen from a role, the role parameter must be specified.

Example Scripts

1 Initialise a screen on a group

If you are in the main file and wanted to initialise screen “ABC” on group “ABCD” on add-in “SALEAA”, the syntax is:

[&Reset ABC, group=“ABCD, add-in=“SALEAA”]

2 Initialise fields on screen, add-in

If you are in an add-in and want to initialise the fields on screen ABC on the parent file, the syntax is:

[&Reset ABC, owner]

3 Reset fields on screen, entity

If you are in an entity and want to reset fields on screen “ABC” on role “ALLCLIENTS”, the syntax is:

[&Reset ABC, role=“ALLCLIENTS”]
