&Post

Syntax    

[&Post <TransactionCode>]

Example Scripts

Description

Ledger posting commands are used in documents, forms or scripts to post transactions to the file Ledger. The alternative to using the &Post command is to manually add an item to the Ledger.

A set of transaction codes is predefined for each application type.  TF and DB codes can be used to identify a transaction code.

The &Post command can also be used to post Ledger items within a Billing Module Bill file.

SOS Practice Manager - Specify Funds Transfer Information

You can use the &Post command to send electronic funds transfer information to SOS Practice Manager when posting a ledger transaction in Visualfiles. The information is specified by using the EFT parameters.

Parameters

account = <AccountNumber>

Specify the Office Bank Account to be used in SOS (the bank account the money will be paid out of), or it may be a Client Bank Account for client type postings.

amount = <Amount>

Specify the amount of the posting, if not already specified in the transactions code. This overwrites any amount set in the transaction code.

billable

Mark the Ledger item as billable. LG20 contains the Billing Status.

category

create, category = <Category>

Used only with the create parameter to populate the category of a new Ledger posting. LG21 contains the category.

create

Enables the Ledger item dialog to be called from the &Post command.

CSB-Stage="<csb stage>"

Used to specify the case stage budgeting stage.

date="<Date>"

Specify a date posted for an item rather than use the system default.

desc = "<TransactionDescription>"

Override the default transaction description.

dest-account = "<AccountCode>"

Enable a transfer to another account from within scripting.

This parameter can only be used with the SOS link and within this link it only works with OFFICE TRANSFER and CLIENT-TRANSFER transaction types.

dest-case = "<Internal_Filecode_for_destination_File>"

Enable a transfer to another case from within scripting.

This parameter can only be used with the SOS link and within this link it only works with OFFICE TRANSFER and CLIENT-TRANSFER transaction types.

Note: <Filecode> refers to the internal code MT30 the unique identifier for each file. See Developers' Tip for more information on this parameter.

direct

Enable direct posting to SOS Practice Manager accounts Ledger (this bypasses the Posting Slips screen).

EFT-Account-Entity-Type=

EFT-Account-Entity-Ref=

EFT-Account-Entity-Bank-No=

EFT-Dest-Bank-Name=

EFT-Dest-Bank-Account-Name=

EFT-Dest-Bank-Sort-Code=

EFT-Dest-Bank-Account-No=

EFT-Dest-Bank-BIC=

EFT-Dest-Bank-IBAN=

EFT-Type=

These parameters enable you to send electronic funds transfer information to SOS Practice Manager when posting a ledger transaction in Visualfiles.

Note: All these parameters expect a string expression.

Note: You will need a version of SOS Practice Manager that supports sending EFT information via this method.

noconfirm

Do not display the Ledger screen to the user for confirmation.

notes = "<ExtraNarrativeDetail>"

Update the extra narrative of the Ledger item.

payee = "<{PayeeName|PromptText}>"

Include a payee field. A user can be prompted for the value for the parameter payee.

pencil

A pencilled entry is an anticipated or known future action: transactions posted to the Ledger from a document can be given the status of pencilled. Only one pencilled transaction of each type can be posted.

A pencilled transaction overwrites an existing pencilled transaction with the same transaction code (if the value in the field is likely to change, a pencilled transaction can be repeated to ensure the new value is recorded).

reference="<Value>"

Update the reference of the Ledger item.

supplier=<SupplierCode>

Specify the supplier to be used in SOS.

vat = <VATRate>

Assign the VAT amount during scripting.

vat-rate

create, vat-rate = <{0|1|NV|V}>

Used only with the create parameter to populate the vat-rate of a new Ledger posting.

<TransactionCode>

Transactions are identified by a four letter code, specific to both a case type and a specific action.

Example Scripts

1 Disbursement

This script posts an item using the transaction code, DISN (set to disbursement) and prompts the user to enter a payee name. The script assigns the description Disbursement posting. The user is prompted to enter an amount and the office account O003 is assigned automatically. The user is not prompted to confirm the posting.

[&Post DISN, payee = 'Enter payee name', desc = "Disbursement posting", amount = 'Enter amount', account = "O003", noconfirm]

2 Client payment

This script posts an item with the transaction code, CLIP (set to a client payment.

The user is prompted to enter the payee name, description, amount and a reference but is not prompted to confirm.

[&Post CLIP,payee='Enter payee name',desc='Enter a description',amount='Enter an amount',ref='Enter a reference',noconfirm]

3 Client to client transfer

This script posts an item with a transaction code of CCTR (set to client to client transfer). The user is prompted to enter a destination client case code, description and amount.  The destination bank account is set to C003 and the user is not prompted to confirm the posting.

[&Post CCTR,dest-case='Enter destination case code',dest-account="C003",desc='enter a description',amount='Enter amount',noconfirm]

Developers' Tip

When using the &Post command to create a client to client transfer dest-case = "<Internal_Filecode_for_destination_File>" parameter is used.

The "<Internal_Filecode_for_destination_File>" has to be the internal code MT30 and not the actual matter number MT05.

Here is some sample scripting on how to obtain the internal code for the destination matter:

Script to post client transfer

[&Assign TF09=MT05] *Store current matter number

[&Assign TF06='Enter Destination Matter Number'(Text)]

[&Assign TF80="TRF"]

[&Select MT,code=TF06,ref="DestMatter"] *Select destination matter

[&If status(Select)<>""] *Is it ok?

  [&Message "Cannot post - problem with destination matter "+TF06+": "+status(Select)]

  [&Select MT,Code=TF09] *Select orginal current matter

  [&Quit] *Failed so quit.

[&Else]

 [&With DestMatter]

   [&Assign TF99=MT30] *Store internal code of destination matter

 [&Endwith]

 [&Select MT,Code=TF09] *Select orginal current matter

 [&If status(Select)<>""] *Is it ok?

   [&Message "Cannot post - problem with source matter "+TF09+": "+status(Select)]

   [&Quit] *Failed so quit.

 [&Else]

   [&Assign TF21='Enter the amount you wish to transfer to Client account'(number)]

   [&Assign TF03='Enter a description for this transaction'(text)]

   [&Post CLLC,amount=TF21,dest-case=TF99,account="C099",dest-account="C099",Desc=TF03,reference=TF80]

 [&Endif]

[&Endif]
