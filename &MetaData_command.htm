&MetaData

Syntax    

[&MetaData <type>]

Example Scripts

Description

Opens the metadata screen for the current item of the selected history item type.

It enables metadata to be added and amended. In the script the history item you want to add metadata for must be selected first and then the &Metadata command is applied.

History Plus Scripting Options

Note: Additional History Plus scripting information is available at Developer Information.

Bulk Update Custom Metadata Fields

In History Plus you can bulk update all items for custom metadata fields in a file via scripting. Three optional parameters are available on the &Metadata command that enable you to do this:

  • mt-int-code=
  • name=
  • value=

Note: These parameters are only used for updating metadata held in History Plus.

Store Time and Document Metadata in File History

It is possible to store both Time and Document metadata against file history items. When a history item is created manually with time attached then a document is imported to the history item, you can create and update both Time and Document metadata.

The document parameter and the parameter :D (that is used in conjunction with the 99 system codes) enable you to control which metadata screen fields are updated.

Update Metadata in History Plus

The submit parameter is used to control when metadata is updated for history documents in History Plus. When metadata is created and updated via scripting, each field on the metadata screen is updated individually (HS99(1) etc.). To avoid updating History Plus metadata every time we assign to a metadata screen field, the submit parameter is used to control when History Plus is updated.

Parameters

Create

Create metadata screens with no user input.

Enables you to create a new blank metadata screen with no screen display and with no user involvement. The metadata screen created in this way can then be updated with the relevant '99' system code (see Display and Update Metadata Fields), e.g. for history items:

[&Assign HS99(1)="First field"].

The create parameter can only used to add metadata screens silently. If a metadata screen already exists then the &Metadata command is ignored when the create parameter is included.

If the &Metadata command is used with no create parameter and there is no existing metadata, the metadata screen is displayed as before.

Debt

Debt invoice.

Document

Enables you to specify document metadata. See example 4.

It is used in conjunction with the :D parameter that accompanies 99 codes. See Display and Update Metadata Fields.

History

History item.

Ledger

Ledger item. See example 5.

mt-int-code="<FileInternalCode>"

Holds the internal code of the file (MT30) to search the History Plus.

This is an optional parameter that is used for running a bulk update of metadata held in the History Plus. See example 3.

If this parameter is not specified the current MT30 value is used.

name="<Field_name>"

Holds the custom metadata fields that are to be updated.

This is an optional parameter that is used for updating metadata held in History Plus. See example 3.

If this parameter is missing from your script the bulk update of metadata is not performed because there is no field defined to update.

Payment

Payment item.

Schedule

Schedule item.

Submit

Forces an update of history document metadata in History Plus. See example 4.

Note: The submit parameter is only required for systems connected to History Plus because its sole purpose is to control when HS99 assigns are updated on History Plus.

In order to update custom metadata in History Plus, use the command [&Metadata history ,submit] to bulk update documents with edited metadata. This method prevents each individual piece of metadata being updated after every &Assign statement. Refer to example 3

value="<value>"

Holds the value to assign to the name= field. If value= is not specified then blank is assigned.

This is an optional parameter that is used for updating metadata held in History Plus. See example 3.

Example Scripts

1 Display metadata screen

Display the metadata screen for the most recently added history item.

[&MetaData history]

2 Create a new history item with attached metadata

Assign metadata fields:

[&History "Test", save]

[&Select History:last]

[&Metadata History, create]

[&Assign HS99(1)="Metadata created"]

This code creates a metadata screen for the selected history item and populates the second field.

Note: A history item can have both a Time Metadata screen and a Document Metadata screen attached. When this occurs the Time Metadata screen is the default metadata screen and can be created using the standard [&Metadata history, create] command, to create or access the Document metadata screen use the additional ,document parameter, e.g. [&Metadata history, document, create].

3 History Plus parameters

Bulk update of all history items in a file with the specified metadata field and value via scripting:

[&Metadata history, mt-int-code=LF01, name="Standard Metadata", value="Metadata Testing"]

 

The following syntax looks in History Plus for all metadata on the 'Standard Metadata' customer field for the file with MT30 = 100 and assign 'Metadata test' to it.

[&Metadata History ,mt-int-code="100",name="Standard Metadata",value="Metadata test"]

4 History Plus Document and Submit parameters

In this example the last history item is selected and a document metadata screen attached to it:

[&Select HS:last]

[&MetaData history, document, create]

[&Assign HS99(1:D)="Metadata field 1"]

[&Assign HS99(2:D)="Metadata field 2"]

[&MetaData history, Submit]

The two assigns populate the data on the document metadata screen.

In the event that the history item has Time attached to it, these document and :D parameters ensure that the metadata is stored for Document not Time.

The addition of the submit parameter updates the Document Repository when this line is processed.

Note: The submit parameter is only required for systems connected to Document Repository as its sole purpose is to control when HS99 assigns are updated on the Document Repository.

 

Edit metadata using the submit parameter to commit the changes to metadata in one hit:

[&Select history:last]

[&Metadata history, create]

[&Assign HS99(1)="A1 /E1"]

[&Assign HS99(2)="01/01/2017"]

[&Assign HS99(3)="Leeds"]

[&Assign HS99(4)="Debt"]

[&Metadata history ,submit]

 

Manually edit metadata to display the metadata screen to the users and commit the changes made by the user to metadata using the submit parameter:

[&Select history:last]

[&Metadata history]

[&Metadata history ,submit]

 

Edit and update metadata to display to the user for verification and submit to metadata:

[&Select history:last]

[&Metadata history, create]

[&Assign HS99(1)="A1 /E1"]

[&Assign HS99(2)="01/01/2017"]

[&Assign HS99(3)="Meta test"]

[&Assign HS99(4)="Test"]

[&Metadata history]

[&Metadata history ,submit]

5 ledger parameter

The &Metadata command can also be used to display the metadata screen to the user:

[&Select ledger:last]

[&Metadata ledger]

 

Metadata codes can be used to retrieve metadata:

[&Select payment:first]

[&Message PY99(01)]

This code returns the data stored in the first field of the metadata screen associated with the selected payment item.
