&Output

Syntax    

[&Output Name = <Name>]

Example Scripts

Description

The &Output command is used within the Data Extract Module to define the parameters of the file to which the extracted data will be output.

&Output and Visualfiles Browser

&Output is supported in Visualfiles Browser but its use is restricted because scripts run on the server not the client PC; this means that output files are saved to the server not the client.

Parameters

access = <{Create|Append|Update|Delete|Overwrite}>

Specifies how each output row is to be handled. The default is Append.

close

If the output type is Excel, specifying this parameter closes any open connections.

Tip: The close parameter could be used before opening a second Excel spreadsheet to ensure the first was closed. Alternatively, separate the outputs into different scripts and call them.

delimiter = <Delimiter>

The character chosen to delimit the fields. The default is a comma (,).

fields

Fields to output: a delimited list. No default is set for this parameter, but if no fields are supplied, the &Output command will not have any data with which to modify the destination file. For Text or Excel outputs, this will most likely result in a blank file, while an empty &Output command to a database may produce errors.

Each field within the delimited list may have a number of secondary parameters:

name=<title>

Allows the user to specify the column header for the output field.

type=<datatype>

Specifies the type of data the field contains

<{char|num|int|date|autonumber|notes}>

When using &OutPut command to send data to an SQL database, numeric values can be specified as Type="INT" (created as an Integer datatype) or Type="NUM" (created as a Decimal datatype).

index

Denotes that the output field is an index field.

primary

Denotes that the output field is the primary index.

unique

Specifies that the field must be unique in the destination file.

header = <{Yes|No}>

Specifies whether headers should be added to output. The default is No (except when the output destination is an Access file, where columns inherently have headers).

index = <{Yes|No}>

Denotes whether index fields should be added to the output file. The default is No when the access parameter is Append or Create; where the access parameter is Update, Delete or Overwrite, an index field is needed in the output file.

length = <length>

Specifies the length of the character or decimal field.

log=

Used to create a log file and in a specified directory. See example 6.

log="path\directory_name"

Name = <Name>

The name identified in the &OutputTo command which acts as a reference. No default is set for this parameter.

no-validate

Specifies whether table schema validation should be carried out.

primary

Denotes that the output field is the primary index.

table = <TableName>

The name of the table to output to. No default is set for this parameter.

titles = "<{fldno|fldno|UserDefinedDescription}>"

Field description format. To display field numbers in the table, rather than  descriptions, use title=fldno.

type = <{char|num|int|date|autonumber|notes}>

Specifies the type of data the field contains

unique

Specifies that the field must be unique in the destination file.

warehouse=<{Yes|No}>

Specifies that the script is of type Data Warehouse.

Example Scripts

1 File output

This single line of script will append a line, containing the details of a file, to the specified destination file.

[&Output Name = "Example1",Table = "Files",Access = "Append",MT05,MT01,MT04,MT09,MT11]

2 Use of the &OutputTo command in creating a destination file

This example shows how the &Output command is used in conjunction with the &Outputto command.

[&OutputTo Name = "example2",File = "C:\Example2.xls",Type = "excel",Access = "Create"]

[&ForEach PRIV]

   [&Output Name = "example2",Table = "Private Entities",Access = "Create",EN02,EN06]

[&EndFor]

3 Using the secondary parameters to name fields

[&Output Name = "TotDisbFactStaging",Table = "TotDisbFactStaging",Type = "excel",Access = "append", LG22:WHOLE{Name = "Ledger_ID"},LG04{Name = "Date"}, LG06{Name = "Case_ID"}, LG08{Name = "Trans_Code"}, LG13{Name = "Bill_Status"}, LG02{Name = "Net"}, LG03{Name = "VAT"}, LG01{Name = "Total"}]

4 Using standard modifiers to format output

Rather than outputting the value 123.00, this example outputs 123 for the primary key of the TotDisbFactStaging table in an Access database. This is important when updating or deleting rows, as Access can see 123 and 123.00 as different values.

[&Output Name = "TotDisbFactStaging",Table = "TotDisbFactStaging",Access = "Update", LG22:WHOLE{Unique = yes,Primary = yes,Index = yes},LG04, LG06, LG08, LG13,LG02,LG03,LG01]

5 log parameter

Create a log file in a specified directory:

LOG="C:\SQL_LOG"

6 Testing for errors using Status system variable

[&Assign LF01='Enter Matter Ref']

 [&Assign LF02="a Desc"]

[&Assign LF03="01/02/2s001"]

[&Assign LF04="1"]

[&Assign LF05="1"]

[&Assign LF06="1"]

[&Assign LF07="7"]

[&Outputto name="Warehouse",warehouse=yes, log="c:\sql-log",file="Warehouse;sa;;DataWarehouse;Sql Server;localhost",type="ODBC",access="update",LOG="C:\sql-log\"]

[&Message  status(outputto)]

 [&Output name="Warehouse" ,warehouse=yes,log="c:\sql-log",table="test_tbl",access="update",LF01{name="MatRef",type="char",unique=yes,primary=yes,index=yes,length=50},LF02{name="MatDesc",type="char",length=20},LF03{name="mydate",type="date"},LF04{name="bigint",type="int"},LF05{name="MYDEC",type="char",length=20},LF06{name="MYDECa",type="char",length=20},LF06{name="MYDECb",type="char",length=20}]

[&Message status(output)]

This script outputs details to a test table and outputs any errors to the screen using &Message and to a log file in c:\SQL-LOG. Log files are named LNVF-DEBUG-99-99-9999.log

Some ways of generating an error include:

  • Entering invalid data in a field, for example, a date of 01/012a.2001

  • Trying to update a database that does not exist

  • Trying to append a row where a row with that key already exists in the database

The same error in the log file would be:

15:41:34 | LNVF Error Number: LNVF00003| Source: Microsoft Cursor Engine| Number: -2147217887| Description: Multiple-step operation generated errors. Check each status value.:| Further Details: Error writing data to field LF03 (mydate) Value = 01/02/2s001

7 Notes parameter

[&Output Name="Test",Table="Test1",Access="Append",Mt30{Name="Code",Type="int",index="yes","unique=yes",primary="yes"},PIT01{Name="Notes",Type="notes"}]

Developers' Notes

The &Output command must be used in conjunction with the &Outputto command; without a corresponding &Outputto command to provide an output record, the user will see an error.

As an alternative to hard coding the parameters, you can also use TF fields for the following parameters:

  • Name
  • Table
  • Access
  • Delimiter
  • Header
  • Titles
  • Indexing
  • XML

Test if the &Output Command Worked

The system variable Status, is used to return the status of the last execution of selected commands including the &Output command.

Error trapping can be reported in two ways:

  • Use status(output) in scripts. See example 6 above.
  • Specify the parameter log=. See the log= parameter above.

Note: Messages are displayed on screen if an error is unexpected and therefore is not being trapped, or if the message requires user interaction.
