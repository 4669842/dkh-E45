&Event

Syntax    

[&Event EventCode]

Example Scripts

Description

The &Event command automatically calculates interest when an event is triggered in a Debt Collection application. These events, triggered by an &Event command in a document, form or script, determine when and how interest on the debt is calculated.

If an event is triggered more than once for a given case, a warning message is displayed and the user is prompted to recalculate the corresponding values.

&Event is used only in applications with a type of Debt Collection.

Note: This command is not supported in Visualfiles Browser.

Parameters

?Date

[&Event CISSUE,?DATE]

[&Event CJUDGE,?DATE]

[&Event HISSUE,?DATE]

[&Event HJUDGE,?DATE]

Prompts the user to enter an alternative date to trigger the action.

CIssue

Run when County Court proceedings are issued.

Enter the CISSUE event code into the Issue Milestone script or an item which is scheduled from the issue milestone. When the file reaches the issue stage, this will be run. As interest is accumulating through the lifetime of the file, but cannot be demanded until proceedings are issued, the CISSUE event code will calculate the interest.

CISSUE will: set the County Court Issue Date (DC10) to today's date, set the County Court Issue Amount (DC11) as the Total Debt Due on the file, and the County Court Interest at Issue (DC12) to the amount of the Interest on the file up to today's date.

CJudge

Run when County Court judgment is entered.

Enter the CJUDGE event code into the Judgment Milestone script or an item which is scheduled from the Judgment Milestone.

CJUDGE will set County Court Interest at Judgment (DC34) to the date when the judgment was requested. Further, the amount of the debt will be used to calculate the interest. If the debt amount is less that £5000, the accumulated interest will remain but no more will be added. However, if the debt amount is more than £5000, any future interest is set at 8%.

HIssue

Run when High Court proceedings are issued - the HISSUE parameter may be used in the same manner and with similar syntax to the CISSUE parameter.

HJudge

Run when High Court judgment is entered. The HJUDGE parameter may be used in the same manner and with similar syntax to the CJUDGE parameter.

IntOn

Allows the interest to be added to the debt where proceedings have been issued.

X-CIssue

Cancels a CISSUE event.

The script sets the corresponding DC code to zero for number fields or blank for the date field. Any user-defined data fields, the Ledger and Milestone must all be cleared manually.

X-CJudge

Cancels a CJUDGE event.

The script sets the corresponding DC code to zero for number fields or blank for the date field. Any user-defined data fields, the Ledger and Milestone must all be cleared manually.

X-HIssue

Cancels a HISSUE event.

The script sets the corresponding DC code to zero for number fields or blank for the date field. Any user-defined data fields, the Ledger and Milestone must all be cleared manually.

X-HJudge

Cancels a HJUDGE event.

The script sets the corresponding DC code to zero for number fields or blank for the date field. Any user-defined data fields, the Ledger and Milestone must all be cleared manually.

Example Scripts

1 CISSUE

This code calculates the interest, using the INTON event and set the codes DC10, DC11 and DC12 using the CISSUE event. The Issue Milestone script schedules the form UN1. This extract is from the prescript of UN1.

[&If 'Calculate costs for Claim Form?'(Question) = "Yes"]

   [&Event INTON] * Sets Interest Calculations on *

   [&Event CISSUE] * Sets the Issue Status *

[&EndIf]

2 CISSUE

Where forms are in use, the CISSUE event may be included as the first command in the N1 County Court Issue form. The necessary interest calculation is then carried out at the head of the form, so that the results may be reflected in the values printed further down the form.

3 CISSUE

The CISSUE event may be in included in a separate script which updates the database with the necessary interest and balances and directs the To-Do list to generate the appropriate documents.

4 CJUDGE

The Judgment Milestone script will schedule a Request for Judgment form UN22: this is an extract of a script in the form's prescript. This will calculate the County Court Interest at Judgment, stored in DC34.

[&If 'Do you wish to calculate the costs for the Judgment form?'(Question) = "Yes"]

   [&Event CJUDGE]

[&EndIf]

5 Using the Escalate parameter

Schedule the message Completion Overdue one working day after the date held in COM01 for the supervisor of the file's File Holder.

[&Schedule,"Completion Overdue", Date-due = COM01+1(Working days),Escalate]
