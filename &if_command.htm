&If

Syntax    

[{&If <subject> <test> <object>}

[&And{<subject> <test> <object>}|&Or{<subject> <test> <object>}|&Xor{<subject> <test> <object>}]...]

<Text or further Visualfiles commands>

[[&Else]]

<Text or further Visualfiles commands>

[{&EndIf}]

Example Scripts

Description

A conditional statement that tests the value contained in a field and then carries out additional commands based upon the result of the test.

Each &If command must have a corresponding &EndIf command.

A simple conditional command comprises three components:

  • <subject>
  • <test>
  • <object>

Alternative text can be:

  • Assigned using the &Else command.
  • Combined using one of the joins.
  • Added using an Input Prompt.

Error Trapping

The &If command can be used in combination with the Status system variable to test if a selection exists in order to trap errors on selected commands when scripts are run.

Parameters

<subject>

The subject to be tested can be any type of user input: direct text/number/date input, a database field, a temporary field, a global field, an input parameter or an output parameter.

The <subject> and <object> must be the same type: text must be compared to text, numbers to numbers, date to date.

<object>

The <object> of the test can be any type of user input: direct text/number/date input, a database field, a temporary field, a global field, an input parameter or an output parameter.

The <subject> and <object> must be the same type: text must be compared to text, numbers to numbers, date to date.

(Question)Where there is not a specific database field to test against, a user can be prompted for a Yes/No response using the question modifier, see example 6.

<test>

The <test> used may be any of the following:

=

<>

>

<=

>=

&Contains

&Begins

&IsOneOf

Equal to

Not equal to

Greater than

Less than or equal to

Greater than or equal to

Tests if text in the Subject contains the Object value.

Tests if text in the Subject begins with the Object value.

Tests if text in the Subject is one of the list of values.

The following commands can be used to join tests:

&And

&Or

&Xor

Condition is met if both tests are true.

Condition is met if either test is true.

Condition is met if only one test is true.

&Else

An optional part of the &If construct. It offers a second group of statements which are executed if the condition is not true in the first group of statements.

&EndIf

Terminates the &If command.

Example Scripts

1 Date testing

Tests whether today's date is the date stored in database field ABC01.

[&If ABC01 = DATE]

   [&Message "Today's the day"]

[&EndIf]

2 &If/&Else testing

Tests whether a file is open (as stored in the database field ABC01), and prints an appropriate message.

[&If ABC01 = "Open"]

   [&Message "File Open"]

[&Else]

   [&Message "File Closed"]

[&EndIf]

3 String testing

Uses &IsOneOf to check whether the individual has the surname Smith, Jones or Patel.

[&If NAM01 &IsOneOf "Smith,Jones,Patel"]

   [&Message "Individual has a common surname"]

[&EndIf]

4 Looking for text within a notes field

EFG01 is a notes field belonging to the group code ABCD, this example uses an &ForEach command to count the number of members that mention Smith in the notes field.

[&Assign TF01 = 0]

[&ForEach ABCD]

   [&If EFG01 &Contains "Smith"]

      [&Assign TF01 = TF01 + 1]

   [&EndIf]

[&EndFor]

[&Message TF01+" Smiths were mentioned"]

5 Using &Or as a join

Where DAY01 contains the day of the week, this example checks whether the day specified is a weekend or working day.

[&If DAY01 = "Saturday" &Or DAY01 = "Sunday"]

   [&Message "It's the weekend."]

[&Else]

   [&Message "It's a working day."]

[&EndIf]

6 Question modifier

[&If 'Is Nick good at golf?'(Question)="Yes"] Nick is good at golf!

[&Else] Nick is bad at golf

[&EndIf]

7 Add a database field as part of a question via scripting

[&If 'Do you want to write to an alternative address?'(Question)="Yes"]

Where the alternative address is held in field ABC08.

The contents of field ABC08 can be added as part of the message so that the prompt reads:

"Do you want to write to alternative address -1 High Street?"

This can be achieved using the prompt parameter on the &If command; e.g.:

[&If prompt("Do you want to write to alternative address:"+ABC08)(Question)="Yes"]

[&Endif]
