&Count

Syntax    

[&Count type = "<Group>",code = <GroupCode>,result = <FieldName>]

[&Count type = "<Relation>",result = <FieldName>]

[&Count type="<Relation>",from-type="<Case>",int-code=MT30,to-type="<Entity>",result=LF01]

[&Count type="<Relation>",from-type="<Entity>",int-code=EN56,to-type="<Case>",result=LF01]

Example Scripts

Description

Provides a quick method for counting the number of instances of a group or relations on a file or entity.

When viewing screens a default uddetail record is created. &Count returns a value if there is a blank iteration of a group screen such as when a screen has been viewed but no data entered, or where an iteration of a group has been created but then deleted. Use the field and value parameters to exclude these default screens.

When counting relations, all relations for that file or entity are returned by default. Use the From-Type, Int-Code and To-Type to restrict the type of relations you want to count.

Parameters

code = <GroupCode>

Specify the code of the group to be counted. If this parameter is not included, the error message You must specify a valid code is displayed.

field = <FieldName>

Enter a relevant field to the group. If this is not a field name belonging to the group, a count value of 0 is returned. If this is blank or not included in the command the count function works as before.

Relation fields not recommended; they are only populated when a field is amended on the screen.

Note: field = pointer field does not work.

From-Type

File/case or entity. This parameter must be used if Int-Code = is also populated.

  • Blank or not present = From Current Entity/Case
  • “Entity or E” – Entities related
  • “Case or C” – Files/cases related

Int-Code

Int-code of file or entity from which to calculate relations. Return the correct count for a specified type and count.

This parameter must be used if From-Type = is also used.

  • Blank or not present = Int-code of current entity/file is used
  • Entity/file Internal code i.e. MT30 / EN56

No-Duplicates

Count multiple links to an entity or case as one link. Returns the count of entities and cases linked to, rather than relation count.

result = <FieldName>

Specify where the count is to be stored.

status =  

Enables you to return a count of All files, or Open files, or Completed files.

The status parameter can only be used where to-type = case. If it is used with any other &Count parameter it is ignored. See example 8.

The options available for status= are:

  • All (By default the command looks for all files)
  • Open or O
  • Complete or C

To-Type =

Restrict the relation count returned to either entities or files.

To-Type =

  • Blank or not present = All relations
  • “Entity or E” – Entities related
  • “Case or C” – Cases related

type = "<Group>"

Specify that the command is to count the instances of a group. If this parameter is not included, the error message You have specified an incorrect 'type' is displayed.

type = "<relation>"

Specify that the command is to count the instances of relations. If this parameter is not included, the error message You have specified an incorrect 'type' is displayed.

value =

This parameter is used to store the condition for the field, e.g.: "yes", "0.00", "reference". The count only includes iterations where this condition is true.

If this parameter is not included but the field parameter is, then a count of all iterations where the given field is not blank or 'no' (if field is a check box) is returned.

Note: The parameters code=, field= and value= are only used when counting group iterations. The parameters from-type=, int-code= and to-type= are only used when counting relations.

Example Scripts

1 Fundamental use of &Count

Store the number of instances of the group ABCD to the temporary field TF01:

[&Count type = "GROUP",code = "ABCD",result = TF01]

2 Choosing the group to count

[&Assign TF02 = ’Count which group?’(Choose ABCD:EFGH:IJKL)]

[&Count type = "GROUP",code = TF02,result = TF01]

Note: Use the &Assign command to specify which group you wish to count.

3 Storing the result to a database field

&Count type = "GROUP",code = "ABCD",result = UDF01]

Note: If the result of a count is not to be used immediately, it may be more convenient to store the result to a database field rather than a temporary field.

4 Field and value parameters

The &ForEach command needs to be used with these two examples:

[&Count type = "GROUP",code=TF01,result=TF99,field=TF02,value=TF03]

[&Count type = "GROUP",code=TF01,result=TF99,field="ADO20",value=""]

5 Return a count of all relations from the current entity or file

[&Count type = "Relation",result = <FieldName>]

6 Return a count of all relations to entities from the current case

[&Count type="Relation",from-type="Case",int-code=MT30,to-type="Entity",result=LF01]

7 Return a count of all relations to cases from entities

[&Count type="Relation",from-type="Entity",int-code=EN56,to-type="Case",result=LF01]

8 status= parameter

Return a count of all relations to cases from entities along with a count of all files:

[&Count type="Relation",int-code=EN56,from-type="Entity",to-type="Case",status="all",result=LF01]

Developers' Tip

The &Count command can replace an &ForEach loop whose only purpose is to calculate the number of group members, making the code simpler to read and more efficient to execute.
