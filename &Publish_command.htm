&Publish

Syntax    

[{&Publish | int-code=< >|,msg-code=< >}]

   [, Acknowledge]

Example Scripts

Description

This command is used to silently publish messages at any point from Visualfiles to a system that is connected using the Integration Framework.

Parameters

int-code=<EntityInternalCode|MatterInternalCode>

Specify the entity internal code (EN56) for the current selected entity, or the matter internal code (MT30) for the current selected matter.

msg-code=<VIFMessageCode>

Specify the free form code assigned to the message being sent.

Acknowledge

An optional parameter used in situations where the connected system requires an acknowledgement, along with a resulting data value, to confirm the operation has successfully completed. For example, when a connected system sends a message to create a matter, it requires an acknowledgement along with the Visualfiles internal code for the new matter to be sent back on completion.

Example Scripts

1 Error handling

Use the standard VBL error handling features to trap any errors raised by the &Publish command:

[&Publish int-code=MT30, msg-code="VF001"]

[&If status (publish) <> ""]

  [&Message "Error = " + status (publish)]

[&EndIf]

Operation

This command performs a lookup of the message using the msg-code specified, validates that the entity type or matter type corresponds to the object intending to send the message, then creates an entry in the message outbound queue with all the data fields populated as defined for the message. There is one message for each system flagged as an outbound recipient and each message contains the target system code.

As a standard part of the VBL, the &Publish command is parsed like all other commands, using standard Visualfiles programs to create a row in the msg-out table which lives in the core production database. It is the outbound processor which takes the message from this table and transfers it to the intermediate SQL database using ADO through ODBC.

Developers' Tips

It is possible to call out to a connected system (connected using the Integration Framework) and to execute one of its stored procedures that requests data back to Visualfiles. This feature utilises the &Call command's dsn and procedure parameters, which are used to invoke other VBL scripts or sub-routines within VBL scripts.

The integration can be temporally turned off using the &Sim command e.g. [&Sim VIF, off]. This enables you to stop update messages from being sent out whilst you are creating new items. You can then use the &Publish command to send a single message containing all the updated fields. Once the message has been sent turn the integration back on using the on parameter e.g. [&Sim VIF, on].
