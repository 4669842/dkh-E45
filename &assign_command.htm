&Assign

Syntax    

[{&Assign Code = <Value>}]

Example Scripts

Description

The &Assign command is used in documents and scripts to assign a value to a field. It can also be used to:

  • Assign properties to elements such as buttons.

  • Assign characters from strings using the :char modifier.

  • Assign words from strings using the :word modifier.

  • Access SOS supplier code lookup (see example 19).

Assigning Button Properties

Some of the properties of a button can be assigned from a script. The button's label, its associated script and its current state may be overwritten by an assigned value. See examples below.

History Notes Field, HS14

You can assign a value to the History Notes Field HS14 using the &Assign command, either by default or by flagging another field; i.e., a System Default or another HS field which makes HS14 writable. This enables you to write back to HS14 when importing data.

Parameters

<FieldCode> = "<Value>"

When the value is a number it does not need to be enclosed in double quotation marks.

:proval<>

Converts string into a Progress command.

quote <>

Used to represent a double quote when adding text together.

Example Scripts

1 Assign a gender type to a field

[&Assign NAM15 = "Female"]

2 Prompt to enter a price and assign value to a field

Prompt a user to enter a purchase price and assign that value to the PRO07 field.

[&Assign PRO07 = 'Enter Purchase Price of Property'(number)]

3 Assign a user's choice to a field

Assign a user's choice to the PRO06 field.

[&Assign PRO06 = 'Specify whether a mortgage is required'(Choose unknown:yes:no)]

Tip: You can include an ampersand in the choose label, for example one of the choices could be "Black & White".

4 Assign a telephone number to a file from a related entity

[&Assign CLIENT_CLIENT.EN47 = "0113 226 2020"]

5 Assign temporary fields and make calculations

[&Assign TF01 = 1.175]

[&Assign TF02 = ABC01*TF01]

ABC01 is the cost of a stock item excluding VAT. The temporary field TF01 is given the value of 1.175, which can then be multiplied by ABC01 and allocated to TF02, to give the price inclusive of VAT at 17.5%.  

6 Amend the Fee-Earner (AD10) associated with a Schedule Item

The command accepts any string, User-Defined field, Database field or Temporary field which resolves to either a valid Fee-Earner ID or Fee-Earner Name. If an invalid ID or Name is specified then the current Fee-Earner associated with the Item is retained.

[&Assign AD10 = ABC01]

7 Modify a date to be output in text

[&Assign TF01="29-01-04"]       *** Set your date field here ***

[&Assign TF02=TF01:ds("dd")]  *** Get the day number ***

[&Assign TF03="First,Second,Third,Forth,Fifth,Sixth,Seventh,Eighth,Ninth,Tenth,Eleventh,Twelfth,Thirteenth" +

",Fourteenth,Fifteenth,Sixteenth,Seventeenth,Eighteenth,Nineteenth,Twentieth" +

",Twenty First,Twenty Second,Twenty Third,Twenty Fourth,Twenty Fifth,Twenty Sixth,Twenty Seventh,Twenty Eight,Twenty Ninth,Thirtieth,Thirty First"]

[&Assign TF04 = word( TF03, first=(TF02:number), delimiter=",")]

[&Message TF04]

8 Assigning button properties

[&Assign OPY11 = "Office,OPOffice,Greyed"]

OPY11 represents the button code in a user-defined screen, Office is the label, OPOffice is the associated script and the button has been Greyed to prevent its selection.

[&Assign CLI03="Office,,Greyed"]

CLI03 represents the button code in a user-defined screen, Office is the label and the button has been Greyed to prevent its selection. The second parameter has been left blank: the script will default to the value in the Screen Builder.

[&Assign CLI02="Commercial Client,S-GEN-201,"]

CLI02 represents the button code in a user-defined screen, Commercial Client is the new label, S-GEN-201 is the new associated script and the button is not Greyed.

9 Wrap a long text string over several lines of defined length

Format a long text string into lines of 30 characters maximum:

[&Assign LF02=LF01:down(30,"|")]

[&Assign LF03=word(LF02, first=1, delimiter="|"]

[&Assign LF04=word(LF02, first=2, delimiter="|"]

For example:

Large Bank International Limited

becomes:

Large Bank

International Limited

10 Specify the location when selecting a document

[&Assign TF02="C:\temp"]

[&Assign TF01='Select a file'(filename,default=TF02)]

11 Count the characters in a word or text string

[&Assign LF02=length(LF01)]

12 Locate function

Search for a phrase in a string rather than just a single word.

[&Assign LF998=Locate("upon", text=once upon a time, delimiter= )]

13 Enter Password - bulleted text

Text entered in the password field are represented as bullets.

[&Assign LF01='Enter Password'(password)]

14 Assign a string to a TF field that contains speech marks

[&Assign TF01="md "+Quote+"c:\test one\test two"+Quote]

15 Prompt to enter a price and assign default value to the field

[&Assign TF01 = 'enter price'(number,default = 10000)]

Note: Use default= without 'number' to define a text string in the field.

16 Test against a time limit

To test against a time limit in hours entered in a number field against TA03 (total time), and warn end users when files are approaching their time limits.

[&Assign TF01=TA03:hrs]

Note: This code returns the number of hours only. Use the :mins modifier if you need to calculate a more accurate figure, go to Time Modifier for more information.

17 Add leading zeros

Add leading zeroes for example when creating a reference number:

[&Assign LF01="123"]

[&Assign LF01=LF01:number:text("99999999")]

[&Message LF01]

18 Test if field is numeric

Script to test if a field is numeric.

[&Assign TF01="test 123"]

[&Assign TF01=123]

 

[&Assign TF01=TF01:number]

[&If TF01>0]

[&Message "Has characters"]

[&Else]

[&Message "Is all numbers"]

[&Endif]

19 Query SOS suppliers

[&Assign LF01=’ ’(supplier)]

Calls the SOS Suppliers query, enabling users to select the supplier code in the Supplier dialog which is returned in a local field.
